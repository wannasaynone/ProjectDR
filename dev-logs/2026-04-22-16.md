## 根因分析

### Bug 現象

製作人 D7 playthrough：守衛首次進入互動畫面時，沒有觸發 `guard_first_meet` 對白，導致劍未發放、探索關卡無法重開。

### 根因：StartDialoguePlayback 覆蓋了已啟動的 NodeDialogue

A08 併入 NodeDialogueConfig（dev-log-10）後，觸發路徑如下：

```
OnNavigatedToArea(Guard)
  ├── InitializeCharacterView(Guard)
  │   ├── HasPlayedIntroCG(Guard) = true（GuardReturn 已呼叫 MarkIntroCGPlayed）
  │   └── → else 分支：OnCharacterEnteredAndCGDone(Guard, view)
  │           └── TryPlayFirstMeetDialogueIfNotTriggered("guard_first_meet")
  │               └── PlayNode → _dialogueManager.StartDialogue(guard_first_meet 4 行)  ← A 點
  │                   （此時 View 尚未 active）
  └── _stackController.PushView(Guard)  ← View 變 active
      └── OnShow()
          └── StartDialoguePlayback()  ← B 點
              └── _dialogueManager.StartDialogue(招呼語)  ← 覆蓋 A 點！
```

**A 點**：`TryPlayFirstMeetDialogueIfNotTriggered` 在 View active 前啟動 DialogueManager 播放 guard_first_meet（4 行對白）。

**B 點**：View active 後，`OnShow` 調用 `StartDialoguePlayback`，後者未檢查 `DialogueManager.IsActive`，直接以招呼語呼叫 `StartDialogue`，**覆蓋了 guard_first_meet**。

後果：`NodeDialogueCompletedEvent { NodeId="guard_first_meet" }` 永遠不發布，劍未發放，`ExplorationGateReopenedEvent` 未發布。

## 修復內容

### 修改：`CharacterInteractionView.cs`

在 `StartDialoguePlayback()` 中，`_dialogueManager.StartDialogue(dialogueToPlay)` 呼叫前加入保護：

```csharp
// 若 DialogueManager 已由外部（如 NodeDialogueController）啟動對話，不覆蓋。
// 此情境：OnCharacterEnteredAndCGDone 中 TryPlayFirstMeetDialogueIfNotTriggered 已呼叫
// PlayNode → StartDialogue，但 View 尚未 active；View active 後走到此處時應接手顯示，
// 而非用招呼語覆蓋。只播第一行打字機即可，後續由玩家點擊 Advance 推進。
if (_dialogueManager != null && _dialogueManager.IsActive)
{
    string activeFirstLine = _dialogueManager.GetCurrentLine();
    if (activeFirstLine != null && _typewriterEffect != null)
    {
        _typewriterEffect.OnComplete += OnTypewriterLineComplete;
        _typewriterEffect.Play(activeFirstLine, _typewriterCharsPerSecond);
    }
    return;
}
```

語義：「若已有外部對話在播放（如 NodeDialogue），View 只接手 UI 顯示，不重啟對話。」

### 新增：`GuardFirstMeetDialogueIntegrationTest.cs` — T1b 回歸測試

```
Regression_D7_AfterTryPlayFirstMeet_DialogueManagerIsActiveWithGuardLine
```

驗證：
- `TryPlayFirstMeetDialogueIfNotTriggered` 呼叫後，`DialogueManager.IsActive` 為 true
- `GetCurrentLine()` 回傳 guard_first_meet 第一行台詞，而非招呼語

## 影響分析

**此修改的副作用**：若有其他情境在 View active 前啟動 DialogueManager（目前只有 guard_first_meet 路徑），也會走到這個保護分支。這是期望行為。

**不影響的路徑**：
- 正常招呼語播放（DialogueManager 在 OnShow 時為 inactive）
- ExternalDialogueMode（有獨立的 `_externalDialogueMode` flag 保護）
- FirstMeetOverrideDialogue 路徑（`_firstMeetOverrideDialogueLines != null` 先判斷，優先於 IsActive 保護）
