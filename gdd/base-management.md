# 基地經營系統

> **版本**: v1.0

## 系統目的

基地經營系統承接核心循環（`core-definition.md` v3.0）的第三至第五個節點：

```
帶著物資安全撤離
  → 物資投入基地經營（建設／升級設施）  ← 本系統負責
    → 基地成長解鎖新功能／新探索區域   ← 本系統負責
      → 經營活動自然推進角色關係       ← 本系統觸發
```

本系統是核心樂趣 **#2 經營成就感**的主要載體——玩家探索帶回的素材，在此轉化為可見的基地成長。每一次升級都是探索成果的具體證明，同時開啟下一個目標，驅動玩家繼續探索。

---

## 一、系統定位與設計原則

### 1.1 NPC 導向架構

基地建設並非一塊獨立的「建設區域」，而是整合在每個 NPC 的互動選單中。每個 NPC **擁有自己的設施**，玩家透過與 NPC 互動來升級該 NPC 的設施。

**設計意圖**：
- 強化「每個 NPC 是一個功能入口」的清晰感
- 讓升級行為與角色關係感自然連結——你在幫這個角色升級她的工作環境
- 避免產生「基地管理」與「角色互動」在空間上的割裂感

### 1.2 設計原則

| 原則 | 說明 |
|------|------|
| **節奏自然** | 升級等待時間設計為「≈一次探索時間」，讓玩家在「升級→探索→回來升級完成」之間形成自然節奏，升級等待不造成空等感 |
| **物資共用** | 建設用物資與製作用物資共用同一個「素材（type:素材）」pool，不另設建材分類，減少物資管理複雜度 |
| **解鎖漸進** | 採混合解鎖條件（設施等級 × 村莊等級 × 好感度的組合），確保玩家先深入經營再解鎖進階內容 |
| **可見成長** | 每次升級後，設施的外觀或功能應有可見的變化，符合核心樂趣驗證標準 2（探索成果轉化為可見基地改變） |

---

## 二、NPC 互動選單結構

每個 NPC 的互動選單採統一的四項固定結構：

```
[設施功能]  [升級]  [對話]  [送禮]
```

| 選單項目 | 說明 |
|---------|------|
| **設施功能** | 進入該 NPC 擁有的設施介面（如：倉庫、農田、煉金台、製作台）。具體功能因 NPC 而異，詳見「NPC 設施對應表」 |
| **升級** | 進入該 NPC 設施的升級介面。顯示當前等級、升級所需素材、預計等待時間、升級後解鎖內容（或下一等級預覽） |
| **對話** | 與 NPC 進行對話（好感度三途徑之一：對話問答）。對話內容依好感度階段不同 |
| **送禮** | 開啟送禮介面（好感度三途徑之二/三：送食物 / 送禮物）。可送出 `type:食品` 或 `type:禮物` 的物品 |

**注意**：所有四個選單項目在任何好感度階段下均可見，但部分功能在解鎖條件未達成時顯示為鎖定狀態並提示解鎖條件。

## 三、設施系統框架

### 3.1 設施的歸屬與等級

- 每個 NPC 擁有一個或多個設施
- 每個設施有獨立的等級（從 Lv.1 起始，上限待定）
- 升級的是「NPC 擁有的設施」，而非全局性的「基地等級」

### 3.2 設施等級的效果類型

設施升級後可帶來的效果類型（具體每個設施的每級效果為 TODO，留給各 NPC 的功能定位文件定義）：

| 效果類型 | 說明 |
|---------|------|
| **功能擴充** | 解鎖設施內的新功能或新操作選項（如：農田格子數增加） |
| **效率提升** | 現有功能的參數改善（如：作物生長時間縮短、製作產出數量增加） |
| **配方解鎖** | 解鎖新的製作配方（可疊加配方卷軸機制） |
| **區域解鎖** | 在特定設施等級達到後，解鎖新的探索地圖區域 |
| **劇情觸發** | 在特定設施等級 + 好感度組合達成後，觸發角色劇情 / HCG 里程碑 |

## 四、升級機制

### 4.1 升級流程

```
玩家在 NPC 互動選單選擇「升級」
  → 系統顯示升級介面
    → 顯示：當前設施等級、升級所需素材清單、預計等待時間、升級後效果預覽
      → 玩家確認升級
        → 系統扣除所需素材（來源：背包優先，不足時再扣倉庫）
          → 倒計時開始（≈一次探索時間）
            → 玩家出發探索
              → 返回村莊後升級完成（或倒計時尚未結束，等待完成）
                → 設施等級提升，效果立即生效
                  → 村莊等級重新計算（若有設施等級提升）
```

### 4.2 等待時間設計意圖

等待時間的**設計意圖**是製造「升級→探索→回來升級完成」的自然節奏，讓玩家在等待升級的同時有明確的事情做（出去探索），回來後恰好完成，形成正向閉環。

**具體等待時長**：TODO（留給數值設計師，依目標節奏推導——設計意圖為「≈一次探索所需的現實時間」）

### 4.3 升級消耗物資

- 升級消耗的物品種類為 `type:素材`
- **具體物資種類與數量**：TODO（留給數值設計師，各設施各等級分別定義）
- 建設與製作共用同一個素材 pool（不另設建材分類）

### 4.4 升級限制

- 每個設施一次只能進行一個升級（不可同時排隊多個升級）
- 升級進行中，設施功能仍可正常使用（升級不封鎖設施）
- 升級進行中，可以取消升級：**物資是否退還**為 TODO（留製作人決定）

## 五、村莊等級

### 5.1 定義

「村莊等級」是反映整體基地發展程度的聚合指標。它由所有 NPC 設施的等級共同決定，而非由單一設施的等級代表。

### 5.2 計算方式

**TODO — 製作人決定**。以下提供三個方案選項供製作人參考：

| 方案 | 計算方式 | 特色 |
|------|---------|------|
| **A：所有設施等級加總** | 村莊等級 = 全部設施等級之和 | 鼓勵玩家均衡升級；升級任何設施都有貢獻 |
| **B：特定里程碑觸發** | 村莊等級在特定設施組合達到指定等級時提升（如：所有 NPC 設施均 ≥ Lv.2 時村莊等級升至 2） | 設計可控；容易設定精確的解鎖觸發點 |
| **C：最高 NPC 等級跟隨** | 村莊等級 = 當前最高設施等級的 NPC 數量達到一定比例後提升 | 需要最弱的 NPC 設施也要升到某個水平，避免玩家只升一個 NPC |

### 5.3 村莊等級的用途

村莊等級作為解鎖條件的一個維度（詳見「解鎖機制」章節），用於：

- 判斷玩家是否具備整體發展程度，防止跳過早期內容直達高級內容
- 作為新探索區域、新 NPC 解鎖等全局性事件的條件之一
- 可能作為部分設施升級的前置條件

## 六、解鎖機制

### 6.1 混合制框架

解鎖條件採**混合制**——可能是以下維度的不同組合，視具體解鎖內容而定：

| 條件維度 | 說明 |
|---------|------|
| **設施等級** | 特定 NPC 的設施達到指定等級 |
| **村莊等級** | 整體村莊等級達到指定值 |
| **好感度** | 特定 NPC 的好感度達到指定階段 |

**組合範例（框架示意，非最終設計）**：

```
解鎖內容 X：設施等級 ≥ Lv.3  +  好感度 ≥ 階段 2
解鎖內容 Y：村莊等級 ≥ 2  +  設施等級 ≥ Lv.2
解鎖內容 Z：設施等級 ≥ Lv.4  +  村莊等級 ≥ 3  +  好感度 ≥ 階段 3
```

並非每個解鎖都需要三個條件全部滿足，設計者可依情境選用其中一項或多項。

### 6.2 解鎖條件的設計意圖

- **設施等級**：確認玩家在該 NPC 的經營深度
- **村莊等級**：確認玩家整體發展到了相應階段，防止特化攻略
- **好感度**：確保角色相關的劇情內容與感情進展連動，不能只靠建設跳過感情線

## 七、與現有系統的接口

### 7.1 倉庫 / 背包系統

- 升級消耗的素材：**先扣背包，背包不足時再扣倉庫**（與通用製作系統一致）
- 升級完成後，功能擴充（如農田格子增加）不影響現有倉庫 / 背包的物品

### 7.2 好感度系統

- 升級設施本身**不直接增加好感度**
- 好感度仍透過三途徑累積（對話問答、送食物、送禮物）
- 設施等級達到特定值後，可解鎖與好感度相關的觸發點（如 HCG 里程碑的解鎖條件之一）

### 7.3 農田系統

- 農田格子數由農女的設施等級決定（升級後擴充格子）
- 農田的種植/收穫流程本身不變，升級影響的是**容量與參數**

### 7.4 通用製作系統

- 設施升級可解鎖新配方，加入製作系統的可用配方清單
- 製作系統的底層邏輯（配方驅動、先扣背包後扣倉庫）不受升級影響

### 7.5 探索系統

- 探索帶回的素材是升級的核心輸入物資
- 設施升級可解鎖新的探索地圖區域（升級完成後，探索入口新增可選地圖）
- 升級等待時間設計為「≈一次探索時間」，自然引導玩家在等待期間出發探索

### 7.6 NPC 解鎖順序

NPC 按固定順序解鎖：**村長夫人 → 農女 → 魔女 → 守衛**（`core-definition.md` 決策 #10）

各 NPC 的解鎖條件與基地建設進度的具體連動關係為 TODO，在各 NPC 功能定位文件中定義。
