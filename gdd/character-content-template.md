# 單角色內容量規格模板

> **版本**: v1.7

## 系統目的

本文件定義單一可攻略角色的完整內容規格，作為內容量產的標準模板與成本評估基準。此模板支撐 core-definition.md 的核心循環——角色內容作為探索→經營的里程碑獎勵，透過好感度系統自然解鎖。

本模板的規則覆蓋各角色 character-spec.md 中與系統機制衝突的部分（如五階段關係弧線）。character-spec 仍作為角色人格、說話風格、禁忌清單、立繪提示詞等角色本質設定的參考依據。

## 一、好感度系統

### 1.1 好感度與好感度等級

好感度系統分為兩層：

| 概念 | 說明 |
|------|------|
| **好感度** | 數值，透過對話問答、送食物、送禮物累積 |
| **好感度等級** | 階段，好感度達到當前等級上限時提升 |

- 共 **7 個好感度等級**
- 每次等級提升後，下一等級所需的好感度數值**遞增**
- 具體數值（各等級上限、遞增倍率）由數值設計師定義

### 1.2 好感度累積途徑

| 途徑 | 說明 |
|------|------|
| **對話問答** | 消耗角色體力，從 4 種個性選項中選擇，選中角色偏好 = 好感度多，其餘 = 少或零 |
| **送食物** | 送食品（type:食品）給角色，根據口味配對決定好感度增量，同時恢復角色體力 |
| **送禮物** | 送禮物（type:禮物）給角色，根據喜好配對決定好感度增量 |

### 1.3 好感度等級內事件節奏

每個好感度等級內，依進度百分比觸發以下事件：

```
0%                         50%                        100%
│── 10% 里程碑對話 ×5 段 ──→│── 10% 里程碑對話 ×5 段 ──→│
                            ▼                          ▼
                         一般約會                    H 劇情
                     （約定下次見面）              （等級提升）
```

| 進度 | 事件 | 說明 |
|------|------|------|
| 每 10% | 里程碑對話 | 在互動頁面自動播放的短對話，呈現關係推進的節奏感 |
| 50% | 一般約會 CG 劇情 | 約會演出 = 愛情的結果，結尾約定下次見面，為 H 劇情埋下伏筆 |
| 100% | H 劇情 | 兌現約會時的約定，好感度等級提升，進入下一等級 |

### 1.4 H 劇情觸發條件

H 劇情在好感度等級提升時**自動觸發**，不需要設施等級或基地進度作為前置條件。

**設計依據**：好感度的累積途徑（對話、送食物、送禮物）全部依賴探索帶回的物資——對話受角色體力限制（探索恢復體力），送禮的物品來自製作與料理（需要探索取得素材/種子）。探索循環已透過物資來源自然鎖定，無需額外前置門檻。

## 二、角色體力系統

### 2.1 系統定位

角色體力條的核心設計意圖是**強制中斷對話**，讓玩家無法一次刷完所有對話，必須透過探索或送食品恢復體力後才能繼續互動，維持「探索→回村互動」的循環節奏。

### 2.2 機制

| 項目 | 說明 |
|------|------|
| 體力擁有者 | 每個角色各自擁有獨立的體力條 |
| 消耗 | 選擇功能選單「對話」時消耗體力 |
| 恢復途徑 1 | 外出探索後恢復一部分 |
| 恢復途徑 2 | 贈送食品（type:食品）時恢復 |

- 體力上限、每次消耗量、恢復量等具體數值由數值設計師定義

## 三、對話系統（三層結構）

角色互動中的對話分為三個獨立層次：

| 層次 | 觸發方式 | 體力消耗 | 好感度影響 | 目的 |
|------|---------|---------|-----------|------|
| **招呼語** | 進入互動畫面自動播放 | 無 | 無 | 讓玩家認識角色 |
| **角色發問** | 功能選單「對話」 | 有 | 有（0/少/多） | 考驗玩家對角色的理解 |
| **玩家發問** | 該等級角色發問用完後自動切換 | 有 | 無 | 情報收集（口味/喜好） |

### 3.1 招呼語

- 每個好感度等級 **10 句**，隨好感度等級切換內容
- 進入角色互動畫面時自動播放（打字機效果，可按鍵跳過）
- 不消耗體力、不影響好感度
- 定位為「讓玩家認識角色」的免費內容
- **紅點亮時取代招呼語**（規則層）：L2 角色發問紅點亮起時，招呼語不播放；無紅點時才播招呼語（詳見 `character-interaction.md` §5）

### 3.2 角色發問

**觸發機制**（2026-04-18 修訂）：
- 固定時間倒數（**60 秒 placeholder**）到時，「對話」按鈕亮紅點
- **工作中暫停倒數**（角色執行委託期間停止倒數計時）
- **紅點累積上限：1**（規則層）：亮後不再累積，未被玩家觸發前永久保留
- 玩家進入角色頁面 → 「對話」按鈕顯示紅點 → 玩家點擊觸發角色發問（不自動播放）

**題目配置**（2026-04-18 修訂）：
- 每角色共 **70 題**，**每好感度等級獨立 10 題**（各等級題目不跨等級共用）
- 從**當前等級未看過的題目**中隨機抽取

**選項規格**（2026-04-18 修訂）：
- 每題 **4 種個性選項**，對應 4 種不同的個性類型
- **UI 只顯示選項文字，不顯示好感度數值**（規則層，不可由數值調整）
- 後台好感度：每個個性類型固定對應一檔 — **+0 / +2 / +5 / +10**（**placeholder**，數值設計師後續調整）
- 玩家需從角色反應與表情差分推測哪個選項對應 +10

**動態反應**（維持 v1.0 既有規格）：
- 選項選中後，角色根據結果給予動態反應：
  - 較高好感度選項：表情差分切換為較熱情的表情 + 正向音效 + 熱情回應台詞
  - 較低好感度選項：表情差分切換為較冷淡的表情 + 冷淡音效 + 冷淡回應台詞

### 3.3 玩家發問

**觸發條件**（2026-04-18 修訂）：
- **無 L2 紅點時**，玩家主動點「對話」按鈕（取消原「角色發問 10 題看完才切換」條件）

**題目配置**：
- 共用問題池 **40 題**（不分好感度等級）

**每次顯示規則**（2026-04-18 新增）：
- 剩餘題目 **≥ 4 題** → 隨機抽 4 題顯示
- 剩餘題目 **1~3 題** → 只顯示該 1~3 題（不補 [閒聊]、不補滿 4 格）
- 剩餘題目 **0 題** → 只顯示 **[閒聊]** 選項（40 題耗盡後的 fallback）

**回答規格**（2026-04-18 修訂）：
- 每題**固定 1 對 1** 回答（非閒聊題目）
- 純情報收集用途——玩家從回答中得知口味、喜好等送禮資訊
- 不影響好感度

**[閒聊] 模式**（2026-04-18 新增）：
- 40 題問題池耗盡後觸發，取代原 v1.0「3 種隨機回答」的定位
- 角色**隨機發問** + 角色**隨機回答**（不影響好感度）
- 40 題池耗盡後是否依好感度補題 → **TBD**，暫以閒聊作 fallback

### 3.3.X 角色專屬單次特殊題

部分角色擁有一批「單次特殊題」，與 40 題共用池性質不同，另行管理。

| 項目 | 說明 |
|------|------|
| **計入共用池** | **否**（不佔 40 題池配額，獨立記錄） |
| **出現方式** | 出現在玩家發問清單中，與共用池題目混列顯示 |
| **問後行為** | 問完後此題目**永久從清單消失**（單次問答，不可重複） |
| **觸發效果** | 可觸發 flag 解鎖效果（如取得物品、解鎖功能/系統），依各題設定執行 |
| **數量** | 每位角色 **0~N 題**（非強制每位都有，視角色故事節奏決定） |
| **好感度影響** | 各題獨立設定，允許 0 或定值 |

> **決策 6-13（2026-04-20）**：守衛取劍改為「首次進入 interact view 自動觸發對白」，不再透過玩家發問清單中的單次特殊題實現。「要拿劍」此題目不加入守衛的玩家發問清單（決策 6-9 撤銷），守衛不是此規格的首個實例。

**單次特殊題規格保留供未來其他角色使用**：若未來某角色設計上需要一次性特殊操作（例如解鎖某功能、取得某物件），可依此框架實作。規格詳見上方表格。

詳見 `character-unlock-system.md` § 3.5 守衛取劍現行設計（自動觸發對白）。

| 項目 | 規格 |

## 五之二、角色登場內容（2026-04-17 新增）

### 5-2.1 規格定義

自 2026-04-17 第十四~十八輪定案（新增登場 CG + 短劇情規格）後，每位角色加入村莊時必有：

| 項目 | 規格 |
|---|---|
| **登場 CG** | 1 張／角色 |
| **登場短劇情** | 1 段／角色，500~1500 字 |

登場 CG + 短劇情是**首次相遇的視覺與敘事錨點**，不計入 7 等級循環內容。

### 5-2.2 登場內容原則

- **建立敘事錨點**：為什麼她在這裡？她在村莊/外部做什麼？
- **符合角色基調**：符合各 `character-spec.md` 的個性與說話風格
- **不鋪陳深度**：深入關係留給後續累積（對話 + 送禮 + 好感度）
- **保留懸念**：可留下尚未解答的線索（為後續主線埋伏筆）
- **聚焦於「相遇時刻」**：這是玩家對角色的第一印象

### 5-2.3 四位登場 CG 的整體方向

| 角色 | 登場 CG 氛圍 | 字數備注 |
|---|---|---|
| **村長夫人** | 床邊 / 屋內 — 玩家醒來第一面（安全感） | 500~1500 字（通用範圍） |
| **農女** | 村長夫人帶玩家到田邊，農女抬頭看玩家的瞬間（樸實親切） | 500~1500 字（通用範圍） |
| **魔女** | 山邊小屋門口，魔女煉金時被打斷（神秘距離感） | 500~1500 字（通用範圍） |
| **守衛** | 森林邊界 — 警戒 → 澄清 → 握手/賠罪（戲劇性衝突）；**另有「首次進入取劍對白」**（在 interact view 內自動觸發，非 CG 場景，對白結束後自動贈劍） | **1000~1500 字**（守衛歸來 CG：需涵蓋完整歸來流程：身分誤會 + 澄清 + 收劍 + 贈劍動機 + 收尾；載體為 `character-intro-config.json > intro_guard`，由 `CharacterIntroCGView` 播放）；**另 + 取劍對白**（TBD-content-001，約 5~10 行，interact view 內彈出） |

詳見 `character-unlock-system.md` 第五章；守衛登場劇情載體規格見 `character-unlock-system.md` § 3.3。
