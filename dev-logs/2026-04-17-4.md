# Dev Log 2026-04-17-4

## 概述

Sprint 4 試玩第二輪回饋：(1) 按鈕沒有冷卻倒數顯示 (2) NPC 來訪需要 Dark Room 式 interact view。

## 需求 1：搜索按鈕倒數顯示

**原因**：按下搜索後 UI 無任何行動冷卻的視覺回饋，玩家不知道「還要等幾秒」。

**修改**：
- `MvpMainView` 新增 `_actionTime`（注入）、`_searchButtonLabel`（cache 子 TMP_Text）、`_searchButtonBaseText`（cache 原文字）
- `Initialize(...)` 簽名新增 `ActionTimeManager actionTime` 參數
- 新增 `RefreshActionCooldownLabel()`：每幀查詢 `ActionTimeManager.GetRemaining(MvpActionKeys.Search)`，冷卻中時文字改為 `搜索附近 (0.8s)`，結束時還原
- `MvpEntryPoint.Initialize` 呼叫 `_mainView.Initialize` 時傳入 `_actionTime`

只做搜索按鈕，其他按鈕（生火/延長火堆/蓋小屋）暫不處理（延長火堆若有需求再加）。

## 需求 2：NPC 來訪自動開啟 interact view

**原因**：目前 NPC 到訪只是默默出現在清單，沒有 Dark Room 式的事件氛圍。

**修改**：
- `MvpEntryPoint.Initialize` 尾端新增 `EventBus.Subscribe<MvpNpcArrivedEvent>(OnNpcArrivedAutoOpenInteraction)`
- 新增 `OnNpcArrivedAutoOpenInteraction(MvpNpcArrivedEvent e)`：呼叫既有 `OnCharacterClickedFromMain(e.CharacterId)`，自動打開 interact view
- 取消訂閱由既有 `OnDestroy` 的 `EventBus.ForceClearAll()` 處理

**流程**：
1. `NpcArrivalManager` 發 `MvpNpcArrivedEvent`
2. 多個訂閱者並行處理：
   - `NPCInitiativeManager` 註冊該 NPC 主動發話計時
   - `MvpMainView` 嘗試 spawn character list item（可能被稍後 Hide 中斷訂閱）
   - `MvpEntryPoint` 呼叫 `OnCharacterClickedFromMain` → Hide mainView + Show interact view
3. 玩家在 interact view 對話或按返回
4. 返回時 mainView.Show → OnShow → RefreshAll → `RefreshCharacterList` 從 `_npcArrivalManager.ArrivedCharacters` 補齊所有缺失的 NPC item

此流程下無需改事件結構，既有「返回時補 spawn」機制已保護 race condition。
