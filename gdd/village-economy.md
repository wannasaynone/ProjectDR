# 村莊經濟系統

> **版本**: v1.1

## 系統目的

本文件定義 ProjectDR 的村莊經濟架構，涵蓋物品分類、農田系統、通用製作系統、贈禮效果系統，以及各角色的功能分工。

此系統是核心循環中「物資投入村莊建設」環節的完整實作依據，直接連結探索（物資取得）與感情培養（贈禮、解鎖互動）兩條次循環。玩家探索帶回的種子與素材，透過農田與製作系統轉化為更高階的產品，這些產品再作為送禮與自用 BUFF 的手段，驅動「帶回家」與「羈絆」兩項核心體驗關鍵字。

---

## 一、世界觀補充說明

村莊本身有農女長期維護的農業經濟圈，村民依靠農女的農地基本上可以自給自足。玩家的農田是**額外的**：故事上是玩家主動提出要幫忙擴充糧食來源，農女才劃分出一塊空地給玩家使用。

這套設定解釋了以下兩件事：
1. 為何玩家農田需要從探索取回的種子——農女的農地已有既有品種，玩家負責帶回森林中的新品種
2. 為何村莊不會因玩家不耕種而崩潰——NPC 自給自足的農業圈始終在後台運作，玩家的農田是加成而非生存必要

## 二、物品分類定義

所有物品在資料表中統一為「物品（Item）」，以 `type` 欄位區分種類。

| type 值 | 中文名稱 | 說明 |
|---------|---------|------|
| `種子` | 種子 | 從探索取得，投入農田後倒計時產出作物 |
| `食材` | 食材 | 作物收穫產出、或從探索直接取得的稀有食材；是料理製作的主要輸入材料 |
| `食品` | 食品 | 料理製作產出的成品；用於送禮或自用（獲得 BUFF） |
| `藥劑` | 藥劑 | 藥水製作產出的成品；探索用消耗品 |
| `素材` | 素材 | 探索取得的非食用性物品，包含礦石、木材等；用於製作系統（藥水、裝備等）的輸入材料 |
| `其他` | 其他 | 不屬於上述分類的特殊物品，例如配方卷軸、禮品成品等 |

**擴充規則**：新增 type 種類前，必須先查閱資料表確認不與既有 type 衝突，避免分類混亂。

## 三、農田系統規格

### 3.1 系統定位

農田的核心功能是**種子提煉器**：將探索帶回的種子轉化為可用食材，是「探索 → 製作 → 送禮」鏈中的必要中間環節。

### 3.2 運作流程

```
玩家將種子（type:種子）投入農田格
  → 倒計時開始（使用現實時間）
    → 倒計時結束，收穫作物（type:食材）
      → 作物自動留存，不會腐爛
        → 玩家手動收穫後進入倉庫
```

### 3.3 規格細節

| 項目 | 規格 | 備註 |
|------|------|------|
| 計時方式 | 現實時間 | 即使遊戲未開啟也持續計時 |
| 腐爛機制 | 無 | 收穫物不會因等待過久而消失 |
| 農田格數 | IT 階段 3 格 | 後續可透過設備升級擴充 |
| 倒計時時長 | 依種子種類不同，定義於資料表 | 各種子具體時長見物品資料表 |
| 產出作物種類 | 依種子種類對應，定義於資料表 | 具體對應關係見物品資料表 |

### 3.4 IT 階段實作範圍

- 農田 UI 呈現（倒計時顯示、收穫按鈕）
- 種子投入邏輯
- 倒計時計算（現實時間）
- 作物產出並進入倉庫

**非 IT 階段（後續擴充）**：
- 製作設備升級，例如「鐵鋤」（耕種時間 -10% 等）

## 四、通用製作系統規格

### 4.1 系統定位

所有製作行為由同一套底層系統支援，差異僅在於**不同角色開放不同配方分類**。這確保未來增加新角色或新配方時，底層系統不需重寫。

### 4.2 製作流程

```
玩家在角色互動畫面選擇「製作」功能
  → 顯示該角色可用的配方清單（已解鎖配方）
    → 玩家選擇配方
      → 系統顯示所需材料（合併顯示背包 + 倉庫庫存）
        → 材料足夠 → 確認製作 → 扣除材料 → 產出成品
        → 材料不足 → 提示缺少哪些材料（待定：是否顯示取得途徑）
```

### 4.3 材料來源讀取規則

- 製作時**背包與倉庫合併顯示**，兩者的庫存皆計入可用材料
- 扣除材料的優先順序：**先扣背包，背包不足時再扣倉庫**

### 4.4 配方解鎖機制

| 項目 | 規格 |
|------|------|
| 解鎖方式 | 收集配方卷軸（type:其他），從探索中取得 |
| 解鎖操作 | 玩家使用（或自動讀取）配方卷軸後，對應配方加入可用清單 |
| 初始配方 | 各角色有基礎配方（不需卷軸，開局即可用），具體內容定義於配方資料表 |
| 配方資料格式 | 配方表驅動（輸入材料清單 + 產出成品 + 歸屬角色分類） |

### 4.5 角色功能對應表

| 角色 | IT 階段開放的配方分類 | 後續擴充 |
|------|---------------------|---------|
| **村長夫人** | 料理製作（食材 → 食品） | — |
| **農女** | — （農田入口，不參與配方製作） | 製作設備升級（如鐵鋤） |
| **魔女** | 藥水製作（素材/食材 → 藥水） | — |
| **守衛** | — （IT 階段僅提供探索入口） | 裝備製作 + 角色裝備分派 |

> 農女的製作設備升級在後續階段新增，IT 階段農女的主要互動功能為農田管理。

### 4.6 IT 階段實作範圍

- 配方表資料結構
- 製作 UI（配方清單、材料顯示、製作確認）
- 背包 + 倉庫合併庫存讀取
- 配方卷軸解鎖邏輯
- 村長夫人料理製作配方（至少一組測試用）
- 魔女藥水製作配方（至少一組測試用）

**非 IT 階段（後續擴充）**：
- 守衛裝備製作配方
- 角色裝備分派系統
- 農女製作設備升級

## 五、贈禮效果系統規格

### 5.1 系統定位

贈禮系統是感情培養循環的主要驅動機制，將製作系統的產出（食品、禮品）連結到角色的好感度提升與劇情解鎖。

**重要原則**：好感度提升不能只是數字 +X。每個角色有個人的口味喜好設定，對不同禮物必須有差異化反應（不同對話與表情），讓玩家感受到「這個禮物對這個角色是特別的」。

### 5.2 贈禮流程

```
玩家在角色互動畫面選擇「送禮」功能
  → 顯示背包 + 倉庫中可送的物品清單
    → 玩家選擇物品
      → 角色根據口味喜好觸發對應反應（對話 + 表情）
        → 好感度依物品與角色喜好等級提升
          → 達到閾值時解鎖新對話 / 劇情事件
```

### 5.3 角色口味喜好框架

每個角色在角色設定文件中定義以下喜好資訊：

| 喜好等級 | 說明 | 好感度變化 |
|---------|------|----------|
| 最愛 | 角色特別偏好的物品，有專屬反應對話 | 待定（最高加成） |
| 喜歡 | 普通喜好的物品，有正面反應對話 | 待定（標準加成） |
| 普通 | 無特別偏好的物品，有平淡回應 | 待定（基礎加成） |
| 不喜歡 | 角色不喜歡的物品，有負面反應對話 | 待定（可能是低加成或 0） |

> 具體好感度數值（各等級的加成量、解鎖閾值）待製作人確認。
> 各角色的具體喜好物品清單將另外撰寫獨立設計文件定義。

### 5.4 自用 BUFF

食品（type:食品）除了送禮外，玩家可以自用以獲得探索用 BUFF，具體 BUFF 效果待製作人確認後定義於物品資料表中。

### 5.5 IT 階段實作範圍

- 送禮 UI（可送物品選擇）
- 基礎好感度增減邏輯
- 角色口味喜好資料表（至少定義「最愛」與「普通」兩級）
- 對應口味的差異化反應對話（每個 IT 階段角色至少各一組）
- 好感度閾值解鎖新對話觸發邏輯

**非 IT 階段（後續擴充）**：
- 完整的表情系統差異化呈現
- 更細緻的好感度階段事件
- 角色口味喜好對話的完整撰寫

## 六、完整物品流向圖

```
探索
  │
  ├─ 種子 (type:種子)
  │     └─→ 農田（倒計時）
  │               └─→ 作物 (type:食材)
  │
  ├─ 稀有食材 (type:食材)  ←──────────────────────────┐
  │                                                    │
  ├─ 素材 (type:素材)                                  │
  │   └─→ 藥水製作（魔女）→ 藥水 (type:藥劑)             │
  │   └─→ 禮物製作 → 禮品 (type:其他) → 送禮 (好感↑)  │
  │   └─→【後續】裝備製作（守衛）→ 裝備 → 探索戰鬥用   │
  │                                                    │
  ├─ 配方卷軸 (type:其他)                               │
  │     └─→ 解鎖新配方（加入製作清單）                  │
  │                                                    │
  作物 (type:食材) ─────────────────────────────────────┘
        └─→ 料理製作（村長夫人）
                  └─→ 食品 (type:食品)
                            ├─→ 送禮 → 好感↑/劇情解鎖
                            └─→ 自用 → 探索 BUFF
```

## 七、IT 階段整體實作範圍總覽

以下彙整 IT 階段必須實作的功能，以及明確排除在 IT 階段之外的後續擴充。

### IT 階段必須實作

| 分類 | 功能 |
|------|------|
| **農田** | 種子投入、現實時間倒計時、收穫作物入倉庫 |
| **製作** | 配方表資料結構、配方卷軸解鎖、背包+倉庫合併讀取、製作 UI |
| **製作：村長夫人** | 料理製作（至少一組測試配方） |
| **製作：魔女** | 藥水製作（至少一組測試配方） |
| **贈禮** | 送禮 UI、基礎好感度邏輯、差異化口味反應對話 |
| **物品系統** | type 欄位區分、統一物品資料格式 |

### 後續擴充（非 IT 階段）

| 分類 | 功能 |
|------|------|
| **農田** | 製作設備升級（鐵鋤等） |
| **製作：守衛** | 裝備製作配方 |
| **角色** | 角色裝備分派系統 |
| **贈禮** | 完整好感度事件與表情系統 |
