# 核心定義文件

> **版本**: v1.0

## 系統目的

本文件定義 ProjectDR 的遊戲本質，包含核心循環、樂趣來源、驗證標準、目標受眾等基礎設計方向。所有後續設計決策應以本文件作為最高依據，確保每一個系統與內容設計都在強化核心樂趣，而非稀釋它。

---

## 一、遊戲概念一句話描述

在魔物棲息的奇幻森林深處，一座與世隔絕的村莊中醒來的男人，帶領夥伴們擴建停滯已久的家園、與她們培養深厚情感，同時冒險探索不斷變化的危險森林，帶回村莊發展所需的關鍵物資。

> **世界觀**：奇幻冒險。被不斷變化的森林圍困、發展停滯的村莊，玩家作為唯一能安全往返森林的男性，擴建村莊並與多種族女性角色（人類、精靈、獸人等）培養感情。

---

## 二、核心循環 (Core Loop)

### 主循環（宏觀層）

```
村莊資源不足
  → 派遣（玩家操作）角色出發探索
    → 在地圖上搜集物資、擊退魔物
      → 帶著物資安全撤離
        → 物資投入村莊建設
          → 村莊成長解鎖新設施／新對話選項
            → 與特定角色的親密度提升
              → 解鎖更深層的故事與角色內容
                → 村莊更進一步擴張需要更多資源
                  → 回到「村莊資源不足」
```

### 次循環 A：探索循環（單次出擊）

```
進入地圖
  → 移動探索格子（踩地雷式逐格揭開）
    → 發現探索點 → 採集物品（等待解鎖計時）
    → 遭遇魔物 → 射擊戰鬥（Top-down 射擊）
      → 擊敗或迴避
  → 背包趨近飽和 或 風險升高
    → 判斷：繼續深探 vs 撤離
      → 返回出發點 或 進入撤離點等待 6 秒
        → 物資自動入倉
```

### 次循環 B：感情培養循環

```
與村莊角色互動（對話、送禮、完成任務）
  → 親密度提升
    → 解鎖新的對話選項與劇情事件
      → 加深對角色的情感投入
        → 產生「想讓這個人過得更好」的動機
          → 更積極投入村莊建設與探索
```

**兩條次循環的交匯點**：探索帶回的物資 → 村莊建設 → 解鎖角色互動內容。感情培養與探索的成果直接互相驅動。

## 三、核心樂趣 (Core Fun)

### 主要樂趣

| # | 樂趣類型 | 描述 |
|---|---|---|
| 1 | **經營成就感** | 看著停滯已久的村莊在自己手中一點一滴擴建起來，每一棟新建築、每一項新設施都是玩家努力的具體證明。資源積累與村莊擴張帶來的「可見成長感」是本作最核心的情緒回報。 |
| 2 | **戀愛養成的情感連結** | 隨著時間累積與玩家的付出，角色從陌生人逐漸成為真正的夥伴乃至戀人。解鎖角色私密故事、看見角色因玩家而改變，是本作第二核心的情緒回報。 |

### 附屬樂趣

| # | 樂趣類型 | 描述 |
|---|---|---|
| 3 | **探索的緊張刺激感** | 每一步都可能是未知，未翻開的格子藏著資源也藏著危機。背包漸滿時的「再多探一格還是撤離」抉擇製造緊張感，安全撤離帶來小型成就感。 |

> **設計方向備忘**：探索系統的定位是服務於經營與戀愛的**資源取得手段**——探索本身不是核心樂趣來源，但它是一個有完整機制的子系統（WASD 移動、逐格探索、射擊、6 秒撤離、採集計時等，均為製作人既定設計規則）。設計新內容時，應確認該內容能強化「帶回物資→推動主循環」的流暢度，而非讓探索自成一個目的。探索的緊張感是調味，不是主菜。

## 四、核心樂趣驗證標準

以下標準用於判斷「這個設計是否強化了核心樂趣？」。所有新系統、新功能、新內容在提案時，應能通過至少一項標準。

### 標準 1：是否讓村莊的成長更加「可見且有意義」？

> 玩家能否清楚地看到自己的付出轉化為村莊的改變？這個改變是否讓玩家感到驕傲？
>
> - **通過**：新建一棟建築後，村莊視覺上有明顯變化，且這棟建築帶來實際的新功能或新可能性
> - **不通過**：資源消耗後只有數字變化，無視覺或功能上的具體呈現

### 標準 2：是否深化玩家對角色的情感投入？

> 這個設計是否讓玩家更了解角色、更在乎角色的感受？
>
> - **通過**：觸發一段讓玩家看見角色脆弱面或獨特個性的對話；角色對玩家行為有感受得到的情感反應
> - **不通過**：互動只是重複的數字交換（送禮 +X 好感），缺乏角色個性的體現

### 標準 3：探索的風險是否在「可承受範圍內」且直接服務於村莊成長？

> 探索帶來的壓力與回報，是否在鼓勵玩家繼續投入主循環，而非讓人沮喪退出？
>
> - **通過**：玩家死亡或探索失敗的代價是有限且可恢復的；取得的物資有明確的村莊用途
> - **不通過**：探索懲罰過重導致玩家不願出發；物資堆積但無法在村莊中使用

### 標準 4：這個設計是否創造了讓玩家想看下一步的驅動力？

> 玩家在這個設計中，是否有一個「下一個目標」在前方召喚他繼續玩下去？
>
> - **通過**：玩家完成當前動作後，自然浮現「接下來想做 X」或「想看看 X 之後會怎樣」的念頭；存在一個清晰可見但尚未達到的下一個解鎖/進展
> - **不通過**：一個階段結束後，玩家感到「已經完成了、可以關掉了」，缺少向前拉動的鉤子

## 五、目標受眾與玩家畫像

### 基本輪廓

| 項目 | 描述 |
|---|---|
| **年齡層** | 主要 18–35 歲；核心受眾 20–30 歲 |
| **性別傾向** | **男性向** |
| **內容分級** | **成人向 (18+)**，上架 Steam 含成人內容 |
| **遊戲經驗程度** | 輕中度玩家為主；能接受一定深度，但不追求硬核操作門檻 |
| **平台習慣** | 習慣 Steam PC 端；可能有大量 Visual Novel、獨立遊戲遊玩經驗 |

### 核心動機

這群玩家在遊戲中追求的是：

- **「我的努力有意義」**：看到自己親手建設的東西成長，獲得「這是我做出來的」的歸屬感
- **「被角色真正看見」**：在故事中體驗被特定角色深度接受與喜愛的情感滿足
- **「安全的冒險刺激」**：在可控的風險範圍內體驗緊張感，但不希望因失誤而痛失所有努力
- **「下一個解鎖在哪」**：增量遊戲式的「再看一下接下來有什麼」的持續驅動力

### 浪漫／情感場景期待（提案方向，待製作人確認）

> 以下為設計團隊根據參考作品分析所提出的角色互動設計方向建議，需製作人確認是否採用。

- 角色有**立體的個性與過去**，不是單純的服務型角色
- **漸進式的信任建立**：角色從防備到敞開心扉的過程有清晰的情感弧線
- **因為玩家的選擇而不同的關係走向**：感覺自己的行為真的影響了角色
- **私密感**：特定劇情是「只有親密度夠才能看到的」，製造專屬感
- **成人向的肢體與情感親密描寫**，作為親密度達到高階段後的情感回報

### 參考作品借鑑要素

> **說明**：以下三款參考作品的目標受眾並不重疊，本作並非以任何一款的受眾為全部目標。下表標注的是**本作從各作品借鑑的機制與設計理念**，而非借鑑其受眾。

| 作品 | 玩家群特徵（僅供背景理解） | **本作借鑑要素** |
|---|---|---|
| **A Dark Room** | 喜愛極簡文字冒險與增量成長、對隱藏敘事充滿好奇、容忍等待過程的玩家 | 資源驅動的村莊建設循環；增量成長帶來的「再做一次」驅動力；以極少的呈現承載強烈的成就感 |
| **HuniePop** | 主動追求戀愛養成內容、對角色立繪與個性投入、接受成人內容的成年玩家 | 角色差異化的好感培養系統；角色個性與解鎖內容的情感投入設計；親密度解鎖的漸進式內容結構 |
| **Escape from Tarkov** | 享受高風險高回報的搜打撤體驗、對裝備管理與撤離決策有樂趣 | 撤離機制帶來的緊張感與決策張力（再深探一格 vs 安全撤離）；物資不能全帶走的取捨壓力 |

## 六、遊戲體驗關鍵字

| # | 關鍵字 | 說明 |
|---|---|---|
| 1 | **重建 (Rebuilding)** | 讓停滯的村莊一點一滴恢復生機與擴張的成就感 |
| 2 | **羈絆 (Bond)** | 與角色之間隨時間深化的真實情感連結 |
| 3 | **帶回家 (Bring It Home)** | 每次探索的核心動作：冒險出去，安全回來，把資源帶給重要的人 |
| 4 | **已知與未知之間 (Known vs Unknown)** | 踩地雷式的揭示張力，每一步都是選擇 |
| 5 | **恰到好處的緊張 (Comfortable Tension)** | 有壓力但不絕望；危險存在但後果可承受 |
