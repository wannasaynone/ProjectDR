# 角色解鎖系統（Character Unlock System）

> **版本**: v1.8

## 系統目的

本系統承接 `core-definition.md` 決策 #10「NPC 固定順序解鎖」與核心樂趣驗證標準 4「是否創造讓玩家想看下一步的驅動力？」，並落實 A Dark Room **漸進揭露哲學**於開局階段的具體運作方式。

本系統的定位明確為**開局漸進揭露結構**（製作人 2026-04-17 第十二~十八輪定案）：

- **解鎖 ≠ 遇見**：角色（前 3 位）本來就在村莊內，「解鎖」= Hub 按鈕從隱藏變可見
- **解鎖 = 功能 + 初始資源綁定**：每次解鎖都帶即時可用的玩法內容，玩家永遠不會「解鎖了但不知道怎麼開始」
- **解鎖節奏 = 主線任務驅動**：節點的觸發條件為主線任務完成（QC-D），節奏可控

本系統**不處理**：
- 解鎖後的日常互動邏輯（見 `character-interaction.md`）
- 委託本身的流程（見 `commission-system.md`）
- 好感度累積（見 `character-content-template.md`）

## 一、開局結構總覽

### 1.1 三個節點與七個階段

製作人設計的節點結構為「村長夫人 → 選擇 1 → 村長夫人 → 選擇 2 → 村長夫人 → 最後一位（守衛）」。其中**三次村長夫人**即三個**節點**。

| 節點 | 位置 | 劇情功能 |
|---|---|---|
| **節點 0** | 開局（玩家醒來） | 介紹世界觀 + 觸發選擇 1 |
| **節點 1** | 選擇 1 後累積條件達成，再訪村長夫人 | 對新角色加入的反應 + 推進主線 + 觸發選擇 2 |
| **節點 2** | 選擇 2 後累積條件達成，再訪村長夫人 | 推進至探索功能開放 + 鋪墊守衛 |

節點之間的時段，玩家做**日常循環**（與已解鎖角色互動、委託、累積好感度）。

### 1.2 前期七階段流程

```
【階段 1 — 節點 0】開場強制流程（包含選擇 1）
  [1-1] 村長夫人登場 CG（= 開場劇情，首張畫面）
  [1-2] 世界觀介紹短劇情（打字機，介紹世界觀與村莊狀態）
  [1-3] 自動進入村長夫人 interact view（此時無 Hub 主動權，返回按鈕隱藏）
  [1-4] 對話推進至選項（VN 式）：
        「今天你想先認識誰？」
        A. 田那邊的女孩（→ 農女）
        B. 山邊小屋的那位（→ 魔女）
  [1-5] 玩家選擇 → 該角色 Hub 按鈕在背景解鎖 + 初始物資發放
        （無額外劇情演出，村長夫人可能有一句回應台詞）
  [1-6] 開啟返回按鈕（玩家首次獲得 Hub 主動權）
  [1-7] 玩家按下返回 → 村莊 HUD（Hub）
  [1-8] Hub 顯示 2 個角色按鈕：[村長夫人] + [剛選的那位]
        （探索入口尚未出現；守衛按鈕尚未出現）

【延續 — 首次點擊新角色按鈕】（玩家自主時機）
  玩家自己點擊剛解鎖的角色按鈕（農女 或 魔女）
    → 進入該角色 interact view
      → 首次進入：播放登場 CG + 短劇情（500~1500 字）
        （角色自我介紹、敘事錨點）
      → 劇情結束後進入正常互動模式
        （可看到功能選單、可使用委託等）
  之後再進入該角色時不再播放登場 CG（只播日常對話）

【階段 2】日常循環（選擇 1 與節點 1 之間）
  玩家自由使用 2 位已解鎖角色的功能：
    - 村長夫人 [倉庫][送禮][CG 圖鑑][對話]
    - 選擇 1 角色 [委託功能][送禮][CG 圖鑑][對話]
  累積條件：選擇 1 角色登場 CG + 短劇情完成（自由日常循環，委託為自主發現，無強制要求）

【階段 3 — 節點 1】
  選擇 1 角色登場 CG + 短劇情完成 → 村長夫人 L4 紅點亮起
  → 玩家進入村長夫人 interact view
  → 對話推進：對選擇 1 角色加入的反應 + 劇情推進 + 引見剩下那位
  → 選項（VN 式）：「你準備好見 XX 了嗎？」或類似選擇
  → 劇情演出：村長夫人帶玩家去見剩下那位
  → 該角色登場 CG + 短劇情
  → 返回 Hub，新增該角色按鈕

【階段 4】日常循環（選擇 2 後）
  玩家自由使用 3 位已解鎖角色的功能
  累積條件：選擇 2 角色（魔女）登場 CG + 短劇情完成（自由日常循環，委託為自主發現，無強制要求）

【階段 5 — 節點 2】
  魔女登場 CG + 對話結束瞬間 → 村長夫人 L4 紅點立刻亮起
  → 玩家進入村長夫人 interact view
  → 對話推進：推進至探索功能開放 + 鋪墊守衛
     劇情台詞方向：「我們等守衛很久了……你準備好出去看看外面了嗎？」

【階段 6】探索功能開放
  節點 2 結束後（前 3 位全部解鎖 + 新 T1 完成）→ Hub 新增「探索」入口
  （視覺：按鈕從隱藏變可見，TBD 可配合亮起效果）

【階段 7】首次探索 → 守衛歸來事件
  玩家首次點擊探索 → 進入森林 → 守衛歸來劇情自動播放（身分誤會，純劇情演出）
  → CharacterIntroCGView 播放 intro_guard CG 對話 → 守衛收劍 → 守衛加入（Hub 按鈕解鎖）
  → 返回村莊 HUD：守衛按鈕可見（FirstMeet 紅點）、【探索按鈕暫時關閉】
  玩家點探索按鈕 → 提示「要去找守衛對話拿劍...」→ 無法進入探索（每次點擊皆顯示）
  → 玩家進入守衛 interact view
  → 【首次進入自動觸發對白】（取劍對白，在 interact view 內彈出，無全螢幕 CG）
  → 對白結束 → 劍自動進入背包 + 探索周圍委託解鎖
  → 返回村莊 HUD：探索入口重新開啟
  下次進入守衛：走正常流程（招呼語 + 功能選單）

【後續】完整循環開放
  四位角色 + 自由探索 + 所有委託類型可用
  進入長期循環（委託 / 擴建 / 探索 / 感情）
```

### 1.2.1 節點 0 的特殊 UX 狀態

節點 0 是**強制線性流程**，玩家在流程完成前**沒有 Hub 主動權**：

| 步驟 | 玩家可做的事 |
|---|---|
| 開場劇情 | 僅觀看（可跳過？TBD） |
| 進入 interact view | 自動發生，無法取消 |
| 對話播放 | 打字機推進 / 點擊跳過 |
| 選項呈現 | **必須選擇一個**（無「不選」選項） |
| 選擇後劇情 | 自動播放 |
| 開啟返回按鈕 | **此時才首次出現** |
| 返回 Hub | 玩家首次看到 Hub 主畫面 |

**關鍵設計**：玩家在節點 0 完成前無法逛 Hub、無法進入其他功能、無法做其他事。節點 0 結束後，玩家才進入正常遊戲循環。

**與 `character-interaction.md` 的整合**：
- 節點 0 的 interact view 是**特殊狀態**（返回按鈕隱藏、無 Hub 可返回）
- 既有 `CharacterInteractionView` 需支援「強制模式」（無返回按鈕）vs「正常模式」（有返回按鈕）
- 選項呈現使用 VN 式分支，需 `DialogueManager` 擴展

### 1.3 與四條次循環的關係

對照 `core-definition.md` v3.3 的次循環定義：

| 階段 | 可運作的循環 |
|---|---|
| 階段 1（節點 0，強制流程含選擇 1） | 階段內無自由循環；階段結束後 B（委託，對選擇 1 角色，自主發現）+ D 立即可用 |
| 階段 2（自由日常循環，選擇 1 角色 CG 完成前） | B（委託對選擇 1 角色，自主發現）+ D（對 2 位已解鎖角色） |
| 階段 3（節點 1 + 選擇 2） | B + D 擴展至 3 位已解鎖角色 |
| 階段 4（自由日常循環，魔女 CG 完成前） | B + D（3 位）+ C（擴建，若倉庫觸發） |
| 階段 5（節點 2，魔女 CG 完成後立刻觸發） | B + C + D（3 位）|
| 階段 6~7（探索開放 + 守衛解鎖） | A + B + C + D 四循環完整啟動 |

## 二、VN 式選項解鎖機制（前 3 位）

### 2.1 機制定位

前 3 位角色（村長夫人、農女、魔女）**本來就在村莊內**——這是世界觀一致性的要求（村莊被森林圍困的封閉社會）。

「解鎖」**不是遇見**，而是：

- **UI 層**：Hub 上的角色按鈕從隱藏變可見
- **劇情層**：村長夫人帶玩家去見該角色（發生在村長夫人對話中的 VN 式選項）
- **玩法層**：伴隨解鎖的初始資源與功能開放

### 2.2 選擇 1 的對話流程（節點 0 內部，強制流程）

**重要**：選擇 1 發生在節點 0 內部，不是獨立階段。此時玩家**尚無 Hub 主動權**（返回按鈕尚未出現）。**選擇後不播放劇情演出**，只是在背景解鎖按鈕 + 發放物資，玩家返回 Hub 後需要**自己點擊新按鈕**才會看到該角色。

```
【節點 0 內部】
村長夫人登場 CG（= 開場畫面）
  ↓
世界觀介紹短劇情（打字機推進）
  ↓
自動進入村長夫人 interact view（強制模式，無返回按鈕）
  ↓
對話推進至選項時機
  ↓
村長夫人（對話台詞方向）：
  「我們這裡還有幾個人，今天你想先認識誰？」
  ↓
玩家選項（VN 式，兩個選項，必須選擇其一）：
  A. 田那邊的女孩（→ 農女）
  B. 山邊小屋的那位（→ 魔女）
  ↓
玩家選擇後：
  → 村長夫人一句回應台詞（選擇反饋，TBD）
  → 背景解鎖該角色 Hub 按鈕
  → 初始物資發放至背包/倉庫（先背包後倉庫邏輯）
  ↓
開啟返回按鈕（玩家首次獲得 Hub 主動權）
  ↓
玩家按下返回 → 村莊 HUD：
  - Hub 顯示 [村長夫人] + [剛選擇的角色] 兩個按鈕
  - 該角色的委託功能可立即使用（但玩家尚未見過她）

【節點 0 結束，進入延續時段】
玩家自由選擇：
  - 點擊新角色按鈕 → 首次進入觸發登場 CG + 短劇情（500~1500 字）
  - 或先做其他事（看倉庫、送禮給村長夫人等）
該角色的登場 CG 和短劇情在**玩家首次點擊該按鈕時**播放，
不是節點 0 強制播放的一部分。
```

### 2.2.1 「登場 CG + 短劇情」的播放時機

**非強制，玩家首次點擊該角色 Hub 按鈕時觸發**：

- 玩家選擇 1 結束後返回 Hub
- 玩家可能立刻點擊新角色按鈕 → 看到登場 CG + 短劇情
- 也可能先逛一下倉庫、送禮給村長夫人後才點擊 → 這時才播放
- **該角色的登場 CG + 短劇情是該角色 interact view「首次進入專屬流程」**

優點：
- 玩家保有主動性（不是被拖著走）
- 登場時刻感更強（是玩家主動「去認識」她）
- interact view 的強制模式只用於節點 0 的選項階段

### 2.3 選擇 2 的對話流程

與選擇 1 結構相同，但：

- 村長夫人開場話題是「引見剩下那位」
- 選項簡化為單選（「見」/「再等等」— OB 延後機制 TBD，見待決事項）
- 登場 CG + 短劇情 + 解鎖 + 初始物資同階段 2

### 2.4 延後機制（TBD）

選項 1 未選的那位是否進入延後機制？
- **工作室建議**：前期結構已設計為「選 1 → 節點 1 → 選 2」，選擇 1 沒選的那位自然是選擇 2 的必選目標（不提供第二次「不見」選項），實作最簡
- **可後續擴展**：若加入更多選項，則採 OB-3 延後機制

## 三、守衛特殊流程（身分誤會 + 純劇情演出）

### 3.1 設計定位

守衛是**唯一外部事件解鎖**的角色——她的職責就是「外部執勤」，她的加入來自一次性的「歸來」事件，與世界觀自然契合。

製作人 2026-04-17 第十八輪決定守衛劇情包裝為**身分誤會 + 純劇情演出**（OA-B + QA-劇情演出）：

- **OA-B 身分誤會**：守衛不是精神污染，而是單純認為玩家是入侵者
- **QA 純劇情演出**：玩家不參與戰鬥（首次探索時尚無裝備），全程為劇情自動播放

### 3.2 觸發條件

- **前置條件**：階段 6 探索功能已開放（前 3 位全部解鎖 → PA-A）
- **觸發時機**：玩家**首次**點擊 Hub 的探索入口進入森林
- **觸發位置**：進入森林後的劇情事件自動播放（玩家不需移動至特定地點）

### 3.3 劇情流程（純劇情演出）

> **Sprint 6 擴張（2026-04-20）**：守衛歸來事件結尾**不贈劍**。守衛收劍 + 守衛解鎖後，探索暫時關閉，玩家需主動向守衛發問「要拿劍」才能取得劍並重開探索。

> **Sprint 6 F7 決策 6-12（2026-04-20）**：守衛歸來完整劇情（身分誤會 + 澄清 + 收劍 + 贈劍動機 + 收尾）由 **`CharacterIntroCGView` 播放 `character-intro-config.json > intro_guard` CG 對話承載**。此段劇情不經由獨立 DialogueManager 路徑播放。

```
玩家首次探索 → 進入森林
  ↓
CharacterIntroCGView 自動啟動，播放 intro_guard CG 對話：
  [1] 守衛從森林深處走出，警戒狀態
  [2] 守衛：「你是誰！入侵者！」（準備拔劍）
  [3] 村長夫人登場（或其他已解鎖角色）出面澄清
  [4] 解釋：「他是我們的新朋友」
  [5] 守衛收劍 + 道歉（涵蓋贈劍動機鋪墊）
  [6] 登場 CG（警戒 → 澄清 → 賠罪場景）
  [7] 村長夫人離場 + 收尾台詞
  ↓
intro_guard CG 對話播放結束
  [8] 守衛加入村莊（Hub 守衛按鈕解鎖）
  [9] 返回村莊 HUD：探索入口暫時關閉
```

**劇情載體說明**：
- `intro_guard` 需涵蓋完整守衛歸來流程（身分誤會 + 澄清 + 收劍 + 贈劍動機 + 收尾），建議字數 1000~1500 字
- 實作由 `CharacterIntroCGView` 統一處理，與農女/魔女登場 CG 使用相同的播放機制
- `guard-return-config.json` 原有的 31 條對話路徑已由 `intro_guard` 取代，該 JSON 檔保留為劇本參考

**注意**：原流程中的「贈劍」步驟已移除。玩家此時**未持有劍**，探索入口亦暫時無法使用。劍的取得路徑見 § 3.5。

### 3.4 設計優勢（相較於精神污染版本）

### 3.4 設計優勢（相較於精神污染版本）

| 面向 | 精神污染版 | 身分誤會版（採用） |
|---|---|---|
| 實作複雜度 | 需戰鬥/QTE 系統 | 純劇情演出，簡單 |
| 玩家首次探索無武器的合理性 | ❌（打不過） | ✅（不用戰鬥） |
| 角色人設契合 | 需要額外解釋 | ✅ 守衛警戒心人設自然契合 |
| 劇情字數 | 中（含戰鬥與恢復） | 短（對話即可） |
| 實作工作量 | 高 | 低 |

### 3.5 贈劍後續流程（首次進入自動對白觸發）

> **決策 6-13（2026-04-20）**：原「玩家主動向守衛發問取得」改為「首次進入守衛 interact view 自動觸發對白 → 對白結束自動贈劍」。撤銷決策 6-9（「要拿劍」不加入玩家發問清單）與決策 6-10（移除自主發問教學關卡設計）。

**前置狀態**：守衛歸來事件（§ 3.3）完成，守衛 Hub 按鈕已解鎖（FirstMeet 紅點），探索入口暫時關閉。

```
玩家點探索入口（暫時關閉）
  ↓
系統顯示提示：「要去找守衛對話拿劍...」（modal 或 dialog 形式）
  → 每次點擊皆顯示完整提示訊息，不使用一次性或 toast 機制
  → 探索入口仍不可進入

玩家進入守衛 interact view
  → 【首次進入自動觸發取劍對白】
     - 在 interact view 內彈出（類似守衛發問的回應形式）
     - 無全螢幕 CG
     - 對白含贈劍敘事（具體文本見 TBD-content-001）

對白結束
  → 劍自動進入背包（基本戰鬥裝備）
  → 探索周圍委託解鎖

返回村莊 HUD
  → 探索入口重新開啟
  → 進入完整四循環

下次進入守衛
  → 走正常流程（招呼語 + 功能選單 [探索周圍][送禮][CG 圖鑑][對話]）
```

**設計說明**：

## 五、登場 CG + 短劇情規格

### 5.1 規格定義

每位角色加入村莊時**必有**：

- **登場 CG 1 張**（首次相遇的視覺錨點）
- **登場短劇情 1 段**（500~1500 字）

此規格作為 `character-content-template.md` G5 模板的新增項目（見 G5 更新）。

### 5.2 登場劇情內容原則

- **建立敘事錨點**：為什麼她在這裡？她在村莊/外部做什麼？
- **符合角色基調**：符合各 `character-spec.md` 的個性與說話風格
- **不鋪陳深度**：深入關係留給後續累積（對話 + 送禮 + 好感度）
- **保留懸念**：可留下尚未解答的線索（為後續主線埋伏筆）
- **聚焦於「相遇時刻」**：這是玩家對角色的第一印象

### 5.3 四位登場 CG 的整體方向

| 角色 | 登場 CG 氛圍 | 節奏定位 |
|---|---|---|
| **村長夫人** | 床邊 / 屋內 — 玩家醒來第一面 | 安全感（開場基調） |
| **農女** | 村長夫人帶玩家到田邊，農女抬頭看玩家的瞬間 | 樸實親切（日常延伸） |
| **魔女** | 山邊小屋門口，魔女煉金時被打斷 | 神秘距離感（專業氛圍） |
| **守衛** | 森林邊界 — 警戒 → 澄清 → 握手/賠罪（守衛收劍後，贈劍在後續主動發問取得） | 戲劇性衝突（唯一外部場景，氛圍對比） |

**登場氛圍節奏**：前三位 = 村莊內的**安靜介紹**；守衛 = **戲劇性衝突** → 形成強烈對比，強化守衛登場的記憶點。

### 5.4 工作量影響

- 原 G5 全量估計：122~170 天（4 位）
- 新增：4 張登場 CG（約 2 天）+ 4 段短劇情（約 0.5 天）≈ 2~4 天
- **新總量：124~174 天**（影響極小）

詳細工作量見 `character-content-template.md` v1.2 更新。
