## 基本資訊

- **遊戲名稱**: ProjectDR（暫定）
- **遊戲類型**: 探索搜打撤 + 基地經營增量 + 角色養成/HCG（里程碑獎勵）
- **世界觀**: 奇幻冒險
- **目標平台**: Steam (PC)
- **目標受眾**: 18-35 歲男性、偏好獨立遊戲與視覺小說的輕中度玩家
- **內容分級**: 成人向 (18+)
- **性別傾向**: 男性向
- **結局條件**: 無終局（不設破關條件，玩家自由遊玩）
- **可攻略角色數量**: 4 位

## 目標受眾畫像

- **基本輪廓**: 18-35 歲男性（核心 20-30 歲），輕中度玩家，習慣 Steam PC 端，有 Visual Novel 與獨立遊戲遊玩經驗
- **核心動機**: 安全的冒險刺激（「再多探一格還是撤離」的緊張決策）、經營成就感（「我的努力轉化為可見成長」）、情感連結（「在經營過程中自然地被角色看見」）、增量式持續驅動力
- **情感體驗期待**: 在基地經營過程中，與村莊的多種族女性角色（人類、精靈、獸人等）從陌生到深厚的情感羈絆自然推進，透過經營里程碑漸進式解鎖角色劇情與親密內容
- **參考受眾**: A Dark Room（增量經營愛好者）、HuniePop（男性向成人戀愛養成玩家）、Escape from Tarkov（搜打撤決策愛好者，取其機制概念而非硬核受眾）

## 遊戲概念

在魔物棲息的奇幻森林深處，一座與世隔絕的村莊中醒來的男人，冒險探索不斷變化的危險森林，帶回關鍵物資擴建停滯已久的基地，並在經營過程中與村莊的女性夥伴們培養深厚情感。

## 核心樂趣

**核心樂趣（按優先順序）**:
1. **探索的緊張刺激感** — 每一步都可能是未知，未翻開的格子藏著資源也藏著危機。背包漸滿時「再多探一格還是撤離」的抉擇製造緊張感，安全撤離帶來成就感。探索是本作的核心玩法。
2. **經營成就感** — 看著停滯已久的基地在玩家手中一點一滴擴建起來，每一項新設施、每一次升級都是探索成果的具體證明，基地成長解鎖新探索區域與新功能。

**附屬樂趣**:
3. **角色與 HCG（里程碑獎勵）** — 隨著基地經營推進，角色關係自然深化。角色劇情與 HCG 作為經營里程碑的獎勵自然解鎖，是玩家持續經營的情感回報。角色設計維持完整深度，但驅動力來自經營進展而非主動攻略。好感度系統保留所有行為（對話問答、送食物、送禮物），開發比重低於探索與經營。

**核心循環（宏觀主循環）**:
```
想探索新區域／需要物資
  → 出發探索森林（核心玩法）
    → 搜集物資、擊退魔物
      → 帶著物資安全撤離
        → 物資投入基地經營（建設／升級設施）
          → 基地成長解鎖新功能／新探索區域
            → 經營活動自然推進角色關係
              → 里程碑觸發角色劇情／HCG
                → 回到「想探索新區域／需要物資」
```

### 核心樂趣驗證標準

> 所有設計決策都必須通過以下驗證：

1. **探索是否提供足夠的緊張感與新鮮感？**（每次探索是否讓玩家面對有意義的決策）
2. **是否讓基地的成長更加「可見且有意義」？**（物資投入後，基地有可見的變化與新功能）
3. **探索的風險是否在「可承受範圍內」且直接服務於經營成長？**（懲罰有限、物資有明確用途）
4. **是否創造了讓玩家想看下一步的驅動力？**（存在清晰可見但尚未達到的下一個目標）

## 開發環境

- **遊戲引擎**: Unity 6 (6000.3.8f1)
- **開發語言**: C#
- **UI 系統**: UGUI（Canvas + GameObject）
- **版本控制**: Git
- **預設解析度**: 3840 x 2160 (4K)
- **主要框架**: KahaGameCore（GameData、GameEvent、ValueContainer、EffectProcessor、DialogueSystem、UserInterfaceSystem）、UniTask、DOTween、Addressables
