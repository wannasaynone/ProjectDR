# 對話撰寫 SOP

> **版本**: v1.2

## 系統目的

本文件是製作人撰寫單一角色完整對話內容的線性 checklist。從等級 1 逐級推進，每個等級的所有項目完成後再進入下一等級。完成本 SOP 的全部項目即代表該角色的對話內容量已達 character-content-template.md 規格要求。

---

## Step 0：撰寫角色基礎設定（character-spec.md）

> 所有對話內容的根基。沒有這份文件，後面的等級內容無從下筆。
> 參考格式：`gdd/narrative/village-chief-wife/character-spec.md`

### 基礎資料

- [ ] 身份與背景設定
- [ ] 對應功能（村莊中負責什麼系統/設施）
- [ ] 外觀方向（體型、髮色、服裝風格、年齡外觀、特徵物件）

### 說話風格

- [ ] 主要風格定義（語氣類型、句子長度、用詞傾向）
- [ ] 核心魅力（這個角色說話時要讓玩家感受到什麼）
- [ ] 對話範例 3～5 句

### 稱呼系統

- [ ] 初期稱呼（對應好感度低等級）
- [ ] 中期稱呼（對應好感度中等級）
- [ ] 後期稱呼（對應好感度高等級）
- [ ] 公眾/私下場合的稱呼差異規則

### 核心內心矛盾

- [ ] 表層（外在行為表現）
- [ ] 中層（行為背後的防衛機制）
- [ ] 深層（真正的渴望）
- [ ] 戀愛線的突破口（主角如何打破這個封閉迴路）

### 關係發展曲線

- [ ] 各階段的態度與典型行為（作為等級內容的情感錨點）
- [ ] 核心弧線一句話總結

### 情緒表現規則

- [ ] 關心、生氣、悲傷、開心、害羞各情緒的具體表現方式

### 禁忌清單

- [ ] 列出對話撰寫時絕對不可出現的行為/語氣（至少 3 項）

### 角色發問個性偏好

- [ ] 確認該角色偏好哪種個性類型的選項

### 立繪圖片提示詞

- [ ] 正向提示詞
- [ ] 反向提示詞

### Step 0 完成確認

- [ ] `character-spec.md` 已建立於 `gdd/narrative/<角色名>/` 下
- [ ] `characters.md` 角色總覽已同步更新
- [ ] `FILE_MAP.md` 已同步更新
