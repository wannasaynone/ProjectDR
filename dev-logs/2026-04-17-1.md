## 做了什麼

### 1. MVP 配置外部化（G3）
- `Assets/Game/Resources/Config/mvp-config.json`：所有 Sprint 4 可能調整的數值一律寫入 config，程式碼中完全不出現數值字面值
- `Assets/Game/Scripts/Village/Mvp/MvpConfigData.cs`：JSON DTO + 不可變 `MvpConfig` 建構驗證（非法值拋 ArgumentException）

### 2. MVP 邏輯層模組（B1/B2/C1~C4/D1/D2 邏輯部分/E1/E2/E4）
- `ResourceManager`：木材等資源管理，Add/TrySpend/Has/GetAmount，發 `MvpResourceChangedEvent`（B1）
- `ColdStatusSystem`：訂閱火堆狀態，IsCold 屬性（C3）
- `ActionTimeManager`：統一冷卻計時，寒冷時自動套倍率（B2/C3 整合）
- `SearchSystem`：搜索成功 +1 木材 + 隨機文字（C1）
- `FireSystem`：生火 / 延長 / 倒數 / 熄滅（C2）
- `HutBuildSystem`：蓋小屋解鎖條件（火堆曾點燃）+ 10 木材 + 10 秒（C4）
- `PopulationManager`：人口 Cap / Count（D1）
- `NpcArrivalManager`：訂閱人口上限變更 → 抽 placeholder NPC 加入（D2）
- `DialogueCooldownManager`：玩家對話冷卻，派遣倍率預留（E1）
- `NPCInitiativeManager`：每角色獨立主動發話計時（E2）
- `MvpDialogueSession`：整合既有 DialogueManager + AffinityManager + Cooldown + Initiative，處理紅點路徑與玩家主動路徑的 direction 區分（E4）

### 3. MVP UI 骨架（A1/A2/D3/D4/E3 腳本部分）
- `MvpMainView`（繼承 `ViewBase`）：資源欄 / 狀態列 / 動作按鈕區 / 角色清單區
- `MvpCharacterListItemView`：角色清單單項（名字 / 好感度 / 紅點 / Button）
- `RedDotView`：紅點 UI 元件（E3）
- `MvpCharacterInteractionView`（繼承 `ViewBase`）：MVP 版角色互動，選單 [對話][派遣]，不做送禮/回憶

### 4. MVP 進入點（G1）
- `MvpEntryPoint`（MonoBehaviour）：Start 組裝所有邏輯系統，Update Tick 所有計時類系統

## 為什麼這麼做

- 所有數值外部化，避免未來平衡調整要改程式
- 純邏輯類別（非 MonoBehaviour）讓測試簡單（EditMode，無需場景）
- 既有 DialogueManager / AffinityManager 重用，不動既有系統
- 另外寫 `MvpCharacterInteractionView` 避免破壞既有村莊 CharacterInteractionView（保留備用）
- 事件驅動通訊，模組間不互相強耦合；只有 `MvpDialogueSession` 直接持有既有 manager 作為 method injection
- 本次不建 Prefab/Scene，依製作人指示切分階段：dev-head 只做 Runtime + 規格文件，ui-ux-designer 再依規格建 UI
