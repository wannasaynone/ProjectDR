# 單角色內容量規格模板

> **版本**: v1.1

## 系統目的

本文件定義單一可攻略角色的完整內容規格，作為內容量產的標準模板與成本評估基準。此模板支撐 core-definition.md 的核心循環——角色內容作為探索→經營的里程碑獎勵，透過好感度系統自然解鎖。

本模板的規則覆蓋各角色 character-spec.md 中與系統機制衝突的部分（如五階段關係弧線）。character-spec 仍作為角色人格、說話風格、禁忌清單、立繪提示詞等角色本質設定的參考依據。

## 一、好感度系統

### 1.1 好感度與好感度等級

好感度系統分為兩層：

| 概念 | 說明 |
|------|------|
| **好感度** | 數值，透過對話問答、送食物、送禮物累積 |
| **好感度等級** | 階段，好感度達到當前等級上限時提升 |

- 共 **7 個好感度等級**
- 每次等級提升後，下一等級所需的好感度數值**遞增**
- 具體數值（各等級上限、遞增倍率）由數值設計師定義

### 1.2 好感度累積途徑

| 途徑 | 說明 |
|------|------|
| **對話問答** | 消耗角色體力，從 4 種個性選項中選擇，選中角色偏好 = 好感度多，其餘 = 少或零 |
| **送食物** | 送食品（type:食品）給角色，根據口味配對決定好感度增量，同時恢復角色體力 |
| **送禮物** | 送禮物（type:禮物）給角色，根據喜好配對決定好感度增量 |

### 1.3 好感度等級內事件節奏

每個好感度等級內，依進度百分比觸發以下事件：

```
0%                         50%                        100%
│── 10% 里程碑對話 ×5 段 ──→│── 10% 里程碑對話 ×5 段 ──→│
                            ▼                          ▼
                         一般約會                    H 劇情
                     （約定下次見面）              （等級提升）
```

| 進度 | 事件 | 說明 |
|------|------|------|
| 每 10% | 里程碑對話 | 在互動頁面自動播放的短對話，呈現關係推進的節奏感 |
| 50% | 一般約會 CG 劇情 | 約會演出 = 愛情的結果，結尾約定下次見面，為 H 劇情埋下伏筆 |
| 100% | H 劇情 | 兌現約會時的約定，好感度等級提升，進入下一等級 |

### 1.4 H 劇情觸發條件

H 劇情在好感度等級提升時**自動觸發**，不需要設施等級或基地進度作為前置條件。

**設計依據**：好感度的累積途徑（對話、送食物、送禮物）全部依賴探索帶回的物資——對話受角色體力限制（探索恢復體力），送禮的物品來自製作與料理（需要探索取得素材/種子）。探索循環已透過物資來源自然鎖定，無需額外前置門檻。

## 二、角色體力系統

### 2.1 系統定位

角色體力條的核心設計意圖是**強制中斷對話**，讓玩家無法一次刷完所有對話，必須透過探索或送食品恢復體力後才能繼續互動，維持「探索→回村互動」的循環節奏。

### 2.2 機制

| 項目 | 說明 |
|------|------|
| 體力擁有者 | 每個角色各自擁有獨立的體力條 |
| 消耗 | 選擇功能選單「對話」時消耗體力 |
| 恢復途徑 1 | 外出探索後恢復一部分 |
| 恢復途徑 2 | 贈送食品（type:食品）時恢復 |

- 體力上限、每次消耗量、恢復量等具體數值由數值設計師定義

## 三、對話系統（三層結構）

角色互動中的對話分為三個獨立層次：

| 層次 | 觸發方式 | 體力消耗 | 好感度影響 | 目的 |
|------|---------|---------|-----------|------|
| **招呼語** | 進入互動畫面自動播放 | 無 | 無 | 讓玩家認識角色 |
| **角色發問** | 功能選單「對話」 | 有 | 有（0/少/多） | 考驗玩家對角色的理解 |
| **玩家發問** | 該等級角色發問用完後自動切換 | 有 | 無 | 情報收集（口味/喜好） |

### 3.1 招呼語

- 每個好感度等級 **10 句**，隨好感度等級切換內容
- 進入角色互動畫面時自動播放（打字機效果，可按鍵跳過）
- 不消耗體力、不影響好感度
- 定位為「讓玩家認識角色」的免費內容

### 3.2 角色發問

- 根據角色設定，每角色共 **70 題**（每好感度等級 10 題）
- 選擇功能選單「對話」時，從當前等級的未看過題目中隨機抽取
- 每題提供 **4 種個性選項**，對應 4 種不同的個性類型
- 每個角色有自己的**個性偏好**，玩家需選擇符合角色偏好的選項
- 好感度結果：選中偏好 = +多 / 其餘 = +少或零
- 選項選中後，角色會根據結果給予**動態反應**：
  - 偏好選項：表情差分切換為較熱情的表情 + 正向音效 + 熱情回應台詞
  - 非偏好選項：表情差分切換為較冷淡的表情 + 冷淡音效 + 冷淡回應台詞

### 3.3 玩家發問

- 觸發條件：當前好感度等級的角色發問 10 題**全部看過**後，「對話」功能 100% 切換為玩家發問
- 共用問題池 **40 題**（不分好感度等級）
- 介面從池中隨機抽出 10 題，顯示其中 **4 題**供玩家選擇
- 純情報收集用途——玩家從角色回答中得知口味、喜好等送禮資訊
- 角色每個問題有 **3 種隨機回答**，增加重複遊玩的新鮮感
- 不影響好感度

### 3.4 四種個性選項（規格框架）

對話中的 4 種選項對應 4 種不同的個性類型。具體的四種個性定義待製作人決定。

| 項目 | 說明 |
|------|------|
| 選項數量 | 每題固定 4 個選項 |
| 個性類型 | 4 種（具體定義 TODO） |
| 角色偏好 | 每個角色偏好特定的個性類型 |
| 玩家策略 | 從招呼語、對話內容、角色反應中觀察推測角色偏好 |

## 四、約會與 H 劇情規格

### 4.1 一般約會 CG 劇情（50% 觸發）

| 項目 | 規格 |
|------|------|
| 觸發時機 | 好感度達到當前等級 50% 時自動觸發 |
| 文字量 | 1,500 字 |
| CG 數量 | 5 張（每 300 字切換一張） |
| 敘事定位 | 約會 = 愛情的結果，結尾約定下次見面，為 H 劇情埋下伏筆 |

### 4.2 H 劇情（100% 觸發 / 等級提升）

| 項目 | 規格 |
|------|------|
| 觸發時機 | 好感度達到當前等級 100% 時自動觸發，觸發後好感度等級提升 |
| 文字量 | 3,000 字 |
| CG 數量 | 10 張（每 300 字切換一張） |
| 敘事定位 | H = 兌現約會時的約定，從赴約開場 |

### 4.3 H 劇情內部五階段結構

每場 H 劇情內部包含五階段敘事節奏：

| 階段 | 內容 | 字數 | CG 數 |
|------|------|------|-------|
| 1 | 情境鋪陳（場景起點：兌現約會時約定的「下次見面」，從赴約開場） | 600 字 | 2 張 |
| 2 | 試探開始 | 600 字 | 2 張 |
| 3 | 升溫加深 | 600 字 | 2 張 |
| 4 | 高潮 | 600 字 | 2 張 |
| 5 | 餘韻 | 600 字 | 2 張 |

### 4.4 H 場景激烈程度遞增

7 個好感度等級對應 7 個遞增的 H 場景類型：

| 好感度等級 | H 場景類型 | 激烈程度 |
|-----------|-----------|----------|
| 1 → 2 | 口交 / 手交 | ★ |
| 2 → 3 | 乳交 / 素股 | ★★ |
| 3 → 4 | 傳教士 / 背後位 | ★★★ |
| 4 → 5 | 背後位 / 傳教士 | ★★★★ |
| 5 → 6 | 特殊玩法 1 | ★★★★★ |
| 6 → 7 | 特殊玩法 2 | ★★★★★★ |
| 7 → MAX | 特殊玩法 3 | ★★★★★★★ |

特殊玩法 1~3 的具體內容由各角色的內容設計時決定。

## 五、角色立繪表情差分

所有角色統一使用以下 5 種表情差分：

| # | 表情 | 說明 |
|---|------|------|
| 1 | 一般 | 預設表情 |
| 2 | 驚訝 | |
| 3 | 難過 | |
| 4 | 生氣 | 通用的輕度不滿/嘟嘴（非憤怒，所有角色共用定義） |
| 5 | 高興 | |

表情差分在對話問答中根據選項結果動態切換，提供即時的視覺回饋。
