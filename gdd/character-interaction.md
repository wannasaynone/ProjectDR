# 角色互動系統

> **版本**: v2.3

## 系統目的

角色互動系統是村莊經營與戀愛養成的核心介面。玩家透過此系統與角色見面、對話、使用角色對應的專業功能（委託／倉庫）、承接主線任務、送禮。此系統直接支撐 core-definition.md 的核心樂趣 **#2 經營成就感**（委託循環、擴建循環）與 **#3 角色/HCG**（每次互動都是情感累積的機會）。

自 2026-04-17 定案後，角色互動系統涵蓋委託流程的「交給我吧 → 工作中 → 完成回報」循環、主線任務承接、紅點主動發話等機制。

## 一、互動流程

### 1.1 基礎流程（正常狀態）

```
Hub 主畫面 VillageHubView
  → 點擊角色按鈕
    → 進入角色互動畫面 CharacterInteractionView
      → 自動播放對話（打字機效果）
        → 對話結束
          → 顯示功能選單（4 項）
            → 點擊功能 → 進入對應功能流程
```

### 1.2 委託中狀態流程（工作中）

角色正在執行委託時（詳見 `commission-system.md`），玩家仍可進入（GC 決策：仍可進入）：

```
玩家進入角色 → 顯示工作中狀態
  立繪保留，對話改為描述角色認真工作的樣子
  選單區變為「工作中 mm:ss」倒數顯示
  玩家僅能退出返回 Hub
```

**工作中紅點延後規則**（2026-04-18 新增）：
- 工作中選單被倒數顯示取代，「對話」按鈕不可見，L2 角色發問紅點**無法顯示**
- 若 L2 紅點在工作中累積（倒數時間已到）→ 工作完成領取後才能看到
- 具體行為：玩家完成領取、回到正常選單狀態的**當下**，若 L2 倒數已到，紅點**立即亮起**
- 玩家主動對話冷卻 CD 在工作中仍按 ×2 計算，但不顯示 UI

### 1.3 完成領取狀態流程

紅點亮起時（L1 委託完成 / L4 主線事件）：

```
玩家進入角色 → 播放完成回報對話（取代日常招呼）
  → 顯示產出/事件內容 + [領取/確認] 按鈕
    → 玩家確認 → 產出進入背包 / 事件處理完成
      → 回到第一狀態
```

### 1.4 強制模式狀態流程（節點 0 / 特殊劇情事件）

節點 0 開場時（詳見 `character-unlock-system.md`）與某些主線劇情事件中，interact view 進入**強制模式**：

```
（自動觸發，非玩家主動點擊）
進入角色 interact view → 無返回按鈕（Hub 主動權被鎖）
  → 對話播放（打字機）
    → VN 式選項（必須選擇）
      → 選擇結果在背景生效（按鈕解鎖 / 物資發放 / 劇情推進）
        → 開啟返回按鈕（玩家首次獲得 Hub 主動權）
          → 返回 Hub
```

**特殊規則**：
- 強制模式 = 流程結束前玩家無法逛 Hub、無法進入其他角色
- 返回按鈕在特定時機才開啟（通常是選項完成 + 必要對話結束時）
- 選項必須做出選擇（無「不選」或「取消」選項）
- **選擇後不強制播放劇情演出**（如「帶玩家去見新角色」），角色的登場時刻由玩家主動觸發

**應用場景**：
- 節點 0（開局）：選擇 1 流程
- 節點 1/2 的劇情推進（視劇情設計，可能部分使用強制模式）

### 1.5 首次進入狀態流程（角色登場 CG + 短劇情）

各角色首次被玩家主動點擊時，進入專屬的「首次進入流程」：

```
玩家點擊新解鎖的角色按鈕（首次）
  → 進入 interact view（正常模式，有返回按鈕）
    → 播放角色登場 CG
      → 短劇情（500~1500 字，敘事錨點）
        → 劇情結束 → 進入正常對話狀態
          → 可操作功能選單
```

**特殊規則**：
- 每位角色的「首次進入流程」**只觸發一次**，之後進入為正常流程（1.1）
- 登場 CG + 短劇情不在節點 0 內播放，而是**玩家主動點擊才見到**
- 此流程不是強制模式（玩家可隨時按返回離開）
- 實作需記錄「該角色是否已首次進入」的狀態

## 四、規則

1. **所有角色統一流程**：點擊任何角色都進入相同的互動畫面結構
2. **每次進入都播放對話**：不區分首次/重複，每次進入自動播放對話
   - **例外**：委託工作中狀態下，對話改為「認真工作」描述（§六）
   - **例外**：紅點亮時分流處理（2026-04-18 修訂）——**L1 委託完成 / L4 主線事件** 進入自動播放並**取代招呼語**（§1.3 / §5）；**L2 角色發問** 下沉為「對話」按鈕紅點，玩家點擊才觸發（§5.1），招呼語仍正常播；**L3 新任務** 下沉為「任務」按鈕紅點（§8）
3. **送禮完成後返回互動選單**：送禮完成後自動關閉送禮畫面，返回角色互動畫面，播放角色感謝對話，並預留好感度提升特效顯示空間
4. **立繪動畫**：IT 階段預留 Animator 欄位，使用 placeholder 或留空
5. **功能懸浮覆蓋**：點擊 `[倉庫]`、`[送禮]`、`[任務]` 時，功能頁面在角色互動畫面上方懸浮顯示
6. **提煉介面狀態切換**（新）：點擊專業委託按鈕（`[耕種][煉製][探索周圍]`）時，**右側對話框與功能選單消失**，顯示格子式工作台；按「開始」後切回第二狀態並播放「交給我吧」敘事對話，選單變為「工作中 mm:ss」
7. **工作中可進入**（GC 決策）：委託執行中玩家仍可進入該角色 interact view，但無功能按鈕可點，僅能退出
8. **紅點優先序**（2026-04-18 修訂）：
   - **L1 / L4**（自動播放類）：依優先序 L1 → L4 進入後立即自動播放，播完後若 L2/L3 紅點仍存在則**保留顯示**
   - **L2 / L3**（按鈕紅點類）：下沉至對應按鈕（對話/任務），與自動播放類並存而非互斥
   - 詳見 §5 與 `commission-system.md` §4.3

## 五、對話發起方向規則（2026-04-18 修訂為紅點分流架構）

對話分為兩條路徑，由紅點有無決定分流：

| 路徑 | 觸發條件 | 觸發方式 | 內容來源 | 體力消耗 |
|---|---|---|---|---|
| **a 路徑 — 角色主動發話** | L2 紅點亮（對話按鈕紅點） | 玩家點「對話」按鈕觸發 | 角色發問 70 題 | **不消耗** |
| **b 路徑 — 玩家主動發話** | 無紅點時 | 玩家點「對話」按鈕觸發 | 玩家發問 40 題 | **消耗** |

**紅點分流規則**：
- L1 委託完成 / L4 主線事件：進入角色 → **自動播放**（維持 §1.3 既有規格）
- L2 角色發問：進入角色 → 「對話」按鈕**顯示紅點** → 玩家**點擊才觸發**（不自動播放）
- L3 新任務：`[任務]` 按鈕紅點 → 玩家點擊才觸發（既有規格）
- **L1/L4 自動播放結束後，若 L2/L3 紅點仍存在 → 保留顯示**（不會因 L1/L4 播完而清空）
- **招呼語**：無紅點時才播；L2 紅點亮時，紅點觸發對話取代招呼語

### 5.1 角色主動發話（a 路徑 — L2 紅點觸發）

**紅點觸發規則**：
- 固定時間倒數（**60 秒 placeholder**，數值設計師後續調整）到時，「對話」按鈕亮紅點
- **工作中暫停倒數**（角色正在執行委託時，倒數計時停止）
- **紅點累積上限：1**（規則層，不可由數值調整）：紅點亮後，即使再次達到倒數條件也不累積；紅點永久保留，直到玩家點擊觸發對話後才清除

**UX 流程**：

```
進入角色頁面 → 「對話」按鈕顯示紅點
  ↓ 玩家點擊「對話」
關閉右下功能選單
  ↓
播放角色發問對話（打字機）
  ↓
顯示四個回應選項（只顯示文字，不顯示好感度數值）
  ↓ 玩家選擇
播放對應回應對話 + 表情差分切換
  ↓
回復原本的功能選單（四項：專業/任務/對話/送禮）
```

**四選項規格**：
- 四個選項對應 4 種個性類型（具體定義見 `character-content-template.md` §3.4）
- 每個個性類型固定對應一檔好感度：+0 / +2 / +5 / +10（**placeholder**，數值設計師後續調整）
- **UI 只顯示選項文字，不顯示好感度數值**（規則層）：玩家需從角色反應與表情差分推測偏好

**體力**：a 路徑**不消耗體力**

### 5.2 玩家主動發話（b 路徑 — 無紅點時點「對話」）

**觸發條件**：對話按鈕**無紅點**時，玩家點擊「對話」按鈕

**UX 流程**：

```
點擊「對話」（無紅點狀態）
  ↓
關閉右下功能選單
  ↓
依剩餘題目數量顯示選項：
  - 剩餘 ≥ 4 題 → 隨機抽 4 題顯示
  - 剩餘 1~3 題 → 只顯示該 1~3 題（不補閒聊、不補滿 4 格）
  - 剩餘 0 題  → 只顯示 [閒聊] 選項
  ↓ 玩家選擇問題
角色給予固定對應回答（1 對 1）
  ↓
回復原本的功能選單
```

**[閒聊] 模式**：當 40 題問題池全部耗盡，改為閒聊模式 fallback
- 角色**隨機發問** + 角色**隨機回答**（不影響好感度）

**冷卻 CD**（規則層）：
- 玩家主動對話冷卻基礎值：**60 秒（placeholder**，數值設計師後續調整**）**
- 工作中 CD 加倍：×2（= 120 秒 placeholder）（**工作中 ×2 倍率為規則層，不可由數值調整**）
- 冷卻中不能點「對話」（b 路徑）；工作中冷卻仍計算，但不顯示 UI

**體力**：b 路徑**消耗體力**；體力不足時，點擊「對話」顯示提示文字「現在好累了」

### 5.3 兩種路徑的互斥與優先

- 對話按鈕有 L2 紅點時 → 固定走 a 路徑（角色發問）
- 對話按鈕無紅點時 → 走 b 路徑（玩家發問），需通過冷卻 CD 與體力檢查
- 兩者使用不同的內容池（角色發問 70 題 vs 玩家發問 40 題），不混用

## 六、工作中狀態完整設計

製作人 2026-04-17 第八輪 GC 決策：工作中**仍可進入**，選單變倒數顯示、對話改描述角色認真工作。

### 6.1 佈局

```
┌───────────────────────────────────────┐
│                                       │
│   ┌─────────┐      ┌──────────────┐   │
│   │         │      │ 對話：        │   │
│   │  立繪   │      │（認真工作中   │   │
│   │         │      │  的描述）     │   │
│   └─────────┘      ├──────────────┤   │
│                    │ 工作中...    │   │
│                    │    03:11     │   │
│                    └──────────────┘   │
└───────────────────────────────────────┘
```

### 6.2 設計細節

| 元素 | 規格 |
|---|---|
| **立繪** | 使用「認真工作」表情差分（若無則使用「一般」） |
| **對話** | 描述角色正在認真工作的樣子（例：「她專心地混合著藥草，沒有注意到你進來」） |
| **倒數顯示** | `mm:ss` 格式（例：「工作中... 03:11」） |
| **功能選單** | 不顯示四項選單按鈕，以倒數取代 |
| **可執行操作** | 僅能退出 interact view 返回 Hub |
| **情感魅力點** | 玩家看到角色專注工作的樣子——本身就是一種親密感的來源 |

### 6.3 適用對象

- 委託型三位（農女/魔女/守衛）：專業委託執行中
- 村長夫人：倉庫擴建執行中（同樣採「工作中 mm:ss」模式）

## 八、紅點系統整合

紅點出現的位置、觸發條件與下沉規則：

| 層級 | 紅點位置（外層） | 進入角色後行為 | 內容類型 |
|---|---|---|---|
| **L1 委託完成** | Hub 畫面角色頭像 | 自動播放完成回報對話（取代招呼語） | 完成回報 + 領取介面 |
| **L4 主線事件** | Hub 畫面角色頭像 | 自動播放主線事件對話（取代招呼語） | 主線事件敘述 |
| **L2 角色發問** | Hub 畫面角色頭像 → **下沉至「對話」按鈕紅點** | 玩家點「對話」才觸發（不自動播放） | 角色發問（a 路徑） |
| **L3 新任務** | 直接在 Interact view 的 `[任務]` 按鈕 | 玩家點「任務」才觸發 | 任務描述 + 承接提示視窗 |

**L2 紅點下沉規則**（2026-04-18 新增）：
- L2 紅點先在 Hub 角色頭像顯示（提醒玩家進入）
- 玩家進入角色 Interact view 後，紅點**下沉至「對話」按鈕**顯示
- 玩家必須主動點擊「對話」按鈕才觸發角色發問，不會自動播放

**L1/L4 播完後的保留規則**（2026-04-18 新增）：
- L1/L4 自動播放結束後，若 L2 或 L3 紅點仍存在 → **保留顯示**，不因 L1/L4 完成而清除
- 範例流程：進入角色 → 自動播 L1 完成回報 → 領取完成 → 「對話」按鈕仍顯示 L2 紅點 → 玩家點擊觸發角色發問

詳細紅點機制見 `commission-system.md` 第四章「紅點觸發四層」。
