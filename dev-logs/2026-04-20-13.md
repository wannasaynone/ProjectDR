## 根因分析

### F10 盤點的分類錯誤

F10 在分析 `InitializeCharacterView` callback 的 side effects 時，製作了以下分類表：

| 分類 | 程式碼 | 說明 |
|------|--------|------|
| **(a) CG 播放本身** | `MarkIntroCGPlayed` | only-once 旗標，防止重播 |
| **(a) CG 播放本身** | `capturedView.SetState(Normal)` | ← **誤分類** |
| **(b) 首次登場狀態** | `SetFirstMeetFlag(false)` | 清除 Hub 引導紅點 |
| **(c) 常規進入流程** | `SetMainQuestEventFlag(VCW, true)` + 節點旗標 | 引導回村長夫人 |

`SetState(Normal)` 的實際效果是「進入 interact view 後切換至正常顯示模式」，**不是 CG 播放本身**。CG 播放的 only-once 旗標是 `MarkIntroCGPlayed`；`SetState(Normal)` 是顯示模式切換，兩條路徑都需要執行。

### F11 bug 的觸發機制

路徑 B（守衛特例，CG 已由守衛歸來事件提前標記）執行流程：

```
InitializeCharacterView(Guard)
  ↓
SetCharacter → _pendingDialogueStart = true
  ↓
路徑 B：HasPlayedIntroCG = true → 跳過 CG
  ↓（F10 共用方法，但缺少 SetState(Normal)）
OnCharacterEnteredAndCGDone → (b)/(c) side effects
  ↓
PushView → OnShow
  ↓
_currentState 未明確設為 Normal（雖然預設是 Normal，但若之前被設為其他狀態則殘留）
  ↓
StartDialoguePlayback → 若 state ≠ Normal → GreetingPresenter.TryGreet 不被呼叫
                      → greeting 不觸發
```

路徑 A（農女/魔女/VCW 標準路徑）：
```
PlayIntroCG callback:
  MarkIntroCGPlayed
  SetState(Normal)  ← 明確設定
  OnCharacterEnteredAndCGDone
```

路徑 A 明確設了 `SetState(Normal)`，路徑 B 沒有。`CharacterInteractionView._currentState` 雖然預設是 `Normal`，但若其之前曾因委託狀態或其他原因被設成 `CommissionInProgress` 等非 Normal 狀態後，路徑 B 的 `SetState` 缺失就會導致 view 留在上次的 state。

## 修正內容

### Phase 2：重新分類 `SetState(Normal)`

將 `SetState(Normal)` 從「(a) CG 播放本身」重新分類為「(display) 進入 interact view 後的常規顯示流程」，移入 `OnCharacterEnteredAndCGDone` 共用方法。

### Phase 3：修改 `VillageEntryPoint.cs`

#### `InitializeCharacterView` 路徑 A callback（~行 1026）

**修改前**：
```csharp
_cgPlayer.PlayIntroCG(characterId, () =>
{
    MarkIntroCGPlayed(capturedCharId);
    if (capturedView != null && capturedView.gameObject != null)
    {
        capturedView.SetState(CharacterInteractionState.Normal);  // ← 從這裡移走
    }
    OnCharacterEnteredAndCGDone(capturedCharId);  // 舊簽名 (string)
});
```

**修改後**：
```csharp
_cgPlayer.PlayIntroCG(characterId, () =>
{
    MarkIntroCGPlayed(capturedCharId);
    OnCharacterEnteredAndCGDone(capturedCharId, capturedView);  // 新簽名 (string, view)
});
```

#### `InitializeCharacterView` 路徑 B（~行 1037）

**修改前**：
```csharp
OnCharacterEnteredAndCGDone(characterId);  // 舊簽名 (string)
```

**修改後**：
```csharp
OnCharacterEnteredAndCGDone(characterId, interactionView);  // 新簽名 (string, view)
```

#### `OnCharacterEnteredAndCGDone` 方法簽名與內容（~行 1075）

**修改前**：`private void OnCharacterEnteredAndCGDone(string characterId)`

**修改後**：`private void OnCharacterEnteredAndCGDone(string characterId, CharacterInteractionView view)`

新增在方法開頭的 (display) 副作用：
```csharp
// (display) 切換至 Normal 顯示模式（兩條路徑共用）
if (view != null && view.gameObject != null)
{
    view.SetState(CharacterInteractionState.Normal);
}
```
