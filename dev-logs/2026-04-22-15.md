# Dev Log 2026-04-22-15 — Sprint 7 D7 緊急修復：VillageContext gameDataAccess null

## 任務

Sprint 7 D7 實機 playthrough 觸發 Runtime Exception：

```
ArgumentNullException: Value cannot be null.
Parameter name: gameDataAccess
ProjectDR.Village.Core.VillageContext..ctor (VillageContext.cs:93)
ProjectDR.Village.Core.VillageEntryPoint.RunInstallers (VillageEntryPointInstallers.cs:79)
```

## 根因分析

`VillageEntryPointInstallers.cs:79` 傳入 `gameDataAccess: null`，但 `VillageContext.ctor` 有明文 null check：

```csharp
GameDataAccess = gameDataAccess ?? throw new System.ArgumentNullException(nameof(gameDataAccess));
```

此為 Sprint 7 B3 建立 VillageContext 時設計的必要欄位，但後續批次未補真實 delegate，導致 placeholder null 在 runtime 炸出。

## 診斷過程

1. Grep `ctx.GameDataAccess` 全專案 Scripts 目錄
   - 結果：無任何 Installer 的 `Install(ctx)` 內呼叫 `ctx.GameDataAccess(id)` 做實際查詢
   - `AffinityInstaller.cs:9` 與 `CGInstaller.cs:4` 只在**注釋**中提到「未來用」
2. 確認 ADR-003 Appendix B（第 604 行）：「Sprint 7 A 類完成後 ADR-002 退出，屆時 GameDataAccess 才真正成為唯一資料入口；Sprint 7 期間與 ADR-002 並行存活」
3. 確認 ADR-002 退出 Gate 仍為 Sprint 8 目標
4. 結論：Sprint 7 期間 `ctx.GameDataAccess` 無實際使用者 → 修法 A（放寬 null check）正確

## 修復內容

### 1. `VillageContext.cs` — 放寬 ctor null check

- 移除對 `gameDataAccess` 的 `ArgumentNullException` throw
- 改為 `GameDataAccess = gameDataAccess;`（允許 null）
- 新增詳細 XML 文件說明 Sprint 7 的允許理由與 Sprint 8 的恢復計畫
- 新增 `// TODO Sprint 8`：「ADR-002 退出後恢復 null check」

### 2. `VillageEntryPointInstallers.cs` — 加 TODO 說明

- 在 `gameDataAccess: null` 上方新增三行注釋，說明 Sprint 7 期間傳 null 的原因、Sprint 8 真實實作方向
