## 背景：dev-head 審核 DEV-CODE-REVIEW FAIL

本次實作為 dev-head 對 Sprint 6 C+D 類審核後發現的關鍵缺漏補修。
root cause：前一輪 dev-agent 修了 MainQuestManager / CharacterUnlockManager / RedDotManager 等下游，
但漏改了 `VillageEntryPoint.cs` 這個訊號發送的上游入口，導致：

1. 新 T1 訊號源仍是舊的 `FirstCharIntroComplete`（送自「非 VCW CG 播完」lambda），
   但 B1 已把 T1 的 `completion_condition_value` 改為 `node_2_dialogue_complete`，
   兩端不匹配 → 新 T1 永遠無法完成 → 探索永遠不開放（100% 驗收失敗）

2. `SetMainQuestEventFlag(VCW, true)` 在 production code 完全沒被呼叫 → 節點 1 L4 紅點永不亮 → 玩家找不到入口

3. 節點 1 完成時的 `CharacterUnlockManager.ForceUnlock` + `DispatchInitialResourceGrants` 仍殘留
   （但 Sprint 6 已在 CharacterUnlockManager.OnDialogueChoiceSelected 處理 unlock；
   B2 已移除 unlock_farmgirl/witch grants，DispatchInitialResourceGrants 呼叫變死碼）

4. `MainQuestConfigData.cs` 的 `FirstCharIntroComplete` 兩常數只有 XML 說明，無 `[System.Obsolete]` attribute

5. `InitialResourcesConfigTests.GetGrant_ExistingGrantId_ReturnsEntry` 仍測試已刪除的 `unlock_farm_girl_seed`
