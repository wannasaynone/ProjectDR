# 核心定義文件

> **版本**: v3.3

## 系統目的

本文件定義 ProjectDR 的遊戲本質，包含核心循環、樂趣來源、驗證標準、目標受眾等基礎設計方向。所有後續設計決策應以本文件作為最高依據，確保每一個系統與內容設計都在強化核心樂趣，而非稀釋它。

---

## 一、遊戲概念一句話描述

在魔物棲息的奇幻森林深處，一座與世隔絕的村莊中醒來的男人，冒險探索不斷變化的危險森林，帶回關鍵物資擴建停滯已久的基地，並在經營過程中與村莊的女性夥伴們培養深厚情感。

> **世界觀**：奇幻冒險。被不斷變化的森林圍困、發展停滯的村莊，玩家作為唯一能安全往返森林的男性，探索森林、經營基地並與多種族女性角色（人類、精靈、獸人等）培養感情。

## 二、核心循環 (Core Loop)

### 主循環（宏觀層）

```
想探索新區域／需要物資
  → 出發探索森林（核心玩法）
    → 在地圖上搜集物資、擊退魔物
      → 帶著物資安全撤離
        → 物資投入基地經營（建設／升級設施）
          → 基地成長解鎖新功能／新探索區域
            → 經營活動自然推進角色關係
              → 里程碑觸發角色劇情／HCG
                → 回到「想探索新區域／需要物資」
```

### 次循環 A：探索循環（單次出擊）

```
進入地圖
  → 自由移動探索（角色中心進入新格子即翻開，踩地雷式揭露）
    → 發現物品箱 → 開啟並拾取物品（等待解鎖計時）
    → 接觸魔物 → 受傷擊退 / 射擊戰鬥（Top-down 射擊）
      → 擊敗或迴避
  → 背包趨近飽和 或 風險升高
    → 判斷：繼續深探 vs 撤離
      → 返回出發點 或 進入撤離點等待 6 秒
        → 回村後手動將背包物品存入倉庫
```

### 次循環 B：委託循環（經營日常，委託型角色：農女/魔女/守衛）

```
進入角色 interact view（自動播放對話）
  → 點擊專業功能按鈕（耕種 / 煉製 / 探索周圍）
    → 格子式工作台（多格子並行）
      → 放入單物品配方輸入（背包物品，僅可處理項可放入）
        → 結果預覽 + 按「開始」
          → 切回 interact view，播放「交給我吧」敘事對話
            → 功能選單變為「工作中 mm:ss」倒數
              → 玩家離開可做別的事（探索、其他角色、其他委託）
                → 倒數完成 → Hub 角色頭像亮紅點（L1 委託完成層）
                  → 玩家進入角色 → 完成回報對話 → 領取產出
                    → 回到可委託狀態
```

### 次循環 C：擴建循環（村長夫人專屬）

```
倉庫空間緊張 或 達到 70% 使用率（可由主線任務引導觸發）
  → 進入村長夫人 → 倉庫 → 擴建按鈕
    → 確認物資（先背包後倉庫扣除）
      → 擴建倒數開始（現實時間）
        → 玩家做別的事（探索、委託、對話）
          → 倒數完成 → 村長夫人頭像紅點
            → 進入村長夫人 → 完成對話 → 倉庫 +50 格
              → 回到日常循環（更大空間支援更多物資）
```

### 次循環 D：感情培養循環（所有角色）

```
與角色互動（對話、送禮、委託流程中的敘事接觸）
  → 好感度提升（對話問答 + 送食物 + 送禮物 三途徑）
    → 紅點主動發話（L2 角色發問）/ 里程碑階段躍遷
      → CG / HCG 解鎖
        → 加深對角色的情感投入
          → 更積極投入探索、經營與主線推進
```

### 主線任務引導貫穿其中

主線任務系統（`main-quest-system.md`）作為**機制引導工具**，通過 L3（任務按鈕紅點）與 L4（事件層）觸發，告訴玩家「下一步可以做什麼」。主線完成 ≠ 破關，與決策 #5「無終局」一致。

**四條循環的交匯點**：探索帶回的物資 → 委託循環轉化為食物/禮物/基礎物資 → 送禮/送食物驅動感情循環 → 擴建循環支撐更大體量的物資流轉 → 主線任務引導解鎖新機制與新區域。四循環彼此驅動、互為補給，形成探索→經營→感情的完整正向迴圈。

### 前期為漸進解鎖循環（Dark Room 漸進揭露節奏）

自 2026-04-17 第十二~十八輪定案後，**前期階段並非四條循環全部啟動**，而是採漸進解鎖結構（詳見 `character-unlock-system.md`）：

```
階段 1（開局）          ：僅循環 D（感情，對村長夫人）
階段 2（選擇 1 解鎖）    ：循環 B（對選擇 1 角色）+ D
階段 3~4（節點 1 + 選擇 2）：循環 B（對兩位委託角色）+ D
階段 5（節點 2）         ：循環 B + C（擴建，若倉庫觸發）+ D
階段 6（探索開放）       ：循環 A（探索）+ B + C + D（四循環完整啟動）
階段 7（守衛加入）       ：四循環完整 + 守衛委託 + 探索周圍與深探分化
```

**後期為完整 4 循環並行運作**——四循環互為驅動的交匯點在階段 7 完全成形。

對應 Dark Room 的「stoke fire → gather wood → check traps → hunt → 村民到來 → **go to the forest**」漸進揭露哲學，前期節奏的核心精神為：**每個新機制都伴隨新角色或新系統的解鎖，玩家永遠有「下一步在哪」的明確召喚**。

## 三、核心樂趣 (Core Fun)

### 核心樂趣（按優先順序）

| # | 樂趣類型 | 描述 |
|---|---|---|
| 1 | **探索的緊張刺激感** | 每一步都可能是未知，未翻開的格子藏著資源也藏著危機。背包漸滿時的「再多探一格還是撤離」抉擇製造緊張感，安全撤離帶來成就感。探索是本作的核心玩法，玩家打開遊戲時心裡想的是「今天要探索哪張地圖」。 |
| 2 | **經營成就感** | 看著停滯已久的基地在自己手中一點一滴擴建起來，每一項新設施、每一次升級都是玩家探索成果的具體證明。基地成長解鎖新探索區域與新功能，形成正向循環。 |

### 附屬樂趣

| # | 樂趣類型 | 描述 |
|---|---|---|
| 3 | **角色與 HCG（里程碑獎勵）** | 隨著基地經營推進，角色從陌生人逐漸成為真正的夥伴乃至戀人。角色劇情與 HCG 作為經營里程碑的獎勵自然解鎖，是玩家持續經營的情感回報。角色設計維持完整深度（五階段弧線、差異化個性），但驅動力來自經營進展而非主動攻略。 |

> **設計方向備忘**：角色與 HCG 的定位是經營基地的**里程碑獎勵**——角色內容維持高品質與完整深度，但玩家不需要主動「刷好感度」。經營活動（送禮、對話、完成任務）自然累積好感度，好感度到達門檻時自動觸發劇情與 HCG。設計新內容時，應確認該內容強化「探索→經營→自然解鎖角色內容」的流暢度。好感度系統保留所有行為（對話問答、送食物、送禮物），但開發比重低於探索與經營。

## 四、核心樂趣驗證標準

以下標準用於判斷「這個設計是否強化了核心樂趣？」。所有新系統、新功能、新內容在提案時，應能通過至少一項標準。

### 標準 1：探索是否提供足夠的緊張感與新鮮感？

> 每次探索是否讓玩家感到刺激，而非例行公事？
>
> - **通過**：玩家在第 N 次探索時仍面對有意義的決策（深探 vs 撤離、背包取捨）；地圖/物品箱/魔物的組合提供足夠變化
> - **不通過**：探索流程固化為重複操作，缺乏決策張力；玩家感到「又要跑一趟」

### 標準 2：是否讓基地的成長更加「可見且有意義」？

> 玩家能否清楚地看到探索成果轉化為基地的改變？
>
> - **通過**：投入物資後，基地有可見的變化（新設施、升級效果），且這些變化帶來實際的新功能或解鎖新探索區域
> - **不通過**：物資消耗後只有數字變化，無可見的成長呈現

### 標準 3：探索的風險是否在「可承受範圍內」且直接服務於經營成長？

> 探索帶來的壓力與回報，是否在鼓勵玩家繼續投入主循環，而非讓人沮喪退出？
>
> - **通過**：玩家死亡或探索失敗的代價是有限且可恢復的；取得的物資有明確的經營用途
> - **不通過**：探索懲罰過重導致玩家不願出發；物資堆積但無法在基地中使用

### 標準 4：這個設計是否創造了讓玩家想看下一步的驅動力？

> 玩家在這個設計中，是否有一個「下一個目標」在前方召喚他繼續玩下去？
>
> - **通過**：玩家完成當前動作後，自然浮現「接下來想做 X」或「想看看 X 之後會怎樣」的念頭；存在一個清晰可見但尚未達到的下一個解鎖/進展
> - **不通過**：一個階段結束後，玩家感到「已經完成了、可以關掉了」，缺少向前拉動的鉤子

## 五、目標受眾與玩家畫像

### 基本輪廓

| 項目 | 描述 |
|---|---|
| **年齡層** | 主要 18–35 歲；核心受眾 20–30 歲 |
| **性別傾向** | **男性向** |
| **內容分級** | **成人向 (18+)**，上架 Steam 含成人內容 |
| **遊戲經驗程度** | 輕中度玩家為主；能接受一定深度，但不追求硬核操作門檻 |
| **平台習慣** | 習慣 Steam PC 端；可能有大量 Visual Novel、獨立遊戲遊玩經驗 |

### 核心動機

這群玩家在遊戲中追求的是：

- **「安全的冒險刺激」**：在可控的風險範圍內體驗緊張感，每次探索都是一場小冒險
- **「我的努力有意義」**：探索帶回的物資轉化為基地的可見成長，每一次升級都是努力的證明
- **「被角色真正看見」**：在經營過程中自然地與角色建立關係，獲得情感回報
- **「下一個解鎖在哪」**：增量遊戲式的「再看一下接下來有什麼」的持續驅動力

### 浪漫／情感場景期待（提案方向，待製作人確認）

> 以下為設計團隊根據參考作品分析所提出的角色互動設計方向建議，需製作人確認是否採用。

- 角色有**立體的個性與過去**，不是單純的服務型角色
- **漸進式的信任建立**：角色從防備到敞開心扉的過程有清晰的情感弧線
- **因為玩家的選擇而不同的關係走向**：感覺自己的行為真的影響了角色
- **私密感**：特定劇情是「只有親密度夠才能看到的」，製造專屬感
- **成人向的肢體與情感親密描寫**，作為親密度達到高階段後的情感回報

### 參考作品借鑑要素

> **說明**：以下三款參考作品的目標受眾並不重疊，本作並非以任何一款的受眾為全部目標。下表標注的是**本作從各作品借鑑的機制與設計理念**，而非借鑑其受眾。

| 作品 | 玩家群特徵（僅供背景理解） | **本作借鑑要素** |
|---|---|---|
| **A Dark Room** | 喜愛極簡文字冒險與增量成長、對隱藏敘事充滿好奇、容忍等待過程的玩家 | 資源驅動的基地經營循環；增量成長帶來的「再做一次」驅動力；以極少的呈現承載強烈的成就感 |
| **HuniePop** | 主動追求戀愛養成內容、對角色立繪與個性投入、接受成人內容的成年玩家 | 角色差異化的好感培養系統；角色個性與解鎖內容的情感投入設計；親密度解鎖的漸進式內容結構 |
| **Escape from Tarkov** | 享受高風險高回報的搜打撤體驗、對裝備管理與撤離決策有樂趣 | 撤離機制帶來的緊張感與決策張力（再深探一格 vs 安全撤離）；物資不能全帶走的取捨壓力 |

## 六、遊戲體驗關鍵字

| # | 關鍵字 | 說明 |
|---|---|---|
| 1 | **已知與未知之間 (Known vs Unknown)** | 踩地雷式的揭示張力，每一步都是選擇 |
| 2 | **帶回家 (Bring It Home)** | 每次探索的核心動作：冒險出去，安全回來，把資源帶回基地 |
| 3 | **恰到好處的緊張 (Comfortable Tension)** | 有壓力但不絕望；危險存在但後果可承受 |
| 4 | **看見成長 (Visible Progress)** | 探索→經營→基地成長，每一步付出都轉化為可見的進展 |
| 5 | **羈絆 (Bond)** | 在經營過程中與角色自然深化的情感連結 |
