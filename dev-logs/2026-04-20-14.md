舊流程（F11 以前）：
```
守衛歸來 → Hub 解鎖 + 探索關閉
  → 玩家點 Hub 守衛按鈕 → 進入 CharacterInteractionView
  → 玩家主動從問題清單選「要拿劍」（guard_ask_sword 特殊題）

  → 首次進入自動播放取劍對白（不需要玩家選題）
  → 對白完成 callback → 發劍 + ExplorationGateReopenedEvent
```

### 修改：`CharacterInteractionView.cs`

新增 override dialogue 機制（timing 解耦）：

**新增欄位**：
- `_firstMeetOverrideDialogueLines`: string[]
- `_firstMeetOverrideCallback`: System.Action

**新增 API**：
```csharp
public void SetFirstMeetOverrideDialogue(string[] lines, System.Action onComplete)
```
必須在 `OnShow()` 前呼叫（在 `InitializeCharacterView` / `OnCharacterEnteredAndCGDone` 中設定）。

**修改 `StartDialoguePlayback()`**：方法開頭偵測 override，若有則：
1. 清除 override（確保只觸發一次）
2. 播放 override dialogue
3. 訂閱 `DialogueCompletedEvent` 的自解訂 lambda，完成後呼叫 callback

時序設計：
```
InitializeCharacterView (view 尚未 active)
  → OnCharacterEnteredAndCGDone
    → SetFirstMeetOverrideDialogue(lines, callback)  ← 設定 override
  → PushView → OnShow → StartDialoguePlayback       ← 偵測並執行 override
```

### 修改：`VillageEntryPoint.cs`

**新增欄位**：
- `[SerializeField] TextAsset _guardFirstMeetDialogueConfigJson`
- `GuardFirstMeetDialogueConfig _guardFirstMeetDialogueConfig`
- `bool _guardFirstMeetDialogueCompleted`（session 記憶體旗標）

**`InitializeManagers()`**：載入 `_guardFirstMeetDialogueConfig`

**`OnCharacterEnteredAndCGDone()`**：在方法末尾新增守衛 first-meet 判斷：
```csharp
if (characterId == CharacterIds.Guard
    && !_guardFirstMeetDialogueCompleted
    && _characterUnlockManager != null
    && _characterUnlockManager.IsUnlocked(CharacterIds.Guard)
    && view != null)
{
    string[] lines = System.Linq.Enumerable.ToArray(_guardFirstMeetDialogueConfig.DialogueLines);
    view.SetFirstMeetOverrideDialogue(lines, OnGuardFirstMeetDialogueCompleted);
}
```

**新增 `OnGuardFirstMeetDialogueCompleted()`**：
1. `_guardFirstMeetDialogueCompleted = true`
2. 透過 `GetGrantsByTrigger(InitialResourcesTriggerIds.GuardSwordAsked)` 取得並派發劍 grant
3. `EventBus.Publish(new ExplorationGateReopenedEvent())`
