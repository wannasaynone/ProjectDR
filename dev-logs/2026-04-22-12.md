## 技術決策

1. **不使用提案二的精確路徑推算**：提案二的 `check_config_json_has_igamedata` 使用 `../../Scripts` 相對路徑，在不同呼叫位置可能不穩定；改用從 JSON 路徑提取專案根 + 固定 `Assets/Game/Scripts` 子目錄，與既有 `check_igamedata_contract` 一致。

2. **警示不阻擋**：依 CLAUDE.md「自動驗證 Hooks」原則，id 欄位缺失僅警示（exit 0），不阻擋寫入。只有 JSON 語法錯誤才阻擋。

3. **jq 降級**：無 jq 環境自動跳過 id 欄位檢查，確保 hook 在各種環境下不會因工具缺失而干擾正常工作流。
