# 委託系統（Commission System）

> **版本**: v1.5

## 系統目的

委託系統承接 `core-definition.md` v3.1 的次循環 A（探索循環）與主循環中「物資投入基地經營」的節點，同時是核心樂趣 **#2 經營成就感**與 **#3 角色/HCG**的交匯點。

本系統的設計本質是「**既有設施功能的敘事升級**」——玩家的種植、煉製、採集等行為不是直接對設施操作，而是**透過角色介面完成**。角色扮演「接受委託、代為執行、交付成果」的敘事角色，使：

- **核心樂趣 #2（經營成就感）**：物資轉化為可見成長的環節被包裝成「拜託角色幫忙」的日常情境
- **核心樂趣 #3（角色情感）**：每一次委託流程都是完整的角色接觸（交付 → 對話 → 等待 → 領取 → 感謝），與好感度系統自然接軌
- **4/16「設施概念移除」決策的修訂**：設施仍作為委託的底層運作邏輯存在（農田、煉金爐、採集路線），但**玩家介面永遠是角色**，角色不被降格為「設施操作員」

委託系統亦借鑒 A Dark Room 的**漸進揭露哲學**與**等待節奏**：配方發現式的格子工作台提供「一直在解鎖新東西」的驚喜感；倒數機制讓玩家「交出去 → 做別的事 → 回來領取」形成並行節奏。

## 零、非強制定位（2026-04-20 設計轉向）

> **設計轉向原因**：Sprint 6 設計討論（2026-04-20）決策 1/2，委託系統從「強制主線教學」改為「玩家自主發現」，徹底移除委託強制教學流程。

### 0.1 委託系統的自主發現定位

委託系統是**玩家自主發現的次要系統**，而非主線強制教學目標：

- **無主線強制要求**：前期主線任務 T0/T1/T2 的觸發與完成條件中，不含任何「完成一次委託」的要求
- **無紅點強推**：系統不會強制將紅點導向委託入口，促使玩家進行委託
- **玩家自主決定**：玩家可能玩很久都不碰委託，也可能第一次進入角色 interact view 就自然發現並使用
- **設計哲學**：沉浸式自主發現 — 委託系統是角色 interact view 功能選單的一個選項，玩家進入任意委託型角色（農女/魔女/守衛）→ 看到功能選單 → 自由點選

### 0.2 進入路徑（自然發現）

```
玩家進入農女/魔女/守衛 interact view
  → 對話播放完
    → 功能選單顯示 [耕種/煉製/探索周圍][送禮][CG 圖鑑][對話]
      → 玩家自由選擇是否點擊專業功能按鈕
        → 點擊即進入委託流程
```

委託系統的完整流程、格子工作台、倒數機制等設計規格維持不變，見下方各章節。

### 0.3 自主發現原則（設計哲學）

> **Sprint 6 擴張（2026-04-20）**：此設計原則與委託自主發現哲學一致，作為全遊戲「自主發現」機制的框架宣告。
> **決策 6-13 更新（2026-04-20）**：守衛取劍改為「首次進入 interact view 自動觸發對白」，不再是有卡關代價的自主發現教學節點。守衛案例已從教學節點表中移除。

本遊戲的核心哲學是**沉浸式自主發現**——系統功能不強制教學，玩家透過探索介面自然發現可用行動。

**自主發現原則定義**：遊戲不強制亮紅點引導、玩家依自身節奏決定何時使用各功能。

**適用系統**：

| 系統 | 位置 | 進入路徑 | 無紅點後果 |
|---|---|---|---|
| **委託系統發現** | 任意委託型角色解鎖後 | 進入角色 → 看功能選單 → 點主功能按鈕 | 玩家不用委託仍可玩（無卡關） |
| **送禮系統發現** | 任意角色解鎖後 | 進入角色 → 點 [送禮] | 好感度累積變慢但不卡關 |
| **CG 圖鑑發現** | 任意角色解鎖後 | 進入角色 → 點 [CG 圖鑑] | 不影響流程 |

**設計哲學**：
- 委託系統採**完全自主發現**（無卡關代價）——玩家可能玩很久都不碰委託，這是設計容忍的
- 「無強制教學」對應 Sprint 6「委託自主發現」核心決策
- 守衛取劍採**自動觸發對白**（決策 6-13），確保零卡關風險，不作為教學節點

**影響範圍**：此設計原則影響 `character-unlock-system.md` § 3.5（守衛取劍後續流程）、`main-quest-system.md` T2 完成條件、`exploration-system.md` § 0.2 探索關閉提示。

## 一、委託型角色定位

委託系統的接口是三位**委託型角色**：

| 角色 | 功能按鈕名稱 | 專業 | 產出類別 | 需交付 |
|---|---|---|---|---|
| **農女** | `[耕種]` | 種植 | 作物（食物線主力） | 種子（玩家需先取得） |
| **魔女** | `[煉製]` | 煉金 | 藥水／寶石／金工品（禮物線主力） | 素材（玩家需先取得） |
| **守衛** | `[探索周圍]` | 採集／護衛 | 基礎物資（種子、素材） | 空手可委託（或工具類，TBD） |

> **村長夫人**不屬於委託型角色。她是劇情型 + 功能型角色，負責倉庫、主線任務承載。詳見 `characters.md` 與 `storage-expansion.md`。

### 1.1 專業化原則

委託採**專業化模型**（製作人 2026-04-17 第三輪決策 1）：每位角色只接受其專業對應的物品作為輸入。

- 農女只處理種子類（玉米種、番茄種、胡蘿蔔種等）
- 魔女只處理煉金素材類（藥草、礦石、魔法水晶等）
- 守衛採「空手委託」為主，可能擴充為工具類委託（TBD）

設計意圖：強化角色差異化與敘事一致性（「農女不會煉製，魔女不會耕種」）。

### 1.2 守衛與玩家親自探索的分化

| 行為 | 範圍 | 風險 | 產出定位 |
|---|---|---|---|
| **玩家親自探索**（Hub `探索` 入口） | 深入森林、未知地圖 | 高（戰鬥、撤離張力） | 稀有素材、特殊物資、深探獎勵 |
| **守衛探索周圍**（守衛 `[探索周圍]` 委託） | 村莊周邊 | 低（穩定產出） | 基礎物資（種子、初階素材） |

兩者在**地理範圍與風險回報**上自然區分，解決 B2（玩家探索 vs 委託價值競爭）問題。

### 1.3 前期可用委託角色（漸進解鎖）

委託角色採**漸進式解鎖**，並非開局全部可用。委託解鎖時機與角色 Hub 按鈕解鎖時機一致，玩家解鎖角色後即可在 interact view 自主發現並使用委託：

| 階段 | 可用的委託角色 | 可執行的委託 | 備註 |
|---|---|---|---|
| 階段 1（開局） | 無委託角色（僅村長夫人） | 僅倉庫 / 對話 / 送禮，無委託 | — |
| 階段 2（選擇 1 解鎖） | 農女 或 魔女（擇一） | 耕種 或 煉製（擇一） | 自主發現，無主線強制 |
| 階段 3~4（節點 1 → 選擇 2） | 農女 + 魔女 | 耕種 + 煉製 | 自主發現，無主線強制 |
| 階段 5（節點 2） | 農女 + 魔女（不變） | 耕種 + 煉製 | — |
| 階段 6（探索開放） | 農女 + 魔女（不變） | 耕種 + 煉製 | — |
| **階段 7（守衛歸來後）** | 農女 + 魔女 + **守衛** | 耕種 + 煉製 + **探索周圍** | 守衛 Hub 按鈕解鎖後可自主使用 |

**關鍵點**：
- **守衛的「探索周圍」委託在階段 7（首次探索完成後）才解鎖**，對應守衛加入村莊的時機
- 此時玩家剛體驗「深入森林的風險」，自然能理解「探索周圍」作為**基礎物資穩定產出**的價值（與深探形成分化）
- 任何階段的委託均為**玩家自主決定**是否使用，前期主線不要求玩家必須完成委託

### 1.4 角色解鎖時的初始資源配置

> **2026-04-20 設計轉向（決策 6）**：「完全不發初始物資」，農女/魔女解鎖時的種子/藥草發放已移除。物資完全依賴探索取得。守衛贈劍（新 T2 完成時）保留不受影響。

| 解鎖角色 | 解鎖的委託 | 初始資源 |
|---|---|---|
| 農女 | 耕種 | （無）— 委託所需物資由玩家探索取得 |
| 魔女 | 煉製 | （無）— 委託所需物資由玩家探索取得 |
| 守衛 | 探索周圍 | **劍**（基本戰鬥裝備，**首次進入守衛 interact view 時自動觸發取劍對白後取得**，非守衛歸來事件自動發放） |

> **注意（決策 6-13 更新）**：守衛的初始資源（劍）在玩家首次進入守衛 interact view 時自動觸發取劍對白，對白結束後劍自動入手——非守衛歸來事件自動發放，亦非角色解鎖瞬間發放。守衛的「探索周圍」為空手可委託（無輸入物品要求），玩家不需預先備料即可開始。詳見 `character-unlock-system.md` v1.8 § 3.5。

## 二、委託完整流程

### 2.1 統一委託流程（圖解）

```
【Hub 畫面 VillageHubView】
  點擊角色按鈕（農女／魔女／守衛）
    ↓
【CharacterInteractionView — 第一狀態】
  左側立繪 + 右側對話區（打字機）+ 功能選單
  對話播放完 → 功能選單顯示 [耕種／煉製／探索周圍][送禮][CG 圖鑑][對話]
    ↓ 點擊專業功能按鈕（如 [耕種]）
【CharacterInteractionView — 提煉介面狀態】
  立繪保留，右側對話框與功能選單消失
  顯示：格子式工作台（多格子，詳見三章）
    - 每格可放入背包物品（僅該角色可處理的物品可選）
    - 有效輸入 → 顯示結果 + 「開始」按鈕亮起
    - 無輸入 → 不顯示結果 + 按鈕灰
    ↓ 按「開始」
【CharacterInteractionView — 第二狀態（交付確認）】
  提煉介面關閉，右側對話框回來
  角色對話：「交給我吧」（敘事包裝）
  功能選單變為「工作中 mm:ss」倒數顯示
    ↓ 玩家離開角色畫面，可做別的事（探索、其他角色、其他委託）
【倒數進行中（現實時間）】
  玩家進入該角色時，interact view 顯示工作中狀態（詳見五章）
    ↓ 倒數完成
【Hub 畫面角色頭像亮紅點】
  紅點層級：L1 委託完成層（詳見四章）
    ↓ 玩家點擊角色按鈕進入
【CharacterInteractionView — 完成狀態】
  角色主動播放完成回報對話（取代日常招呼）
  顯示產出內容 + 領取按鈕
    ↓ 玩家領取
【產出進入背包（先背包後倉庫的邏輯維持）】
  回到第一狀態，可再度委託
```

### 2.2 流程階段定義

| 階段 | 狀態 | 玩家可執行操作 |
|---|---|---|
| **Idle（待委託）** | 角色無進行中委託 | 點擊專業按鈕進入提煉介面 |
| **Setup（設置中）** | 玩家在提煉介面擺放物品 | 放置／取回物品、按「開始」 |
| **InProgress（工作中）** | 倒數中 | 僅能退出角色畫面；工作中狀態顯示倒數（詳見五章） |
| **Completed（可領取）** | 倒數完成、紅點亮起 | 進入角色播放完成回報對話、領取產出 |

### 2.3 敘事包裝「交給我吧」

每次委託開始時，角色會播放一句「交給我吧」類的敘事台詞作為包裝。具體台詞內容：

- **各角色獨立文本**（依角色性格差異化）
- 農女範例：「放著吧，我會好好照顧它們。」（樂觀型）
- 魔女範例：「嗯……這些材料讓我看看。你去忙你的。」（散漫型）
- 守衛範例：「交給我，等我回來。」（乾脆型）

> 具體對話數量與文本分配由 `character-content-template.md` 與 `dialogue-writing-sop.md` 規範。本文件僅規定觸發時機。

## 三、格子式工作台（提煉介面）

### 3.1 介面結構

提煉介面以「**格子式工作台**」呈現：

```
┌─────────────────────────────────────────────┐
│                                             │
│   ┌─────────┐       ┌─────────┐             │
│   │         │       │ 工作台格子       │    │
│   │  立繪   │       │  ┌──┬──┬──┐    │    │
│   │ (保留)  │       │  │  │  │  │    │    │
│   │         │       │  ├──┼──┼──┤    │    │
│   │         │       │  │  │  │  │    │    │
│   └─────────┘       │  └──┴──┴──┘    │    │
│                     │                │    │
│                     │  [結果預覽]    │    │
│                     │  [開始按鈕]    │    │
│                     └─────────────────┘    │
└─────────────────────────────────────────────┘
```

- **立繪保留**：左側角色立繪持續顯示，維持角色在場感
- **工作台格子**：多格子並行結構（FA-A 決策，詳見 3.3）
- **結果預覽區**：放入物品後顯示預計產出
- **開始按鈕**：有效輸入時亮起、無輸入時灰

### 3.2 單物品配方結構（JB）

每個格子執行的是**單物品配方**（製作人 2026-04-17 第十輪 JB 決策）：

```
配方 = 單一輸入物品 → 單一產出物品 + 倒數時間
```

| 角色 | 配方範例 |
|---|---|
| 農女 | 玉米種 → 玉米、番茄種 → 番茄、胡蘿蔔種 → 胡蘿蔔 |
| 魔女 | 藥草 → 藥水、水晶 → 寶石、礦石 → 金工品 |
| 守衛 | （空手）→ 基礎物資 A／B／C（依工具或地點，TBD） |

**資料結構**（與既有 `SeedData` 同構）：

```
CommissionRecipeData {
    InputItemId           // 單一輸入物品 ID
    OutputItemId          // 單一產出物品 ID
    DurationSeconds       // 倒數時間（現實時間）
}
```

既有 `FarmManager` 的 `SeedData` 已是此結構的原型，可直接沿用擴展：

```
SeedData {
    SeedItemId            // = InputItemId
    HarvestItemId         // = OutputItemId
    GrowthDurationSeconds // = DurationSeconds
}
```

### 3.3 多格子並行結構（FA-A）

所有三個委託型設施採**多格子制**（製作人 2026-04-17 第七輪 FA 決策）：

| 設施 | 既有結構 | 新增/延伸 |
|---|---|---|
| 農女（農場） | `FarmPlot[]` 多格 | ✅ 既是多格，UI 擴展為格子式工作台 |
| 魔女（煉製） | 無 | 🆕 `CraftPot[]` 多格煉金爐（新增 CraftManager） |
| 守衛（探索周圍） | 無 | 🆕 `GatheringSlot[]` 多個派遣位（新增 GatheringManager） |

**通用 Commission Facility Pattern**：

- 多個 Slot
- 每 Slot 可獨立放入單一物品配方（輸入）
- 系統判斷有效性 → 顯示結果
- 每 Slot 獨立倒數（狀態：空閒／工作中／可領取）

每個設施的具體工作位數量（固定／可升級／好感度解鎖）為 TBD（HA-1）。

### 3.4 兩階段合一邏輯（GB + JB）

配合 GB「不可處理物品不能放入」與 JB「單物品配方」決策，介面邏輯合為一階段：

| 判斷 | 行為 |
|---|---|
| **物品層級** | 該物品是否存在於該 NPC 的配方表 `Inputs` 中？<br>- 是 → 可放入（顯示為可選）<br>- 否 → 不能放入（灰／隱藏） |
| **組合層級** | 單物品已可放入 → 必然有對應輸出<br>- 顯示結果 + 「開始」按鈕亮 |

**虛擬碼**：

```
function CanPlaceItem(npcId, itemId):
    return itemId in NPC_Recipes[npcId].Inputs

function GetOutput(npcId, itemId):
    recipe = NPC_Recipes[npcId].GetRecipe(itemId)
    return recipe.OutputItemId, recipe.DurationSeconds
```

### 3.5 配方發現的輔助機制

根據 GB 決策，**不可處理物品不能放入**本身已解決「玩家亂試錯」的挫折感。進階配方的發現機制（透過對話、好感度、任務解鎖）為 TBD（詳見待決事項）。

## 四、時間機制與紅點觸發

### 4.1 現實時間等待

委託等待採**現實時間**（製作人 2026-04-17 第三輪決策 2）：

| 時間長度 | 委託類型範例 | 適合的等待期玩家行為 |
|---|---|---|
| 10 秒 | 快速作物、簡單採集 | 在村內切換找其他角色 |
| 30–60 秒 | 煉製一瓶藥水、採集基礎素材 | 逛其他設施、整理背包 |
| 2–3 分鐘 | 大量種植、複雜煉金 | 一次短探索 |
| 5–10 分鐘 | 大規模委託（最長 ≈ 一次探索時間） | 一次完整探索 |

**設計原則**：最短 10 秒，最長 ≈ 一次探索時間。上限遵循製作人 4/16「升級等待 ≈ 探索時間」節奏決策。

各物品配方的具體倒數時間存放於 `SeedData`／`CraftRecipeData`／`GatheringTaskData` 等資料表（TBD）。

### 4.2 倒數的持續性

現實時間倒數即使遊戲未開啟也持續計時（與 `FarmManager` 既有邏輯一致）。

### 4.3 紅點觸發四層

委託系統的紅點機制與全局紅點系統整合，分四層觸發：

| 層級 | 觸發條件 | Hub 頭像紅點 | 進入角色後行為 | 按鈕紅點位置 |
|---|---|---|---|---|
| **L1 委託完成層** | 任一委託倒數完成 | 顯示 | **自動播放**完成回報對話（取代招呼語）+ 領取介面 | — |
| **L2 角色發問層** | 固定時間倒數（60s placeholder）到時 | 顯示 | **下沉至「對話」按鈕紅點**，玩家點擊才觸發（不自動播放） | `[對話]` 按鈕 |
| **L3 任務層** | 新任務可承接 | 不顯示 | — | `[任務]` 按鈕（直接在 Interact view 顯示）。前期主線 T0~T2 不依賴委託完成，L3 紅點在委託角色身上的出現時機由實作階段決定 |
| **L4 事件層** | 主線事件觸發 | 顯示 | **自動播放**主線事件對話（取代招呼語） | — |

**L2 紅點下沉機制**（2026-04-18 新增）：
- L2 紅點在 Hub 角色頭像顯示，提醒玩家進入
- 玩家進入角色 Interact view 後，Hub 頭像紅點消失，紅點**下沉至「對話」按鈕**
- 玩家必須主動點擊「對話」按鈕才觸發角色發問

**L2 紅點累積上限與保留規則**（規則層，2026-04-18 新增）：
- 紅點上限：**1**（亮後不再累積，未被觸發前永久保留）
- 工作中：倒數暫停，對話按鈕不可見，紅點延後至工作完成領取後顯示
- 工作完成領取後，若倒數已到 → 紅點**立即亮起**

**紅點優先序**（2026-04-18 補充）：
- 若多層同時觸發，Hub 頭像顯示優先序：L1 → L4 → L2
- 進入角色後播放優先序：L1 → L4 → （結束後） → L2 按鈕紅點 / L3 按鈕紅點保留
- **L1/L4 自動播放結束後，若 L2 或 L3 紅點仍存在 → 保留顯示**，不因 L1/L4 完成而清除
- 若只剩 L2/L3：進入角色後無自動播放，玩家按對應按鈕紅點觸發

詳細互動流程見 `character-interaction.md` §5 與 §8。

## 五、工作中狀態（GC）

製作人 2026-04-17 第八輪 GC 決策：工作中**仍可進入角色畫面**，但選單變為倒數顯示、對話改為描述角色認真工作。

### 5.1 Interact View 工作中狀態佈局

```
┌───────────────────────────────────────┐
│                                       │
│   ┌─────────┐      ┌──────────────┐   │
│   │         │      │ 對話：        │   │
│   │  立繪   │      │（認真工作中   │   │
│   │         │      │  的描述）     │   │
│   │         │      │              │   │
│   └─────────┘      ├──────────────┤   │
│                    │ 工作中...    │   │
│                    │    03:11     │   │
│                    └──────────────┘   │
└───────────────────────────────────────┘
```

### 5.2 工作中狀態設計細節

| 元素 | 規格 |
|---|---|
| **立繪** | 使用既有立繪差分的「認真工作」表情（若無則使用「一般」） |
| **對話** | 描述角色正在認真工作的樣子（例：「她專心地混合著藥草，沒有注意到你進來」） |
| **倒數顯示** | `mm:ss` 格式（例：「工作中... 03:11」） |
| **功能選單** | 不顯示四項選單按鈕，以倒數顯示取代 |
| **可執行操作** | 僅能退出 interact view 返回 Hub |
| **情感魅力點** | 玩家看到角色專注工作的樣子——本身就是一種親密感的來源（製作人認定的設計精神） |

## 七、好感度與委託的互動

### 7.1 委託本身不直接加好感度

委託流程（交付、領取）本身**不直接累積好感度**。

好感度仍透過既有三途徑累積：
- **對話問答**（玩家點擊 `[對話]`，對應 G5「玩家發問 40 題」）
- **送食物**（送 `type:食品` 物品）
- **送禮物**（送 `type:禮物` 物品）

### 7.2 好感度對委託的影響

製作人 B3 選項未定。可能的影響維度（TBD）：

- 產出量影響（好感高 → 產出多）
- 等待時間影響（好感高 → 時間短）
- 解鎖新委託類型（好感門檻 → 新配方）
- 額外 flavor 對話（好感高 → 完成回報更親密）

## 八、委託與其他系統的接口

### 8.1 與背包／倉庫系統

- **交付物資**：從背包取出（背包不足時可從倉庫補充？TBD）
- **領取產出**：進入背包（背包滿 → 溢出倉庫 → 都滿 TBD）
- **與 `storage-expansion.md` 的接口**：委託產出是倉庫主要消耗來源之一

### 8.2 與探索系統

- 探索帶回的物資是委託的**主要輸入來源**（玩家探索 → 取得種子/素材 → 交給角色委託）
- 探索與守衛「探索周圍」的差異化定位見 1.2 節

### 8.3 與主線任務系統

- **前期主線不依賴委託完成**（2026-04-20 設計轉向）：T0/T1/T2 的觸發與完成條件不含任何委託要求，見 `main-quest-system.md` v2.0 §8.1
- 中後期任務仍可能引導特定委託（機制深度引導），但不作為解鎖前置
- 主線任務完成可能獎勵新配方解鎖（詳見 `main-quest-system.md` v2.0）

### 8.4 與好感度／送禮／CG 系統

- 委託不直接觸發好感度，但委託流程中的「完成回報對話」與角色互動計入玩家對角色的接觸頻率
- CG 解鎖仍由好感度門檻驅動（既有 `CGUnlockManager` 邏輯）

### 8.5 既有程式碼接口

| 既有模組 | 在委託系統中的角色 |
|---|---|
| `FarmManager` + `SeedData` + `FarmPlot` | 農女委託的原型，可直接沿用 |
| `StorageManager` + `BackpackManager` | 交付（從背包取出）與領取（先背包後倉庫）的基礎邏輯 |
| `AffinityManager` | 好感度對委託的影響（TBD） |
| `VillageEvents`（`FarmPlotPlantedEvent`、`FarmPlotHarvestedEvent`） | 可對應紅點觸發的事件來源 |
| `ItemTypeResolver` | 判斷物品是否為該 NPC 可處理類別 |

新增模組（TBD，實作階段處理）：

- `CraftManager`（魔女煉製，類 FarmManager）
- `GatheringManager`（守衛探索周圍，類 FarmManager）
- `CommissionManager` 或各 Manager 獨立，視架構決定
