# Dev Log 2026-04-17-6

## 概述

試玩第四輪回饋：NPC 到訪時 `[MvpEntryPoint] interactionViewPrefab / container 未設定。` log warning，互動 View 無法開啟。

## 根因

`MvpCharacterInteractionView.prefab` 與 `MvpCharacterListItem.prefab` 初版由 ui-ux-designer 手寫時，使用非標準 fileID 格式：
- 19 位數字（`&1000000000000001`、`&2000000000000013`）
- **含非法字母後綴**（`&2000000000000026a`、`&2000000000000002b`）

Unity PrefabImporter 只接受純數字 fileID。字母後綴直接讓 Prefab 無法被識別為有效 asset，Scene 中 `_interactionViewPrefab` 與 `_characterListItemPrefab` 解析為 null。

## 修復

派 ui-ux-designer 以 Unity 標準格式（`&100000`、`&100004`、`&110002` 等六位純數字）重建兩個 Prefab：

### MvpCharacterInteractionView.prefab
- Root GO 100000 / MonoBehaviour 100004（Scene 引用目標）
- 9 個 SerializeField 全部連線到子 GO：
  - `_characterNameLabel` 110002, `_affinityLabel` 120002, `_dialogueText` 130002
  - `_dialogueClickArea` 140003, `_dialogueButton` 150003
  - `_dispatchButton` 160003, `_dispatchButtonLabel` 160012
  - `_returnButton` 170003
  - `_dispatchPlaceholderLabel` 180002（初始 inactive）

### MvpCharacterListItem.prefab
- Root GO 100000 / MvpCharacterListItemView 100004（Scene 引用目標）
- 4 個 SerializeField：
  - `_nameLabel` 110002, `_affinityLabel` 120002
  - `_itemButton` 100003（根部 Button 覆蓋整個 item）
  - `_redDot` 130003（RedDotView MonoBehaviour，含內部 `_dotImage` 130002）

### Scene 引用
`MvpMain.unity`：
- `_interactionViewPrefab: {fileID: 2000000000000013}` → `{fileID: 100004, guid: b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7, type: 3}`
- `_characterListItemPrefab: {fileID: 1000000000000014}` → `{fileID: 100004, guid: a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6, type: 3}`

meta guid 不動（guid 是 asset 識別，fileID 是 asset 內部 object 識別）。

## 驗證

- `dotnet build Game.csproj` → 0 errors, 0 warnings
- Prefab grep `^--- !u!\d+ &\d+[a-z]` → 無結果（所有 fileID 純數字）
- Prefab SerializeField 全部有值（非 `{fileID: 0}`）

## 後續應用

手寫 Unity YAML 時的 fileID 格式鐵則：
1. **純數字**（不含字母、小數點）
2. 建議六位（`100000` 起），與 Unity 自動生成格式一致
3. 同一 Prefab 內唯一，但不同 Prefab 間可重複（guid 區分）
