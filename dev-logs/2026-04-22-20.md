## 完成內容

### VillageEntryPointInstallers.cs — JsonFx 靜態 API 修復
- 移除 `JsonReader reader = new JsonReader();`（建構子不存在）
- 所有 `reader.Deserialize<T[]>(...)` 改為 `JsonReader.Deserialize<T[]>(...)`（靜態方法）
- 新增 `using ProjectDR.Village.Exploration.Combat;`（供 `CombatConfigData` 解析使用）

### ExplorationEntryPoint.cs — 配套修復
- `monsterConfig.GetType(sp.TypeId)` → `monsterConfig.GetMonsterType(sp.TypeId)`（`GetType` 與 `System.Object.GetType` 衝突，Wave 2.5 重構後 API 已改名）
- `CombatConfig.Load(json)` → `new CombatConfig(dto)`（`Load()` 靜態方法已廢棄，改用建構子）
- `MonsterConfig.Load(json)` → `new MonsterConfig(entries)`（同上）
- `LoadCombatConfig()` / `LoadMonsterConfig()` 改為先 `JsonReader.Deserialize<T[]>(json)` 再傳入建構子

### Game.Tests.asmdef — 加入 JsonFx 引用
- `precompiledReferences` 新增 `"JsonFx.Json.dll"`
- 原因：`overrideReferences: true` 時 Plugins 資料夾 dll 不會自動引用，需手動列出

- 保留其他已能驗證空白配置的斷言（`GetPersonalityPreference` 回 null、`GetAffinityDelta` 回 0）

**MonsterStateTests.cs / CombatManagerTests.cs / MonsterManagerTests.cs**

**CombatConfigDataTests.cs（完整重寫）**
- 廢棄 `CombatConfigJson` / `PlayerStatsJson` / `SwordConfigJson` / `MonsterTypeJson` / `ColorJson`
