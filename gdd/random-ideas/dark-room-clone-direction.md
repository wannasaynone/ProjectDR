## A Dark Room 核心設計整理（工作室理解，需製作人確認）

> 本節為工作室對 Dark Room 的理解，若有偏差請製作人指正。

**A Dark Room**（2013，Doublespeak Games）是一款文字 incremental / extraction 探索遊戲。

### 核心特徵

| 面向 | Dark Room 做法 |
|---|---|
| **視覺風格** | 極簡文字 UI（紅/白/黑），幾乎無圖像 |
| **漸進揭露** | 開局只有一個按鈕「stoke fire」，隨遊戲推進解鎖更多機制 |
| **敘事驚喜** | 劇情透過文字事件揭露（最終揭露玩家是外星人） |
| **資源系統** | wood / fur / meat / cloth / leather 等多層次資源，製作鏈複雜 |
| **村民系統** | 陌生人自動出現，分配為 gatherer/hunter/trapper/charcutier/tanner，**無個性、無對話** |
| **探索系統** | 網格地圖，帶補給出發，補給耗盡必須返回營地，否則死亡 |
| **戰鬥系統** | 隨機遭遇 + 簡單回合制 |
| **地圖揭露** | 探索過的區域持久記錄，不會重置 |
| **階段解鎖** | 按時間/事件觸發 — 小徑→森林→村莊→城市→太空艙 |

### Dark Room 的核心魅力

1. **「一直在解鎖新東西」的驚喜感** — 每隔幾分鐘就有新機制出現
2. **文字驅動的想像力** — 沒有畫面反而讓玩家自行腦補
3. **資源管理張力** — extraction 撤離的壓力（帶多少水能走多遠）
4. **節奏控制精準** — 每個機制在玩家剛好厭倦前一個機制時引入
5. **世界揭露** — 從「在森林裡烤火」到「我原來是外星人」的震撼

## 與 ProjectDR 既有設計的相容性分析

### ✅ 高度相容的元素

- **Extraction 探索** — ProjectDR 已採用，與 Dark Room 基礎結構一致
- **資源管理** — ProjectDR 已有食物線/禮物線/素材 pool
- **營地/村莊管理** — 無論 A/B 概念都有
- **角色/工人分配** — 可對應 ProjectDR 的「NPC 綁設施」架構
- **漸進式解鎖** — 製作人 4/15 決策中的「NPC 漸進式解鎖」與此理念一致

### ⚠️ 需要調整才能相容

- **地圖揭露持久 vs 撤離類遊戲重置** — ProjectDR 目前的地圖是可重入的撤離類，與 Dark Room 的持久地圖不完全相同（但製作人 4/16 決策「撤離類包含熟悉地圖要素」方向接近）
- **階段解鎖** — Dark Room 是時間/事件自動觸發，ProjectDR 是好感度/探索進度觸發

### ❌ 根本衝突的元素

- **極簡文字 UI vs 成人遊戲視覺需求**
  - Dark Room 幾乎無圖像（只有文字和極簡地圖）
  - 成人遊戲核心賣點之一是 CG（ProjectDR 預算 420 張 CG）
  - 「完全照抄」會失去成人遊戲的視覺競爭力
- **村民無個性 vs ProjectDR 戀愛養成核心**
  - Dark Room 村民只是工人數字，無對話無個性
  - ProjectDR 核心樂趣是角色養成（無論 A/B 都是）
  - 若照抄就失去 ProjectDR 的核心樂趣（#3 角色/HCG）
- **世界揭露敘事 vs 角色養成敘事**
  - Dark Room 驚喜在於「世界真相」（外星人設定）
  - ProjectDR 現有敘事聚焦角色關係推進
  - 兩種敘事結構難以並存

## 「完全照抄」的定義問題（需製作人釐清）

「照抄 Dark Room」可以指向不同的範圍，不同範圍對 ProjectDR 的衝擊完全不同：

### 範圍 1：結構照抄（最激進）
- 照抄整個遊戲結構 = 文字 UI + 無角色個性 + 世界揭露敘事
- **衝擊**：失去成人遊戲核心、失去 ProjectDR 既有的角色設計
- **現有資產保留率**：極低（探索系統可留，其他幾乎全棄）
- **可行性**：需重新定位為「成人向 Dark Room」，但失去原 Dark Room 極簡美學

### 範圍 2：玩法機制照抄（中度）
- 照抄：extraction + 資源管理 + 工人分配 + 階段解鎖 + 戰鬥
- **保留**：角色深度養成、視覺 CG、戀愛結構
- **衝擊**：工人分配與 NPC 綁設施概念需整合
- **現有資產保留率**：高（ProjectDR 本來就接近這個結構）
- **可行性**：實際上 ProjectDR 現有設計已接近此範圍

### 範圍 3：節奏與驚喜感照抄（最輕）
- 照抄：漸進揭露節奏、機制解鎖的驚喜感、極簡起點+複雜終點
- **保留**：所有角色設計、視覺、戀愛結構
- **衝擊**：需重新編排遊戲節奏（開局要極簡）
- **現有資產保留率**：極高（只調整節奏編排）
- **可行性**：高，實質上是在 ProjectDR 上學習 Dark Room 的節奏哲學

### 範圍 4：氛圍與世界觀照抄（特殊向）
- 照抄：神秘氛圍、世界觀逐步揭露
- **保留**：所有玩法、部分角色設計
- **衝擊**：世界觀需從「封閉村莊+森林」改為帶神秘揭露結構的異世界
- **現有資產保留率**：中等（玩法保留、世界觀重構）
- **可行性**：中等，需設計新的世界揭露敘事線

## 工作室反對意見與風險（供製作人參考）

### 反對意見 1：成人遊戲失去視覺是自殺
- Steam 成人市場玩家預設要有 CG/立繪
- Dark Room 的極簡美學建立在「非成人」前提下（玩家不期待看到女角）
- 「成人向 Dark Room」若真的無 CG，首屏吸引力會極低（Steam 成人玩家看不到女角直接劃走）

### 反對意見 2：沒有先例的藍海可能是賽道不成立
- 第二輪報告已指出「成人 + extraction」幾乎無成熟對標
- 缺乏對標可能是因為：(a) 機會未被挖掘；(b) 結構本身不吸引成人玩家
- 在投入大量資源前必須先做小規模原型驗證

### 反對意見 3：捨棄現有資產成本極高
- 已完成：G5 單角色內容量規格、G7 對話撰寫 SOP、四位角色 character-spec、探索系統 199 測試、好感度/CG/送禮系統
- 若採範圍 1（結構照抄），這些資產幾乎全棄
- 近期完成的 Sprint 3（GDD 完善）成果會失效

### 反對意見 4：模糊的「完全照抄」會導致設計失焦
- Dark Room 與 ProjectDR 在多個面向上根本衝突
- 「完全照抄」在邏輯上難以同時滿足「Dark Room 式極簡」+「成人遊戲視覺」
- 必須先釐清要抄哪部分、要留哪部分，否則落入 Subverse 式失焦

### 潛在機會
- 若採範圍 3（節奏照抄），ProjectDR 可獲得 Dark Room 式的「漸進驚喜感」，強化玩家留存
- 若採範圍 2（機制照抄），ProjectDR 現有設計本來就接近，無額外成本
- 若採範圍 4（氛圍照抄），可為概念 B 的異空間設定提供敘事錨點

| 路徑 | 關係 |

| 路徑 3（A/B 混合 α）| 接近 — 都是 A 為主 + 輕度擴展，但此方向有派遣機制 α 沒有 |

## 🔴 需製作人釐清的核心設計問題

### Q1：角色總數 — 維持 4 位還是擴增？

- **選項 A**：維持 4 位（守衛/村長夫人/農女/魔女）— 現有 character-spec 全用，漸進招募變成解鎖順序的機制化
- **選項 B**：擴增到 5~8 位 — 需新增角色設計，但可為派遣系統提供更多策略深度
- **選項 C**：維持 4 位深度 + 少量路人（派遣用輔助角色）— 類似 A/B 混合 α，但派遣化

### Q2：村莊 vs 據點（製作人用了「村莊(據點?)」的問號）

- **選項 A**：維持「村莊」— 保留 A 的情感歸屬感與既有設定（被森林圍困的封閉村莊）
- **選項 B**：改為「據點」— 更像 Dark Room 的 camp，功能性更強
- **選項 C**：保留「村莊」名稱但重定位 — 村莊仍是情感歸屬，但加入 Dark Room 式的「從空到滿」成長感

### Q3：開局村莊狀態

- **選項 A**：完全空村 — 最忠於 Dark Room，但與「封閉村莊」世界觀有落差（村莊為什麼空了？）
- **選項 B**：僅有玩家 1 人 — 玩家是倖存者，需要出去找到其他角色
- **選項 C**：僅有 1 位 NPC（例：村長夫人留守）— 有敘事起點，其他 3 位散落在森林
- **選項 D**：4 位都在但未互動 — 探索後「找回」她們，解鎖互動（最保守改動）

### Q4：派遣機制的結構

- **派遣後角色是否離村？**
  - 選項 A：離村（派遣期間無法互動）— 權衡強，玩家要在「用她工作」與「陪她互動」間選擇
  - 選項 B：不離村（類似 Dark Room 分配，只是狀態改變）— 情感連結不中斷，但策略張力弱
- **派遣任務類型**：
  - 資源收集（採集素材、食材）
  - 狩獵（獲得肉、毛皮）
  - 情報收集（解鎖新地圖區域）
  - 生產/製作（後台產出物品）
  - 戰鬥支援（跟玩家一起探索？）
- **派遣時間**：
  - 即時（派出去後角色消失一段現實時間）
  - 按探索次數（派出後下一次探索自動回來）
  - 按事件觸發

### Q5：派遣 vs 現有 NPC 互動選單的整合

現有 base-management.md 架構是每個 NPC 固定選單：
```
[設施功能] [升級] [對話] [送禮]
```
派遣如何加入？
- **選項 A**：新增第五個選項 [派遣] — 與現有架構並存
- **選項 B**：取代 [設施功能] — 設施由派遣任務產出替代
- **選項 C**：[設施功能] 與 [派遣] 並存但互斥 — 同一時間只能選其一

### Q6：探索中的角色遭遇機制

新方向需要「角色逐漸加入」，代表探索中要有機會遇到未招募的角色。這如何設計？

## ⚠️ 工作室需提醒製作人的幾個點

### 1. 工作量急劇增加
- 原 A 方案 4 位全量：420 CG / 126,000 字 / 122~170 天
- 若新方向 5~8 位全走 G5 深度：525~840 CG / 157,500~252,000 字 / **152~340 天（5~11 個月）**
- 外加探索女角色 HCG + 派遣系統 + UX 重構的追加工作量
- **這會顯著延長 VS 前置與正式製作時程**

### 2. 設施概念回歸（4/16 的「移除設施」被部分推翻）
- 4/16 決策：「設施概念移除，將角色降格為設施操作員」
- 現在：設施回來了（派遣位），但**角色與設施解耦**
  - 過去：農女綁農田（職能綁角）
  - 現在：任何角色都可派去農田工作（勞動力通用）
- 這是設計迭代，需要正式記錄為「部分推翻 4/16 設施移除決策」

### 3. 「探索女角色不能互動」的玩家體驗風險
- 玩家觸發 HCG 後，帶回的女角色**只剩勞動力屬性**，無對話/好感度/後續發展
- 風險：玩家情感上可能覺得「她變成了單純的工具人」，產生違和感
- 建議考慮：要不要留一句「退場對話」或「簡單 flavor text」緩解違和
- 參考類似案例：Mad Island 的俘虜機制（成為居民後仍有基本對話）

### 4. 「5 分鐘內第一位加入」的設計挑戰
- 5 分鐘是強烈的鉤子，但需要考慮：
  - 玩家 5 分鐘內該做什麼（教學？初步探索？）
  - 第一位加入的觸發方式（自動出現？探索事件？）
  - 第一位是誰（既有 4 位的某位？還是新角色？）
- Dark Room 的參考：約 30 秒後第一個陌生人出現，但是 incremental 節奏快

### 5. 既有 4 位角色的去留
- 若維持 character-spec 4 位全用 + 新增 1~4 位 = 總 5~8 位，工作量平衡
- 若重新設計全部 5~8 位，既有 character-spec 作廢（Sprint 3 成果部分失效）
- 製作人應明確表態
