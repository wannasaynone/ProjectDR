## 2. CharacterIntroLines.txt 資料格式修復

**問題**：Play mode 第一次啟動時發現 `FormatException: narrator is not a valid value for Int32`，位於 `VillageEntryPointInstallers.cs:121`。

**根因**：`CharacterIntroLines.txt` 第 42 筆起（intro_witch 第三行起）欄位錯位：
- `intro_id` 欄位存入了 sequence 數字（3, 4, 5...）
- `sequence` 欄位存入了 speaker 字串（"narrator", "Witch"...）
- `speaker` 欄位存入了 text 文字
- `text` 欄位存入了 line_type 值

影響範圍：id 42-52（intro_witch 第 3-13 行）+ id 53-69（intro_guard 全 17 行）共 28 筆。

**修復方式**：從舊 `character-intro-config.txt` 提取正確原始資料，重建 CharacterIntroLines.txt 完整 69 筆，恢復正確欄位對齊。

**驗後**：Play mode 重啟後 0 error。

## 8. 遇到的問題與解決方式

### 問題 1：CharacterIntroLines.txt 欄位錯位
- **現象**：Play mode `FormatException: narrator is not a valid value for Int32`（`sequence` int 欄位被填入字串）
- **根因**：Sheets 匯出時 intro_witch / intro_guard 部分行（第 42 行起）發生欄位位移
- **解法**：從舊 `character-intro-config.txt` 提取正確原始資料，重建完整 69 筆並正確映射欄位
- **預防**：本問題是 Sheets 資料問題，未來重新從 Sheets Convert 時應驗證所有資料的欄位型別正確性

### 問題 2：manage_components 不持久化
- **現象**：set_property 後 GUID 未寫入 .unity 檔，舊 GUID 仍在場景 YAML
- **根因**：manage_components 只改 Runtime 記憶體，需要 manage_scene save 才持久化
- **解法**：set_property 後立即呼叫 manage_scene save
