## 實作要點

### 架構決策：保留動態生成，改為按解鎖狀態篩選

原本 `VillageHubView` 在每次 `OnShow` 時清除並重新生成所有角色按鈕。重構後：

- **角色按鈕**：保留動態生成方式，但加入解鎖狀態篩選——只生成 `CharacterUnlockManager.IsUnlocked(characterId) == true` 的角色
- **未解鎖角色**：完全不生成按鈕（不是灰色、不是透明），符合 GDD 「漸進揭露哲學」與 UGUI P9（不可用 Color.alpha=0 偽隱藏）
- **探索入口**：改為 Prefab 中的靜態按鈕（`BtnExploration`），預設 `SetActive(false)`，解鎖後才 `SetActive(true)`

### 事件驅動更新

```
CharacterUnlockedEvent → RefreshCharacterButtons()（重新生成按鈕）
ExplorationFeatureUnlockedEvent → SetExplorationButtonVisible(true)
```

事件委派（`Action<T>`）在建構子中快取，用於 `OnDestroy` 時正確取消訂閱（防止記憶體洩漏）。

### 存檔進入的正確性

`OnShow()` 時呼叫 `ApplyExplorationButtonState()`，查詢 `CharacterUnlockManager.IsExplorationFeatureUnlocked` 的當前狀態。確保從存檔進入時，探索按鈕的顯示狀態與解鎖狀態一致（不依賴事件驅動）。同樣，`RefreshCharacterButtons()` 在每次 OnShow 時呼叫，確保角色按鈕狀態正確。

### VillageEntryPoint 組裝

`InitializeViewDependencies()` 新增第三個參數，將已建立的 `_characterUnlockManager` 注入給 `VillageHubView`：

```csharp
hubView.Initialize(_navigationManager, _characters.AsReadOnly(), _characterUnlockManager);
```
