# 基地經營系統

> **版本**: v2.0

## 系統目的

基地經營系統承接核心循環（`core-definition.md` v3.2）的第三至第五個節點：

```
帶著物資安全撤離
  → 物資投入基地經營（委託交付／擴建物資／送禮）  ← 本系統負責
    → 基地成長解鎖新功能／新探索區域              ← 本系統負責（搭配主線任務）
      → 經營活動自然推進角色關係                   ← 本系統觸發
```

本系統是核心樂趣 **#2 經營成就感**的主要載體——玩家探索帶回的素材，在此透過**委託循環**轉化為食物/禮物/基礎物資，透過**擴建循環**擴充倉庫容量。每一次委託完成、每一次擴建都是探索成果的具體證明，同時開啟下一個目標，驅動玩家繼續探索。

自 2026-04-17 委託制定案後，基地經營系統從「設施升級」轉型為「NPC 導向的委託 + 擴建」架構。設施仍存在於委託的底層邏輯（農田、煉金爐、採集路線），但**玩家介面永遠是角色**。

## 一、系統定位與設計原則

### 1.1 NPC 導向架構（修訂版）

基地經營並非一塊獨立的「建設區域」，而是整合在每個 NPC 的互動選單中。每個 NPC **承載自己的功能**，玩家透過與 NPC 互動來使用對應功能：

| 角色 | 類型 | 承載功能 |
|---|---|---|
| **村長夫人** | 劇情型 + 功能型 | 倉庫 + 擴建 + 主線任務 |
| **農女** | 委託型 | 耕種委託（種子 → 作物） |
| **魔女** | 委託型 | 煉製委託（素材 → 藥水/寶石/金工品） |
| **守衛** | 委託型 | 探索周圍委託（空手 → 基礎物資） |

**設計意圖**：
- 強化「每個 NPC 是一個功能入口」的清晰感
- 讓委託行為與角色關係感自然連結——你在拜託這位角色幫你完成工作
- 避免產生「基地管理」與「角色互動」在空間上的割裂感
- 符合 4/16「設施存在但玩家介面永遠是角色」的修訂決策

### 1.2 設計原則

| 原則 | 說明 |
|------|------|
| **節奏自然** | 委託與擴建的等待時間遵循現實時間（10 秒 ~ 一次探索時間），讓玩家在「委託→探索→回來領取」之間形成自然節奏 |
| **物資共用** | 委託交付、擴建建材、送禮物品共用同一個物品 pool；不另設分類，減少物資管理複雜度 |
| **解鎖漸進** | 採主線任務引導（`main-quest-system.md`）+ 好感度門檻組合；確保玩家先深入經營再解鎖進階內容 |
| **可見成長** | 委託產出可見、倉庫容量可見、主線任務進度可見，皆符合核心樂趣驗證標準 2（探索成果轉化為可見改變） |

## 二、NPC 互動選單結構（統一四項）

自 2026-04-17 第九輪定案後，所有四位角色的功能選單統一為四項：

| 角色 | 功能選單 |
|---|---|
| 村長夫人 | `[倉庫][任務][對話][送禮]` |
| 農女 | `[耕種][任務][對話][送禮]` |
| 魔女 | `[煉製][任務][對話][送禮]` |
| 守衛 | `[探索周圍][任務][對話][送禮]` |

### 2.1 各按鈕的說明

| 按鈕 | 說明 |
|---|---|
| **[倉庫]**（村長夫人） | 進入倉庫管理介面（overlay 顯示），內含擴建入口 |
| **[耕種]** / **[煉製]** / **[探索周圍]** | 進入該 NPC 的委託提煉介面（格子式工作台），詳見 `commission-system.md` |
| **[任務]** | 承接主線任務；新任務可承接時亮紅點（GA-B 決策：所有角色皆有），詳見 `main-quest-system.md` |
| **[對話]** | 玩家主動發問（G5 玩家發問 40 題），詳見 `character-interaction.md` 五章 |
| **[送禮]** | 送禮介面（好感度三途徑之一：送食物 / 送禮物） |

**注意**：所有四個選單項目在任何好感度階段下均可見，但部分功能（如特殊配方、高階任務）在解鎖條件未達成時顯示為鎖定狀態並提示解鎖條件。

## 三、委託循環（委託型三位的核心經營）

委託循環是委託型角色承載的主要經營活動。完整流程詳見 `commission-system.md`，此處僅概述接口：

```
玩家探索帶回物資（種子／素材）
  → 進入角色 interact view
    → 點擊專業按鈕（耕種／煉製／探索周圍）
      → 格子式工作台 + 單物品配方
        → 按「開始」→ 「交給我吧」→ 倒數（現實時間 10 秒 ~ 一次探索時間）
          → 紅點亮起（L1 委託完成層）
            → 玩家領取產出
              → 物資進入下一循環（送禮 / 擴建 / 倉庫留存）
```

### 3.1 委託本身不直接加好感度

好感度仍透過既有三途徑累積（對話問答、送食物、送禮物）。委託流程中的「完成回報對話」與角色互動計入玩家對角色的情感累積，但不直接觸發好感度數值增加（B3 對好感度的影響機制 TBD）。

## 四、擴建循環（村長夫人專屬）

擴建循環是村長夫人承載的主要經營活動。完整流程詳見 `storage-expansion.md`，此處僅概述接口：

```
倉庫空間緊張 或 主線任務引導
  → 進入村長夫人 → [倉庫] → 擴建入口
    → 確認物資（先背包後倉庫扣除）+ 確認擴建
      → 村長夫人播放擴建敘事對話
        → 倒數（現實時間，TBD 具體時長）
          → 紅點亮起 → 玩家進入
            → 完成對話 → 倉庫 +50 格
```

### 4.1 規格

- 初始容量：**100 格**
- 每次擴建增量：**+50 格**
- 倉庫結構：**格子制 + 物品堆疊（與背包規格相同）**
- 擴建所需物資：TBD（後續 session 設定）
- 擴建等待時間：TBD（遵循現實時間等待原則）

## 六、與現有系統的接口

### 6.1 倉庫 / 背包系統

- **倉庫**：格子制 + 堆疊（與背包同），初始 100 格、每次擴建 +50
- **委託交付**：從背包取出（背包不足時可否從倉庫補充？TBD）
- **委託領取**：進入背包（先背包後倉庫）；都滿時依溢出策略處理（TBD）
- **扣除順序統一**：背包優先、倉庫次之（與擴建、送禮、委託交付一致）

### 6.2 好感度系統

- 委託與擴建**不直接增加好感度**
- 好感度仍透過三途徑累積（對話問答、送食物、送禮物）
- 好感度門檻可解鎖：新配方（TBD）、CG 里程碑（既有）、特殊任務（主線引導）

### 6.3 探索系統

- 探索帶回的物資是**委託的主要輸入**（種子/素材）
- 主線任務可引導玩家解鎖新地圖區域（探索獎勵）
- 玩家親自探索（深處/稀有）與守衛探索周圍（村莊周邊/基礎）自然分化

### 6.4 主線任務系統

- 主線任務 = 機制引導工具（Dark Room 式漸進揭露）
- 任務按鈕位於所有角色 interact view 的功能選單中（GA-B）
- 任務完成可能解鎖新配方、新區域、新 CG（詳見 `main-quest-system.md`）

### 6.5 通用製作系統（取代說明）

**既有 `village-economy.md` 五章的「通用製作系統」概念已整合為委託系統**——製作行為不再以「製作畫面」形式存在，而是透過委託型角色的格子式工作台執行：
- 食材 → 食品的「料理製作」 → 改為農女的耕種委託（種子 → 食材），食品層的製作是否保留、由誰承載（TBD）
- 素材 → 禮物的「煉金製作」 → 改為魔女的煉製委託
- 探索/採集 → 改為守衛的探索周圍委託

### 6.6 NPC 解鎖順序

NPC 按固定順序解鎖：**村長夫人 → 農女 → 魔女 → 守衛**（`core-definition.md` 決策 #10）。在新架構下，解鎖流程可能整合主線任務的引導序列（詳見 `main-quest-system.md`）。

## 七、重大變更（對照 v1.0）

### 7.1 已推翻的 v1.0 設計

| v1.0 設計 | v2.0 處置 | 理由 |
|---|---|---|
| NPC 選單項目為 `[設施功能][升級][對話][送禮]` | ❌ **取消[升級]** | 升級概念由委託本身的配方擴充、主線任務解鎖取代（B4 決策：升級概念處置 TBD，v2.0 暫無） |
| 設施有獨立等級與升級介面 | ❌ **取消** | 設施存在於委託底層邏輯，但不對玩家暴露為可升級物件 |
| 村莊等級聚合指標（方案 A/B/C） | ❌ **暫不採用** | 主線任務引導已提供進度感，村莊等級可能不必要（TBD） |
| 解鎖混合制（設施等級 × 村莊等級 × 好感度） | ⚠️ **簡化** | 以主線任務引導為主軸，好感度門檻為輔 |
| 設施升級物資消耗 | ❌ **取消** | 改為擴建物資（村長夫人）+ 委託交付物資（三委託型角色） |
| 設施升級等待時間 | ❌ **取消** | 改為委託等待 + 擴建等待，遵循現實時間（10 秒 ~ 一次探索時間） |

### 7.2 已保留的 v1.0 設計

- NPC 導向架構（功能整合在 NPC 互動選單中）
- 物資共用 pool 原則
- 節奏自然原則（等待時間 ≈ 探索時間）
- 可見成長原則
