# 主線任務系統（Main Quest System）

> **版本**: v1.1

## 系統目的

主線任務系統承接 `core-definition.md` v3.1 的核心樂趣驗證標準 4「是否創造讓玩家想看下一步的驅動力？」，同時是本作對 A Dark Room **漸進揭露哲學**的落地實現。

本系統的定位明確為**機制引導（Mechanics Onboarding）**（製作人 2026-04-17 第十輪 JA 決策）：

- **任務不是**史詩劇情
- **任務不是**角色支線
- **任務不是**日常隨機內容
- **任務 = 機制引導** — 每個任務是一個機制／功能的敘事包裝，其存在是為了告訴玩家「你可以做 X」

對應 Dark Room 的漸進揭露節奏（stoke fire → gather wood → check traps → hunt → 村民到來 → 戰鬥 → 太空艙），ProjectDR 的主線任務序列是「認識農女 → 首次委託 → 倉庫擴建 → 深探新區域 → 新配方解鎖 → ……」。

**任務完成 ≠ 破關**（製作人 2026-04-17 第五輪 DB-b 決策，與 4/16「無終局」一致）：主線任務只推進機制揭露與系統解鎖，玩家在完成所有既有主線任務後仍可無限遊玩。

## 一、任務的本質：機制引導

### 1.1 任務設計範式

每個主線任務 = 一個**機制／功能的敘事包裝**。

| 任務序列範例 | 對應引導的系統／機制 | 觸發條件（範例） |
|---|---|---|
| T1. 「先去找農女看看」 | 初次委託（耕種） | 開局自動 |
| T2. 「倉庫快滿了，或許該擴建」 | 初次擴建倉庫 | 倉庫使用率達 70% |
| T3. 「聽說森林深處有新東西」 | 初次深探／新地圖區域解鎖 | T1+T2 完成 |
| T4. 「守衛最近提到……」 | 初次採集委託（探索周圍） | T3 完成 + 守衛好感度 |
| T5. 「她似乎想收到什麼」 | 初次送禮 | 某角色好感度門檻 |
| T6~Tn | 各系統深度引導（委託升級、新區域、CG 解鎖） | 事件序列 + 對應條件 |

> 上表為**中後期任務範式**（前 3 位與守衛皆已解鎖之後），**非設計決定**。實際任務數量、序列、引導內容由製作人後續分批決定。

> 前期（角色解鎖階段）的 T0~T4 主線任務序列已定案，見下方 1.4 章節。

### 1.2 設計優勢

1. **工作量可控**：任務數量不固定，可視製作節奏分批新增
2. **新手教學自然化**：不需獨立教學關卡，主線就是引導
3. **無內容壓力**：任務 ≠ 劇情主菜，可以是短序列對話
4. **增量擴充友善**：新機制加入遊戲時，只需加一段主線任務作為引導
5. **Dark Room 節奏哲學落地**：漸進揭露 = 主線任務序列

### 1.3 任務與其他系統的敘事分工

| 系統 | 敘事功能 |
|---|---|
| **主線任務** | 機制引導（告訴玩家可以做什麼、下一步在哪） |
| **角色對話（G5 招呼/發問/玩家發問）** | 角色個性展示、戀愛養成情感累積 |
| **里程碑對話 / 約會 / H 劇情** | 好感度進展的情感回報 |
| **委託的「交給我吧」與完成對話** | 角色在日常勞動中的形象 |

主線任務不與角色支線、日常任務競爭——它是**系統層的引導**，角色層的敘事由 G5 的對話系統承載。

### 1.4 前期主線任務序列（T0~T4）

自 2026-04-17 第十二~十八輪定案後，**前期主線任務同時扮演「角色解鎖觸發器」的角色**——每個節點的觸發條件 = 主線任務完成（`character-unlock-system.md` QC-D 決策）。

前期主線任務序列如下：

| 任務 | 引導的機制 | 同時觸發的節點 | 完成條件 |
|---|---|---|---|
| **T0 開局** | 認識村長夫人、世界觀介紹 | **節點 0**（開局自動） | 開局自動完成（對話結束） |
| **T1 「先去找村裡的其他人吧」** | 首次進入 VN 式選項解鎖流程 | **選擇 1 解鎖**（節點 0 劇情末尾觸發選項） | 玩家在村長夫人對話中選擇農女或魔女、觀看登場 CG + 短劇情 |
| **T2 「幫 XX 做點事吧」** | 首次委託（耕種 or 煉製） | **節點 1**（T2 完成後 L4 紅點） | 完成 1 次選擇 1 角色的委託（TBD，可能含其他條件） |
| **T3 「再幫 YY 做點事吧」** | 第二次委託 + 節點 2 | **節點 2**（T3 完成後 L4 紅點）+ **探索開放**（節點 2 劇情推進） | 完成 1 次選擇 2 角色的委託（TBD） |
| **T4 「出去看看外面」** | 首次探索 + 守衛歸來事件 | **守衛解鎖**（首次探索自動觸發） | 首次點擊 Hub 探索入口、完成守衛歸來劇情 |

> **重要關聯**：T0~T4 的完成狀態直接驅動 `character-unlock-system.md` 的節點觸發。任務層與解鎖層是**同一條時間軸的雙重視角**——從任務角度看是「機制引導序列」，從解鎖角度看是「角色加入節奏」。

### 1.5 前期任務與後期任務的關係

- **前期**（T0~T4）：任務 = 解鎖觸發器 + 機制引導（雙重角色）
- **後期**（T5 起）：任務 = 純粹機制引導（解鎖已全數完成，任務不再承擔解鎖責任）

前期任務的具體對話字數、完成條件細節、獎勵配置為 TBD，由 `character-unlock-system.md` 實作階段定案。

## 二、任務結構

### 2.1 資料結構（概念）

```
MainQuestData {
    QuestId              // 任務唯一識別
    Title                // 任務標題
    Description          // 任務描述（展示在承接提示視窗）
    DialogueLines        // 任務承接時的對話（在 interact view 播放）
    TriggerConditions    // 觸發條件清單（詳見 2.2）
    CompletionConditions // 完成條件清單（詳見 2.3）
    OwnerCharacterId     // 任務擁有者（哪個角色的 [任務] 按鈕亮紅點）
    Rewards              // 獎勵（詳見 2.4）
    UnlocksNextQuests    // 觸發後續哪些任務（事件序列）
}
```

### 2.2 觸發方式：事件序列 + 觸發條件

製作人 2026-04-17 第五輪 DB-a 決策：任務採**事件序列 + 觸發條件**結構。

觸發條件可為下列維度的組合（支援 AND／OR 組合，TBD）：

| 觸發條件 | 範例 |
|---|---|
| **前置任務完成** | 「T3 完成後解鎖 T4」 |
| **村莊等級** | 探索/委託進度累計達門檻（若保留此概念，TBD） |
| **好感度** | 特定角色好感度達門檻 |
| **物資收集** | 特定稀有物資出現於倉庫／背包 |
| **時間／遊戲日數** | 開局 X 分鐘後或 X 日後自動 |
| **系統行為** | 倉庫使用率達 70%、首次使用某功能等 |

事件序列意味著：任務之間有**明確的順序與依賴關係**，而非全部並行開放。

### 2.3 完成條件

完成條件（TBD，依每任務性質而定）：

| 條件類型 | 範例 |
|---|---|
| 對話完成 | 承接後進入指定角色對話序列結束 |
| 委託行為 | 完成一次耕種／煉製／探索周圍（與委託系統整合） |
| 探索行為 | 抵達特定地圖／格子／擊敗特定魔物 |
| 經營行為 | 倉庫擴建一次／送禮一次／好感度達 X 階段 |
| 混合 | 上述多項達成 |

### 2.4 獎勵類型

| 獎勵類型 | 說明 |
|---|---|
| **新機制解鎖** | 新委託類型、新配方、新區域 |
| **新地圖／探索區域解鎖** | 與探索系統整合 |
| **角色 CG／劇情** | 與既有 CG 解鎖系統整合 |
| **倉庫容量擴展加成** | 與擴建系統整合（可能作為擴建的替代路徑之一，TBD） |
| **世界觀揭露** | Dark Room 式的文字揭露，但限於戀愛養成所以不走外星人式翻盤敘事 |
| **物資獎勵** | 任務完成給予一次性物資 |

## 三、任務按鈕與承接流程

### 3.1 任務按鈕位置（GA-B）

製作人 2026-04-17 第八輪 GA-B 決策：**所有角色都有任務按鈕**，位於其功能選單中：

| 角色 | 功能選單 |
|---|---|
| 村長夫人 | `[倉庫][任務][對話][送禮]` |
| 農女 | `[耕種][任務][對話][送禮]` |
| 魔女 | `[煉製][任務][對話][送禮]` |
| 守衛 | `[探索周圍][任務][對話][送禮]` |

### 3.2 任務按鈕的紅點顯示

| 狀態 | 紅點顯示 |
|---|---|
| 該角色有可承接任務 | `[任務]` 按鈕顯示紅點 |
| 該角色無可承接任務 | 無紅點 |
| 玩家已承接 | 紅點消失 |
| 完成後需回報（TBD，詳見 JA-3） | 可能再次亮紅點 |

紅點與 Hub 畫面角色頭像的紅點之間的層級關係，見 `commission-system.md` 第四章「紅點觸發四層」：任務層（L3）紅點顯示在 `[任務]` 按鈕，事件層（L4）可能顯示在 Hub 角色頭像。

### 3.3 承接流程

製作人 2026-04-17 第七輪 FB-3 決策：點擊任務按鈕 → interact view 有對話 → 跳出承接提示視窗。

```
玩家進入角色 → 對話播放 → 功能選單 [XX][任務][對話][送禮]
  ↓ 看到 [任務] 按鈕紅點
玩家點擊 [任務]
  ↓
Interact View 中播放對話（任務描述對話）
  角色說出任務背景與引導
    範例：村長夫人「聽說森林深處有新東西……要不要去看看？」
  ↓ 對話結束
跳出「承接任務」提示視窗
  - 任務標題
  - 任務描述
  - 完成條件（如適用）
  - [接受] / [拒絕] 按鈕
  ↓ 玩家選 [接受]
任務進入「進行中」狀態
  ↓ 返回 interact view（或 Hub）
```

### 3.4 拒絕的處理（TBD）

玩家拒絕承接後的行為：
- 選項 A：紅點保留，下次進入可再承接
- 選項 B：紅點消失，任務延後觸發
- 選項 C：任務標記為「已拒絕」，不可再承接（極少見，TBD）

## 六、主線完成 ≠ 破關

### 6.1 設計決策（DB-b）

製作人 2026-04-17 第五輪 DB-b 決策：**主線任務全部完成不代表遊戲結束**。

- 與 `core-definition.md` 決策 #5「無終局」一致
- 完成所有主線任務後，玩家仍可：
  - 繼續進行各角色的戀愛養成（好感度、送禮、CG）
  - 繼續執行各項委託與探索
  - 享受已解鎖的全部系統內容
- 主線任務的終點是「所有機制都已被引導認識」，而非「遊戲結束」

### 6.2 主線任務的盡頭敘事

當最後一段主線任務完成時，適合的敘事呈現（TBD）：

- 短段世界觀揭露（輕度 Dark Room 式）
- 村莊狀態的某種「新常態」描繪
- 不走翻盤敘事（不是太空艙離開，不是外星人揭密）
- 保持戀愛養成基調

## 八、與其他系統的接口

### 8.1 與委託系統

- 部分任務要求「完成一次委託」作為機制引導
- 任務獎勵可能解鎖新配方（擴展各 Manager 的配方表）
- 詳見 `commission-system.md` 第四章「紅點觸發」

### 8.2 與倉庫擴建系統

- 任務可能引導玩家第一次擴建倉庫（T2 範例）
- 任務獎勵可能提供額外容量（TBD）
- 詳見 `storage-expansion.md`

### 8.3 與探索系統

- 任務可能引導玩家進入特定地圖／深探特定格子
- 任務獎勵可能解鎖新地圖區域
- 與 `exploration-system.md` 既有 55 條規則整合

### 8.4 與好感度／CG 系統

- 任務獎勵可能包含 CG 解鎖（既有 `CGUnlockManager`）
- 任務觸發條件可能依賴好感度門檻

### 8.5 與紅點系統

- 任務紅點屬於紅點系統的 L3（任務按鈕紅點）與 L4（事件層主線對話）層
- 詳見 `commission-system.md` 第四章「紅點觸發四層」

### 8.6 新增程式模組（TBD，實作階段處理）

- `MainQuestManager`（任務狀態管理、觸發條件判定、完成判定）
- `MainQuestData`（任務資料結構）
- 任務承接提示視窗 UI（`QuestAcceptView` 或類似）
- 任務清單 View（若採全域任務面板方案）
