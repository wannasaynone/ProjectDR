# 角色 response 打字機 Click-to-Skip 修正

> **日期**：2026-04-18
> **階段**：Internal Test（實機回饋修正）
> **類型**：Bug Fix（UI 互動）

## 背景

製作人實機測試回報：

> 現在角色頁面中，選擇回答之後，角色回覆的過程中點擊畫面無法讓角色立刻完成句子

流程：
1. 進入角色互動畫面 → 播招呼 → 選單出現
2. 點「對話」（L2 紅點亮時走 a 路徑 inline）
3. prompt 打字機播完 → 右下顯示 4 個選項
4. 點選項 → `OnCharacterQuestionOptionSelected` → `ClearChoiceContainer` → `PlayDialogue(response)`
5. **response 打字機播放中**，玩家預期點擊畫面應立即顯示完整 response → 實際點擊無反應

招呼語與感謝對話（`StartDialoguePlayback` / `OnGiftSuccess`）則無此問題。

## 根因分析

`CharacterInteractionView.prefab` root 的 sibling 順序（後者 render 在上，raycast 順位也在上）：

| index | 物件 | 活化條件 |
|------|------|---------|
| 0 | PortraitPanel | 常駐 |
| 1 | DialoguePanel（含 DialogueText / DialogueClickArea） | 常駐 |
| 2 | MenuContainer | 選單狀態 |
| 3 | OverlayContainer | 有 overlay 時 |
| 4 | ReturnButton | 非 Forced 狀態 |
| 5 | **FullScreenDialogueArea** | PlayDialogue 期間 |
| 6 | ChoiceContainer | 有選項時 |
| 7 | CommissionCountdownText | CommissionInProgress |
| 8 | CommissionClaimButton | CommissionReady |

`FullScreenDialogueArea` 為 sibling 5，**位於 ChoiceContainer(6) 之下**。當 Normal / 響應播放時，6/7/8 皆應為 inactive，raycast 應穿透到 FullScreenDialogueArea；**但任何殘留 active 狀態**（例如 ChoiceContainer 剛 `Destroy` 當幀尚未真正移除、sibling 順序被 VerticalLayoutGroup 或 ContentSizeFitter 等 UI 佈局操作意外調整）都會攔截 raycast 導致 `OnDialogueClicked` 不會觸發 `typewriter.Skip`。

## 修正

### 1. `PrepareDialogueUI` 啟用 FullScreenDialogueArea 時強制 `SetAsLastSibling`

```csharp
if (_fullScreenDialogueArea != null)
{
    _fullScreenDialogueArea.gameObject.SetActive(true);
    _fullScreenDialogueArea.transform.SetAsLastSibling();
}
```

確保 FullScreenDialogueArea 永遠位於 root 最末端，raycast 順位最高，任何其他 sibling 無法攔截。

### 2. `PlayDialogue` 改為呼叫 `PrepareDialogueUI` 走統一流程

原本 `PlayDialogue` 內聯 `CloseOverlay` / `RefreshAffinityDisplay` / `SetMenuVisible(false)` / `_fullScreenDialogueArea.SetActive(true)` / Subscribe 等步驟，少了 `PrepareDialogueUI` 具備的：

- `SetCommissionCountdownVisible(false)`
- `SetCommissionClaimButtonVisible(false)`
- `ClearChoiceContainer()`（caller 若未自行清，容易有殘留 choice buttons）
- `SetAsLastSibling`（本次修正新增）

改為直接呼叫 `PrepareDialogueUI` 一次取齊。`StartDialoguePlayback` / 外部驅動模式 `OnShow` 已經走此路徑，現在 `PlayDialogue` 也對齊。

## 影響範圍

`PlayDialogue` 的三個呼叫來源均受益：

1. `OnGiftSuccess`（送禮感謝對話）
2. `OnCharacterQuestionOptionSelected`（a 路徑 inline response）
3. `PlayerQuestionsView` / `CharacterQuestionsView` responseAction（b 路徑、舊 a 路徑）

## 追加：prompt 階段也要允許 Click-to-Skip

### 補充回報

> 顯示問題時也無法點擊畫面立刻完成打字機

`StartCharacterQuestionInline` 原本刻意禁用 `_fullScreenDialogueArea`（避免玩家跳過 prompt），但製作人要求改為允許 Skip，與一般 VN 對話行為一致。

### 追加修正

1. **`StartCharacterQuestionInline`**：prompt 階段改為 `SetActive(true) + SetAsLastSibling`，讓點擊觸發 `typewriter.Skip` → `OnComplete` → `OnCharacterQuestionPromptComplete` → 自動進入選項階段。
2. **`ShowCharacterQuestionChoices`**：進場時先 `_fullScreenDialogueArea.SetActive(false)`（若保留 active 會因 SetAsLastSibling 的最高 raycast 順位遮住選項 button），並對 `_choiceContainer` 呼叫 `SetAsLastSibling` 確保選項 button 在最上層可被點擊。

### 更新後完整流程

| 階段 | FullScreenDialogueArea | ChoiceContainer | 玩家互動 |
|------|----------------------|-----------------|---------|
| prompt 播放 | active + LastSibling | inactive | 點擊 skip prompt，skip 後自動進選項 |
| 選項顯示 | inactive | active + LastSibling | 點擊選項 button |
| response 播放 | active + LastSibling | inactive | 點擊 skip response |
| 對話結束 | inactive（OnDialogueCompleted） | inactive | 選單恢復 |
