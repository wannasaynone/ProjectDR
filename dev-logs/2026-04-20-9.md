## 根因分析（Root Cause）

**問題**：守衛歸來事件完成後，守衛 Hub 按鈕未出現。

**事件鏈斷點**：

1. 玩家點擊探索按鈕 → `ExplorationEntryManager.Depart()`
2. 攔截器 `TryIntercept()` → `GuardReturnEventController.TriggerEvent()`
3. `_cgPlayer.PlayIntroCG(Guard, OnCGComplete)` → CG 播放（`CharacterIntroCGView` 展示 17 條 intro_lines）
4. CG 完成後 `OnCGComplete()` 呼叫 `_dialogueManager.StartDialogue(guard_return_lines)`（31 條）
5. **斷點**：此時 Hub view 在顯示，`CharacterInteractionView` 未顯示。整個 codebase 中只有 `CharacterInteractionView.OnDialogueClicked()` 會呼叫 `DialogueManager.Advance()`，但它此時不在場景中。
6. `Advance()` 永遠不被呼叫 → `DialogueCompletedEvent` 不發布
7. `GuardReturnEventController.OnDialogueCompleted()` 不觸發 → `CompleteEvent()` 不執行
8. `GuardReturnEventCompletedEvent` 不發布 → `CharacterUnlockManager.OnGuardReturnCompleted()` 不觸發
9. `ForceUnlock(Guard)` 不執行 → `CharacterUnlockedEvent` 不發布 → `VillageHubView` 不重建按鈕

**設計問題**：`guard_return_lines`（31 條）與 `character-intro-config.json` 的 `intro_guard`（17 條）台詞高度重疊，均描述同一段守衛歸來場景。CG 播放期間 `CharacterIntroCGView` 已完整展示了守衛歸來的劇情，CG 後的 `DialogueManager` 路徑是多餘且無法執行的。

**測試未覆蓋原因**：`GuardReturnIntegrationTest.TriggerGuardReturnAndComplete()` 使用 `while (_dialogueManager.IsActive && _dialogueManager.Advance()) { }` 手動推進對話，繞過了生產環境中不存在此機制的問題。測試通過但生產失敗。

## 修復方式（F7）

**最小改動**：`GuardReturnEventController.OnCGComplete()` 直接呼叫 `CompleteEvent()`，移除 `DialogueManager.StartDialogue()` 路徑。

同時移除所有 `_dialogueManager`、`_dialoguePhase`、`_onDialogueCompleted`、`_config` 死碼欄位，建構函式改為只需要 `ICGPlayer cgPlayer`。

## 學到的教訓

**測試輔助方法（Helper）中的手動推進不應繞過生產路徑**。若 Helper 用 `while (dm.IsActive && dm.Advance()) { }` 手動完成對話，等同替測試提供了生產中不存在的「自動推進」機制，讓測試通過但 bug 潛伏。

**預防機制**：F7 回歸測試明確驗證「不手動推進時事件鏈仍完整」，防止未來有人重新引入 DialogueManager 依賴時立即被偵測到。
