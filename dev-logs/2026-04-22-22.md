## 任務 1：D4 — VillageContext.gameDataAccess delegate 接線（TD-2026-011 處置）

### 問題確認

`VillageEntryPointInstallers.cs` 第 147 行傳入 `gameDataAccess: null`，且 `VillageContext.cs` ctor 的 null check 在 Sprint 7 D7 緊急修復時暫時放寬（加 `TODO Sprint 8`）。

### 接線方式

在 `RunInstallers()` 建立 `VillageContext` 之前：

```csharp
GameStaticDataManager gameStaticDataManager = new GameStaticDataManager();
GameDataQuery<IGameData> gameDataAccess = (int id) => gameStaticDataManager.GetGameData<IGameData>(id);
VillageContext ctx = new VillageContext(_villageCanvas, _uiContainer, gameDataAccess);
```

- `GameStaticDataManager` 是 runtime IGameData 查詢的唯一入口（ADR-001）
- delegate 透過 lambda 包裹，隔離靜態依賴，符合 `GameDataQuery<T>` delegate 設計意圖
- 此 `GameStaticDataManager` 實例為空（各 Installer 的資料透過 ctor 直接注入，Sprint 8 Wave 2.5 已完成）；`gameDataAccess` 提供未來 Installer 可透過 `ctx.GameDataAccess(id)` 查詢的標準入口

### VillageContext.cs 恢復 null check

移除 `// TODO Sprint 8` 的 null check 放寬，恢復嚴格驗證：

```csharp
GameDataAccess = gameDataAccess ?? throw new System.ArgumentNullException(nameof(gameDataAccess));
```

同步更新 XML 文件注釋，移除「Sprint 7 期間允許 null」說明。

### 新增 using

`VillageEntryPointInstallers.cs` 加入：
```csharp

```
