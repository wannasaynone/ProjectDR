# Dev Log 2026-04-17-3

## 概述

Sprint 4 試玩回饋後兩項緊急修復：按鈕文字亂碼與搜索按鈕點一次後失效。

## Bug 1：按鈕文字亂碼

**現象**：場景上搜索按鈕文字顯示為 `u641cu7d22u9644u8fd1` 而非「搜索附近」。

**根因**：ui-ux-designer 手寫 Scene YAML 複製 Prefab 內聯內容時，4 處 `m_text` 丟失雙引號與 backslash，變成 literal 字串。既有 Prefab 中 `"\uXXXX"` 格式 Unity 能正確解析，但無引號的 `uXXXX` 會被當作字面字串。

**影響範圍**：Scene 中 4 個動作按鈕文字
- 搜索附近（line 1508）
- 生火（line 1758，初始 inactive 未暴露）
- 延長火堆（line 2008，初始 inactive）
- 蓋小屋（line 2258，初始 inactive）

**修復**：直接 Edit Scene YAML 將 4 處 `m_text:` 改回 `"\uXXXX"` 標準 YAML double-quoted escape 格式。

## Bug 2：搜索按鈕點一次後失效

**現象**：點搜索顯示「翻找木材」但之後按鈕永遠 disabled。

**根因**：`MvpMainView.RefreshButtonAvailability()` 只在以下事件觸發時呼叫：初始化、資源變化、火堆狀態變化、蓋屋開始/完成。**沒有監聽冷卻結束事件**。ActionTimeManager 冷卻結束時沒發事件，按鈕停在 disabled 狀態。

**修復**：於 `MvpMainView` 新增 `Update()`，每幀呼叫 `RefreshButtonAvailability()`。精簡實作，不引入新事件。

```csharp
private void Update()
{
    if (_searchSystem != null || _fireSystem != null || _hutBuildSystem != null)
    {
        RefreshButtonAvailability();
    }
}
```
