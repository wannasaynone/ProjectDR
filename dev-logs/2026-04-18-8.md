# Dev Log 2026-04-18-8 — 開場劇情接線修正（VCW Forced 模式 + 外部驅動對話）

## 問題描述

製作人回報：進入遊戲後沒有觸發對話，且若再次點擊村長夫人會重播同一段開場劇情。

## 根因分析

C2 階段只組裝了 `OpeningSequenceController`，但沒接線 UI 層：

1. `Start()` 呼叫 `TryStartOpeningSequence()` 只做了 `_openingSequenceController.StartOpeningSequence()`
2. 開場序列內部播放 VCW 登場 CG（overlay）後，呼叫 `NodeDialogueController.PlayNode(node_0)` → `DialogueManager.StartDialogue` 啟動 node_0 對話
3. **但 VCW `CharacterInteractionView` 沒有被 push 到畫面上**，node_0 對話在 `DialogueManager` 裡空跑、沒有 UI 顯示
4. 玩家無法推進節點 0 → `NodeDialogueCompletedEvent` 永不觸發 → `OpeningSequenceCompletedEvent` 永不觸發
5. `MarkIntroCGPlayed(VCW)` 永不呼叫 → 下次玩家點村長夫人會因 `HasPlayedIntroCG == false` 而重播首次進入 CG

## 修正方案

依據 `character-unlock-system.md` v1.2 § 1.2 節點 0 UX 流程 與 `character-interaction.md` v2.2 § 1.4 強制模式的原始設計：

1. 開場啟動時先 push VCW `CharacterInteractionView` 進入 **Forced 模式（無返回按鈕）+ 外部驅動對話模式**
2. CG 以全螢幕 overlay 覆蓋 VCW view 播放
3. CG 完成後 `NodeDialogueController` 驅動 `DialogueManager` 發布 `DialogueStartedEvent`，VCW view 收到後觸發打字機播放
4. 玩家透過 VCW view 的打字機 + VN 選項容器推進 node_0
5. node_0 完成 → `OpeningSequenceCompletedEvent` → VCW 切至 Normal（顯示返回按鈕 + 功能選單）

## 追加修正（同日）：CTRL 快轉 + 返回按鈕失效修復 + Prefab raycast 整理

### 1. CTRL 快轉對話

`CharacterInteractionView.cs` 新增 `Update()`：按住 LeftCtrl / RightCtrl 時連續跳過打字機並自動推進下一行，每 0.05s 一次；`IsWaitingForChoice` 時暫停（必須玩家選擇）。

### 2. 返回按鈕失效（開場完成後按下沒反應）

**根因**：`TryStartOpeningSequence` 以 `_stackController.PushView(VCW)` 直接切畫面，**沒經過 `_navigationManager.NavigateTo`**。`VillageNavigationManager._currentArea` 保持 null。玩家按返回 → `ReturnToHub()` 看到 `_currentArea == null` → early return → 不發 `ReturnedToHubEvent` → Hub 永遠切不回來。

**修正**：改為 `_navigationManager.NavigateTo(VillageChiefWife)`。NavigationManager 會發 `NavigatedToAreaEvent`，觸發既有的 `OnNavigatedToArea` → `InitializeCharacterView`（已在 `_initializedViews` → early return 不重複初始化）+ `PushView`。這樣 `_currentArea` 正確設為 VCW，後續 `ReturnToHub` 可正常運作。

### 3. Prefab Raycast 整理（CharacterInteractionView.prefab）

診斷過程中發現根 `Image`（半透明黑底）以及 `PortraitPanel` / `PortraitImage` 的 `Image` 都開著 `raycastTarget`，會攔截子物件 ReturnButton / MenuButton 的點擊。另外 `ReturnButton` 和 `CommissionClaimButton` 的 Image 沒有 sprite（Unity Image 無 sprite 時會影響 raycast 判定）。

修正：
- Root Image：`raycastTarget = false`（純視覺背景不需要接收點擊）
- `PortraitPanel.Image` / `PortraitImage.Image`：`raycastTarget = false`
- `ReturnButton.Image`：指定內建 `UISprite`
- `CommissionClaimButton.Image`：指定內建 `UISprite`

### 4. 修正：開場結束後 VCW 即出現紅點（噪音）

**問題**：使用者回報「開場>選農女>進村莊 就已經看到村長夫人有紅點了」。正確行為應為「只有跟第一位角色對話完之後才出現紅點」。

**根因**：`main-quest-config.json` 所有 T0~T4 的 `owner_character_id` 都是 `VillageChiefWife`。T0 auto 完成後 T1 Available，`RedDotManager.OnMainQuestAvailable` 觸發 VCW L3 NewQuest。但 T1~T4 的實際互動點都不在 VCW（T1=去見女角、T2=委託她、T3=去見另一位、T4=出去探索），VCW 只作為敘事角色（透過 node_0/1/2 引導），並非「需要回來領任務的 NPC」→ L3 紅點是 UX 噪音。

**修正**：將 `main-quest-config.json` 所有 T0~T4 的 `owner_character_id` 改為空字串。`RedDotManager.OnMainQuestAvailable` 在 `ownerId` 為空時早期 return（原本已有防禦），L3 不再觸發。L4 MainQuestEvent（節點 1/2 待播）仍由 `RedDotManager.OnMainQuestCompleted` 對 `QuestIdsTriggersNode1/Node2` 硬寫 VCW 觸發，不受影響。

**實測確認**：
- 開場結束返回 Hub：VCW redDot = None ✓
- 見農女後返回 Hub：VCW redDot = MainQuestEvent (L4) ✓
- 點 VCW 觸發 node_1，完成後解鎖 Witch，紅點清除 ✓
