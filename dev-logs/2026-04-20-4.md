# Dev Log — 2026-04-20-4

> **日期**: 2026-04-20
> **Sprint**: sprint-6-explore-gate-rework
> **涉及項目**: C6（D4 bugfix）

## 背景：D4 實機測試發現 bug

製作人 D4 實機測試流程：
```
開場 CG → 村長夫人節點 0 → 選農女 → 返回 HUD →
農女按鈕出現紅點 → 點農女 → 登場 CG → 對話 → 返回 HUD →
村長夫人 L4 紅點亮（R2 正確運作）→ 玩家點 VCW → 【bug：節點 1 對話不播放】
```

症狀：VCW L4 紅點正確亮起（R2 成功），但玩家進入 VCW 時播一般打招呼對話，節點 1 不觸發，流程卡死。

---

## 根因分析

**架構缺口**：Sprint 6 重構了 T1 語義，但 `GetPendingMainQuestNodeId()` 的觸發條件未隨之更新。這不是 C5 誤刪，而是設計演進後留下的不一致。

### 問題 1：節點 1 觸發條件依賴已廢棄的 T1 語義

舊程式碼（bug）：
```csharp
if (!_playedMainQuestNodes.Contains(NodeDialogueController.NodeIdNode1)
    && _mainQuestManager.IsQuestCompleted("T1"))
{
    return NodeDialogueController.NodeIdNode1;
}
```

Sprint 6 B1 將 T1 語義改為「認識所有人」（需節點 2 對話結束才完成）。但節點 1 的正確觸發時機是「第一位角色 CG 播完後」，此時 T1 尚未完成（T1 在節點 2 播完後才完成）。`IsQuestCompleted("T1")` 永遠回傳 false → **節點 1 永遠不觸發**。

### 問題 2：節點 2 觸發條件依賴已刪除的 T3

舊程式碼（bug）：
```csharp
if (!_playedMainQuestNodes.Contains(NodeDialogueController.NodeIdNode2)
    && _mainQuestManager.IsQuestCompleted("T3"))
{
    return NodeDialogueController.NodeIdNode2;
}
```

Sprint 6 B1 已刪除 T3（原「再去認識另一個人」）。`IsQuestCompleted("T3")` 永遠回傳 false → **節點 2 也永遠不觸發**。

### 為何 L4 紅點正確但對話不播放？

R2（C5）已正確在選擇 1 角色 CG 播完後設定 `SetMainQuestEventFlag(VCW, true)` → L4 紅點確實亮起。但 `GetPendingMainQuestNodeId()` 因 T1 未完成回傳 null → `isVCWPendingNode = false` → 以一般模式（`openingMode: false`）進入 VCW → 播打招呼對話 → 節點 1 不播。

## 修復內容

### `VillageEntryPoint.cs`

#### 新增觸發旗標（宣告）

```csharp
// _node1TriggerReady：第一位角色 CG 播完後設定（節點 1 尚未播過時）
// _node2TriggerReady：第二位角色 CG 播完後設定（節點 1 已播過時）
private bool _node1TriggerReady;
private bool _node2TriggerReady;
```

兩個旗標與 MainQuest T1 語義完全解耦。

#### `InitializeCharacterView` CG 播完 callback（原 R2 位置）

依 `_playedMainQuestNodes` 是否含 node_1 決定設定哪個旗標：
```csharp
if (capturedCharId != CharacterIds.VillageChiefWife && _redDotManager != null)
{
    _redDotManager.SetMainQuestEventFlag(CharacterIds.VillageChiefWife, true);
    if (_playedMainQuestNodes.Contains(NodeDialogueController.NodeIdNode1))
    {
        _node2TriggerReady = true;  // 第二位角色 CG 播完
    }
    else
    {
        _node1TriggerReady = true;  // 第一位角色 CG 播完
    }
}
```

#### `GetPendingMainQuestNodeId()`（核心修復）

觸發條件從 `IsQuestCompleted("T1/T3")` 改為獨立旗標：
```csharp
private string GetPendingMainQuestNodeId()
{
    if (_nodeDialogueController == null) return null;
    if (_dialogueManager != null && _dialogueManager.IsActive) return null;

    if (!_playedMainQuestNodes.Contains(NodeDialogueController.NodeIdNode1)
        && _node1TriggerReady)
    {
        return NodeDialogueController.NodeIdNode1;
    }
    if (!_playedMainQuestNodes.Contains(NodeDialogueController.NodeIdNode2)
        && _node2TriggerReady)
    {
        return NodeDialogueController.NodeIdNode2;
    }
    return null;
}
```

注意：`_mainQuestManager` 的 null 檢查已移除（新邏輯不依賴 `_mainQuestManager`）。

#### `OnMainQuestCompletedForNodeDialogue`（重回保留空方法）

節點觸發旗標由 CG 播完 callback 設定，此方法不再需要業務邏輯。RedDotManager 在 T1 完成時已自動設定 VCW L4（`QuestIdsTriggersNode2 = "T1"`）。

## 節點 2 連帶驗證

節點 2 觸發路徑（修復後完整流程）：
1. 節點 1 → VN 選擇剩下那位（魔女/農女）→ `CharacterUnlockManager` 解鎖
2. Hub 顯示第二位角色按鈕（FirstMeet 紅點）→ 玩家點第二位角色
3. `InitializeCharacterView(Witch/FarmGirl)` → CG 播完 callback：
   - `_playedMainQuestNodes.Contains("node_1") == true` → **`_node2TriggerReady = true`** ✓
   - `SetMainQuestEventFlag(VCW, true)` → VCW L4 亮
4. 返回 Hub → VCW L4 亮 → 玩家點 VCW
5. `GetPendingMainQuestNodeId()` → `_node2TriggerReady && !played(node_2)` → 回傳 `node_2` ✓
6. 播放節點 2 → 村長夫人介紹完所有人 → 對話結束
7. `OnNodeDialogueCompletedForMainQuest(node_2)` → 送 `Node2DialogueComplete` → T1 完成 → 探索開放 ✓

節點 2 路徑正確，修復完整。
