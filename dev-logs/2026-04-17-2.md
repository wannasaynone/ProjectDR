## 技術決策

- **手寫 Unity YAML**：因無 MCP 連線，改用 Write tool 直接撰寫 `.prefab` 與 `.unity` YAML 檔案。整個 MvpMainView.prefab 約 2200 行，一次性 Write 完成避免 heredoc / Python f-string 的 `{}` 衝突問題。
- **Canvas Scaler**：設定 Reference Resolution 1920×1080、`matchWidthOrHeight=0.5`，與既有 VillageMain 場景設定一致。
- **ScrollView 結構**：Content 使用 VerticalLayoutGroup + ContentSizeFitter（Vertical=PreferredSize），Viewport 使用 Mask。Raycast Target 僅在可互動元素（Button）上啟用。
- **按鈕初始狀態**：BtnLightFire、BtnExtendFire、BtnBuildHut、SldHutProgress、ColdStatusRoot 的初始 `m_IsActive: 0`，由 MvpEntryPoint 透過程式碼控制顯示邏輯。

## Scene Prefab 引用 fileID 追加修復（工作室管理層）

驗收時發現 Scene 中兩個 Prefab 引用 fileID 錯誤：
- `_characterListItemPrefab: {fileID: 100004}` → 實際 MvpCharacterListItemView 組件 fileID 為 `1000000000000014`
- `_interactionViewPrefab: {fileID: 100004}` → 實際 MvpCharacterInteractionView 組件 fileID 為 `2000000000000013`

根因：MvpCharacterListItem.prefab 與 MvpCharacterInteractionView.prefab 使用 19 位手寫 fileID（`1000000000000xxx` / `2000000000000xxx`），但 Scene 引用時沿用了 MvpMainView.prefab 的 6 位格式。

已修正 Scene YAML 兩處引用（Edit tool 直接修改）。修正後 Play 時角色清單項 Instantiate 與 InteractionView Instantiate 才能正確解析 Prefab 引用。
