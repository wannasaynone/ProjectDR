## 內容

製作人在新分支完成 Dark Room like 架構測試後，觀察到兩種架構在角色定位上的根本差異，提出**第三路徑**：不是全盤採用 Dark Room like，也不是堅守 DR 原架構，而是把 Dark Room 的兩個具體機制嫁接到原架構上。

### 製作人觀察

| 架構 | 角色定位 | 玩家-角色連結度 |
|---|---|---|
| Dark Room like（新分支） | 機制、工具（派遣勞動力） | 較低（玩家做事不經過角色） |
| DR 原架構（既有 GDD） | 日常生活的一部分（功能入口） | 較高（每個功能都透過角色進入） |

### 製作人假設

若把 Dark Room like 的兩個機制加入 DR 原架構，也許可改善 game feel：

1. **主動發話（紅點提示）** — 角色有話要說時主動觸發，紅點提示
2. **日常設施操作（dark room like）** — 設施可直接操作的 Dark Room 式手感

## 兩種架構的根本差異（工作室分析）

### DR 原架構的結構

```
主畫面（VillageHubView）
  └── 角色按鈕（村長夫人／守衛／魔女／農女）
       └── 角色互動畫面（立繪+對話+功能選單）
            └── 功能（倉庫／探索／藥水／農場／送禮／對話）
```

**核心特徵**：**功能都在角色之下**。玩家要做任何事，都會經過「進入角色 → 看到立繪 → 播放對話 → 選功能」的路徑。

### Dark Room like 架構的結構

```
主畫面（村莊背景）
  ├── 功能按鈕（各設施直接點擊）
  └── 角色清單（點擊查看狀態、派遣）
       └── 設施旁顯示角色 ICON（誰在哪工作）
```

**核心特徵**：**功能與角色並列**。玩家做事不經過角色，角色是獨立的「狀態/派遣對象」。

### 為什麼造成「連結度」差異

**DR 原架構**的角色連結度高，是因為：

- **強迫接觸**：玩家每次要開倉庫都會看到村長夫人立繪、聽到她說話
- **情境帶入**：功能被包裝在「我去找 XX 做 YY 事」的情境中，不是冷冰冰的操作介面
- **日常感**：即使不主動「攻略」，日常使用功能就在累積接觸頻率
- **角色是目的地**：玩家不是「去倉庫」，而是「去找村長夫人」

**Dark Room like 架構**的角色連結度低，是因為：

- **無強迫接觸**：要開倉庫直接點倉庫按鈕，不用經過任何角色
- **工具化**：角色出現在「派去哪工作」的選單中，像員工資料
- **角色在勞動現場的 ICON** 只是視覺化派遣狀態，不產生情感
- **主動互動才見面**：玩家要「刻意決定去找 XX 聊天」才會遇到角色，日常使用功能不會

## 兩種架構的優劣對照

### DR 原架構（角色為入口）

**✅ 優點**
1. 高強度接觸頻率 → 自然情感累積
2. 戀愛養成本質契合（角色是日常主體）
3. 立繪/對話/個性有密集展示機會
4. 符合已完成的資產與 GDD
5. 玩家心智模型：「她在我的生活中」

**❌ 缺點**
1. 角色-功能強綁問題（4/16 事前驗屍識別：「設施升級 = 角色降格為設施操作員」）
2. 效率低（每個功能都要多一層）
3. **角色被動**：都在等玩家來點她，主動性不足
4. 缺少 Dark Room 式的「發現/驚喜」節奏
5. 功能入口單一化 → 經營深度被壓縮（角色既是戀愛對象又是功能鍵）

### Dark Room like 架構（角色為勞動力）

**✅ 優點**
1. 角色-功能解耦 → 角色個性不被職能限制
2. 派遣系統提供經營策略深度
3. 設施旁 ICON 視覺化經營進度
4. 功能存取效率高
5. Dark Room 的漸進揭露節奏（從空到滿）

**❌ 缺點**
1. 玩家做事不經過角色 → 接觸頻率低
2. 角色工具化感 → 戀愛養成本質被削弱
3. 對話/送禮變成「刻意行為」而非「日常碰面」
4. 需重寫既有 UI 架構與部分 GDD
5. 容易變成「經營遊戲 + 附加戀愛劇情」，而非「戀愛養成 + 經營」

## 機制 1：主動發話（紅點提示）的融入可能性

### 機制解析

Dark Room 是 **event-driven**：事件發生時文字通知，玩家被引導去關注。對應到 ProjectDR：

**具體形式**：角色頭像上出現紅點 → 玩家點進去 → 角色主動開啟對話（不是玩家選功能，而是角色主動有話要說）。

### 融入 DR 原架構的契合度

**✅ 高度契合（不衝突，純加成）**

- 不需改變任何既有架構（主畫面仍是角色按鈕）
- 直接解決原架構「角色被動」問題 → 角色變主動
- 紅點 = 獎勵預期訊號 → 增加玩家回訪動機與驚喜感
- 與既有對話系統相容（只需加觸發邏輯層）
- 美術極簡（一個紅點）

### 紅點觸發設計空間

| 觸發條件 | 內容範例 |
|---|---|
| 好感度變化達到門檻 | 新階段對話解鎖 |
| 特定物品進入倉庫 | 「這個想給你看」「這個你會喜歡嗎」 |
| 久未見面（時間差） | 「最近都沒看到你」 |
| 探索中遇到相關事件 | 「剛才森林裡的事…」 |
| 送禮後回應 | 感謝、後續反應 |
| 階段里程碑 | 新劇情觸發 |
| 隨機日常話題 | 天氣、八卦、食譜 |

### 設計風險

1. **紅點過頻** → 變成待辦清單壓力，失去驚喜
2. **紅點過疏** → 沒存在感，功能形同虛設
3. **與「每次進入角色都播對話」規則衝突** → 需改規則為「有新事件時播新對話，否則播日常招呼」
4. **需搭配對話 SOP 新增「主動話題池」** → G7 對話撰寫 SOP 需擴充

### 與既有設計的相容性

- ✅ 與 character-interaction.md v1.1 相容（只需調整規則 2「每次進入都播對話」）
- ✅ 與對話撰寫 SOP v1.0 相容（可在主動話題類別中擴充）
- ✅ 與 G5 單角色內容量規格相容（可納入招呼語或獨立話題池）
- ⚠️ 需新增事件觸發邏輯層（程式開發部工作量：低）

## 機制 2：日常設施操作（dark room like）的融入可能性

### 製作人表述的理解挑戰

「日常設施操作 dark room like」的具體含義需釐清。工作室識別出三種可能理解：

### 理解 A：主畫面快速操作面板

在 Hub 主畫面加一層「不用進角色也能做日常」的快速捷徑。

**實施**：
- 保留角色按鈕作為主要入口（情感路徑）
- 新增快速操作區（農田收成、煉金製作、倉庫總覽）
- 玩家可選擇「經由角色」或「走快速路」

**優點**：保留原架構，Dark Room 感透過快速操作實現
**缺點**：玩家會傾向走效率路 → 角色接觸頻率反而下降，反而放大原問題

### 理解 B：設施從村莊場景直接點擊（雙路徑並存）

主畫面改為「村莊場景 + 可點擊設施 + 角色按鈕並列」。

**實施**：
- 點設施 → 進入設施操作畫面（Dark Room like）
- 點角色 → 進入角色互動畫面（原架構）
- 兩種路徑並存

**優點**：最接近 Dark Room 手感
**缺點**：與理解 A 相同的問題 — 玩家會走效率路；且接近 Dark Room like 新分支架構，與 DR 原架構本質衝突

### 理解 C：角色「在場」的設施操作

進入設施時，對應或隨機的角色出現在現場，設施操作 + 與在場角色自然互動。

**實施**：
- 保留「從角色入口進功能」的原架構
- **新增**：功能畫面中「角色在場」的視覺呈現（她站在旁邊、在忙、搭話）
- 設施操作時會「撞見」對話、反應、閒聊
- 操作過程中可能觸發角色主動說話（與機制 1 聯動）

**優點**：
- 保留「日常生活一部分」的核心感受
- 加入 Dark Room 的直接操作感（動作輕快、不一定要進入選單）
- **設施操作本身變成互動機會** → 角色在勞動現場的樣貌是最日常的情境
- 玩家感受：「我在煮飯，她走過來問一句」的自然感

**缺點**：
- 每個功能畫面都需要角色的「在場 View」（美術工作量增加）
- 角色 AI 行為設計（誰在哪、何時出現）
- 與現有「功能懸浮覆蓋」UI 規則衝突 → 需要重新設計功能畫面佈局

### 哪一種是製作人的本意？

工作室無法替製作人決定。**需請製作人釐清「日常設施操作 dark room like」具體指哪一種**：
- A：純功能捷徑
- B：村莊場景雙路徑
- C：設施操作中的角色在場
- D：上述以外的其他理解

## 工作室觀察：兩個機制的組合效應

**若選擇 機制 1（主動發話紅點）+ 機制 2 理解 C（角色在場設施操作）**，兩者會產生強烈協同：

- 主動發話解決「角色被動性」
- 角色在場操作解決「操作過程情境化」
- 兩者一起實現 Dark Room 的「事件驅動驚喜」+ DR 原架構的「日常接觸頻率」
- 不需放棄任何既有資產

**若選擇 機制 1 + 機制 2 理解 A/B**，則機制 2 的效率路會部分抵銷原架構的連結優勢，實際效果可能不如想像。

這是工作室觀察，**非設計決定**。

## 補充與演變

### 2026-04-17 第二輪：機制 2 具體化為「委託制」

製作人明確指示機制 2 的含義：**結合 Dark Room 的「點擊取得物資」+「拜託工人工作」**。

具體形式：**玩家把物資交給角色 → 拜託他們工作 → 取得產出**。

範例：
- 種子 → 交給農女 → 拜託幫忙種植 → 取得作物
- 拜託守衛 → 幫忙收集物資 → 取得基礎物資（例如種子）

此即**委託制（Commission System）**，與工作室先前識別的理解 A/B/C 皆不同，是**第四種理解（理解 D）**：
- **不是玩家直接操作設施**（A/B）
- **不是玩家路過碰到角色**（C）
- **而是角色成為物資轉換的唯一介面** — 所有設施都透過「拜託角色」達成

### 委託制的核心結構

```
玩家持有物資（或想要某物資）
  → 找到對應角色
    → 交付物資（input）+ 拜託工作
      → 角色執行（等待時間 / 即時 / 探索次數）
        → 完成後角色主動回報（與機制 1 紅點聯動）
          → 玩家前往領取產出（output）
```

### 委託制解決的結構問題

| 問題 | 委託制如何解決 |
|---|---|
| 「角色-功能強綁」（4/16 識別） | 設施存在但**介面永遠是角色**，角色不被降格為「設施操作員」 |
| 「角色被動性」（原架構痛點） | 委託完成時角色主動回報（紅點），角色變主動 |
| 「日常生活感消失」（Dark Room like 痛點） | 每次要物資都要「找人」，角色回到日常中心位置 |
| 「設施概念移除」的兩難（4/16 決策與 4/17 ICON 回歸的衝突） | 設施仍在（農田、煉金台、採集路線），但**不是點擊對象**；角色是點擊對象，設施是背景的勞動場所 |
| 「派遣機制與角色互動脫節」（Dark Room 方向第三輪） | 委託本身就是互動 — 派遣=委託，派遣過程=情感累積機會 |
| 「物資資源競爭失衡」（事前驗屍風險 #3） | 委託的交付/產出流量可成為經濟平衡的控制點 |

### 委託制的設計潛力

1. **委託時刻 = 自然互動**：每次委託都是一次完整的角色接觸（交付 → 對話 → 等待 → 領取 → 感謝）
2. **設施 + 角色雙層設計**：設施決定「能做什麼」，角色決定「怎麼做、付出什麼代價」
3. **角色差異化可體現在委託上**：
   - 農女的委託可能是溫柔耐心（作物慢慢成長）
   - 守衛的委託可能是乾脆直接（探索歸來直接給物資）
   - 魔女的委託可能是神秘苛刻（煉金需要奇怪的材料）
   - 村長夫人的委託可能是長者的叮嚀（倉庫整理時給建議）
4. **好感度可影響委託效率**：好感度高 → 時間短 / 產出多 / 額外反饋 → 經營-戀愛融合
5. **Dark Room 式「等待」節奏**：委託完成前玩家可去做別的事（探索、跟別人對話），形成 Dark Room 的並行節奏

## 委託制的設計張力點（供製作人決定）

### 張力 1：專業化 vs 通用化

**專業化（角色專精）**：
- 每個角色只能接特定類型委託（農女種植、守衛採集、魔女煉金）
- ✅ 角色個性強、與 character-spec 既有設定完美對應
- ✅ 玩家明確知道找誰做什麼事
- ❌ 回歸「功能綁角」（4/16 已識別的問題）但介面變委託制可能改善
- ❌ Dark Room 的勞動力通用感消失

**通用化（勞動力模型）**：
- 任何角色可接任何委託，角色差異在效率/回饋
- ✅ Dark Room 式的派遣策略深度
- ✅ 與第三輪 Dark Room 方向（「角色=通用勞動力」）一致
- ❌ 角色個性被抹平（誰都能種田？）
- ❌ character-spec 既有設定（農女懂種植、魔女懂煉金）失去獨特性

**混合（專業主項 + 通用基礎）**：
- 每角色有「擅長項」（高效/獨特產出）+ 「通用項」（基礎協助）
- ✅ 角色個性保留 + 派遣策略深度
- ✅ 既有 character-spec 可完整沿用
- ❌ 系統複雜度提高
- ❌ 需設計「擅長 vs 通用」的效率差異

### 張力 2：委託時間機制

- **即時結算**：交付立即產出（Dark Room 式點擊感，但無等待張力）
- **現實時間等待**：放下手機等 10 分鐘（手遊感，但 Steam PC 玩家可能反感）
- **探索次數制**：出去探索 1~N 次後回來領（與探索循環綁定，製作人 4/16「升級等待≈探索時間」節奏一致）
- **混合制**：簡單委託即時、複雜委託需多次探索

### 張力 3：委託物資的初始來源

「拜託守衛幫忙收集基礎物資（例如種子）」 — 這是個重要設計點：
- 守衛的收集**不需要玩家先交付物資**（空手拜託）
- 或是需要某種「工具/裝備」才能派遣

**空手派遣模型**：
- 守衛 = 基礎物資產生器（種子、基礎素材）
- 農女 = 中階物資加工器（種子→作物）
- 魔女 = 高階物資加工器（素材→寶石/金工品）
- 形成**物資階梯**：守衛 → 農女/魔女 → 玩家送禮 → 好感度 → 更多委託可用

**需玩家先取得某物才可委託的模型**：
- 玩家必須先探索取得「工具」「裝備」「情報」才能派遣
- 增加探索價值（探索是委託的前置條件）

### 張力 4：玩家自己探索的價值

若守衛可收集基礎物資，玩家自己探索收集什麼？
- **選項 A**：玩家探索 = 高風險高回報（稀有素材、角色遭遇、HCG 觸發）；委託 = 穩定低產出
- **選項 B**：玩家探索 = 地圖揭露、戰鬥樂趣、情報；委託 = 物資補給
- **選項 C**：玩家探索 = 委託無法取得的特殊材料；委託 = 日常消耗品
- **這個關係決定探索系統的定位**

### 張力 5：同時委託數量

- **單一委託**（一次只能派一個角色工作）：經營張力強、選擇有重量
- **多委託**（每角色可同時進行）：經營深度深、Dark Room 風格
- **階段式解鎖**（初期單一，好感度 / 村莊成長後解鎖多委託）：漸進式節奏

## 與 dark-room-clone-direction.md 的整合可能性

委託制可成為**統一 DR 原架構與 Dark Room 照抄方向的橋樑**：

| Dark Room 方向（第三輪）設計 | 委託制如何整合 |
|---|---|
| 5~8 位通用勞動力 | 委託制可保留「4 位深度 + 1~4 位輕量委託用角色」混合 |
| 空村開局 + 5 分鐘首人加入 | 可保留 — 首人加入時示範第一個委託 |
| 設施旁顯示角色 ICON | ICON = 正在該設施上執行委託的視覺回饋 |
| 派遣 = 角色不離村 | 委託 = 派遣的戀愛養成版本（透過互動介面而非列表派遣） |
| 探索女角色獎勵（不可互動） | 可整合或保留 — 探索女角色帶回 = 新委託角色解鎖 |

**因此委託制不是 Dark Room 方向的替代，而是實現方式**：
- 派遣的「形式」從列表勾選 → 委託互動（有交付、對話、領取的完整流程）
- 角色的「定位」從勞動力 → 能與之互動的勞動夥伴

### 優先級 B（影響系統運作，次之決定）

#### B1. 同時委託數量

- **單一委託**（全局同時只能 1 個）— 經營張力強、但 4 人閒置顯得浪費
- **每角色各 1 個**（4 人可並行）— 符合專業化、Dark Room 多工感
- **全局 2 個**（中間值）
- **好感度解鎖式**：初期 1 個 → 好感度門檻後可並行

#### B2. 玩家自己探索 vs 守衛採集委託的價值差異

守衛可採集基礎物資後，玩家親自探索的獨特價值：

- **選項 A**：玩家探索 = 稀有素材 / 深探高風險；守衛委託 = 基礎物資穩定產出
- **選項 B**：玩家探索 = 主動觸發事件（採集點開啟、撤離張力）；守衛委託 = 省時穩定

#### B3. 好感度對委託的影響

- **產出量影響**（好感高 → 產出多）
- **等待時間影響**（好感高 → 時間短）
- **解鎖新委託類型**（好感門檻 → 解鎖特殊委託）
- **額外 flavor 對話**（好感高 → 完成回報對話更親密）
- 上述多選或單選？

#### B4. 「升級」概念的處置

- **取消升級概念**（委託效率由好感度影響）
- **設施升級**（解鎖新委託類型，物資投入設施而非角色）

#### C3. IT 版本實作範圍

### 🔑 重大洞察：委託不是新系統，是既有功能的敘事升級

製作人重新定義「委託」—— 這個定義改變整個系統的實作複雜度：

**委託 = 既有設施功能 + 敘事包裝 + 倒數計時**

換言之：
- 委託**不是**新增的派遣/招募系統
- 委託**就是**玩家既有的設施操作（種植、煉金、採集），**只是透過角色介面**完成
- 多一層「交給我吧」的角色回應作為敘事包裝
- 時間機制沿用 / 擴展既有的計時系統

### 技術實作衝擊（大幅簡化）

| 項目 | 原先預期 | 重新定義後 |
|---|---|---|
| 委託系統 | 需新增 `CommissionManager` + 狀態機 + 派遣計時 | **不需要新系統**，擴展既有 `FarmManager` 概念到其他設施即可 |
| 委託 UI | 需設計獨立委託清單 + 進度面板 | **沿用各設施現有 UI** + 加入敘事包裝層 |
| 委託時間 | 需新增計時系統 | **FarmManager 已有** `SeedData.GrowthDurationSeconds`，擴展到其他設施即可 |
| 交付流程 | 需新增交付/領取流程 | **FarmManager 已有** Plant/Harvest 流程 |

### 村長夫人分離與任務按鈕的影響

製作人指示「多一個任務按鈕」— 這改變了主線任務的入口設計：

**原先預期**：主線任務由村長夫人承接（玩家進入村長夫人 → 看到任務）

**重新定義後**：任務按鈕是**獨立的全局 UI**

- 邏輯：新任務可承接時亮紅點
- 點擊後顯示：可承接的任務清單

## ⚠️ 命名衝突提醒：守衛的「探索」與既有探索系統同名

### 衝突說明

ProjectDR 既有**探索系統**為核心玩法之一：
- 玩家親自進入森林地圖（自由移動、格子揭露、戰鬥、撤離）
- 既有程式碼：`ExplorationEntryPoint`、`ExplorationAreaView`、`ExplorationEntryManager`
- Hub 畫面的「探索」入口 = 玩家親自探索

若守衛的功能按鈕也叫「探索」，會產生語意重疊：
- 玩家親自行動 = 探索
- 委託守衛行動 = 探索
- 兩者同名

### 工作室識別的可能解法

- **選項 A**：守衛的「探索」改用其他詞
  - 「採集」（強調產出）
  - 「巡邏」（強調守衛職能）
  - 「外出」（強調離村）
  - 「託付」（強調委託感）
- **選項 B**：保留兩者都叫「探索」，透過上下文區分
  - Hub 主畫面的「探索」= 玩家親自出發

## 附錄：與 Dark Room 的差異分析（2026-04-17 製作人反思問題）

### 根本性差異（本質不同）

| 面向 | Dark Room | ProjectDR 目前設計 |
|---|---|---|

| 面向 | Dark Room | ProjectDR |
|---|---|---|
| 工人機制 | 分配職業，只是數字 | 委託角色（敘事包裝「交給我吧」） |
| 等待節奏 | 建築工時實時 | 委託等待 10 秒 ~ 一次探索時間 |
| 資源系統 | 多層鏈 | 食物線 + 禮物線 |
| 地圖 | 網格 + 補給 | 網格揭露 + 自由移動 + 撤離 |
| 節奏揭露 | 30 秒第一個陌生人 | 主線任務引導（較慢） |

### 真正借鑒到的精髓

1. **漸進揭露節奏哲學** → 主線任務作機制引導
2. **委託等待的經營節奏** → 派工 + 做別的 + 回收
3. **事件驅動的驚喜感** → 紅點取代文字跳出
4. **單物品配方的簡潔** → 種子→作物

### 刻意沒借鑒的部分

1. **極簡美學** — 違反成人遊戲視覺本質
2. **角色無個性化** — 違反戀愛養成核心
3. **科幻敘事翻盤** — 不適合本作基調
4. **補給系統** — 已有撤離類機制
5. **多層資源鏈** — 已有兩條線

### 核心定位

**「借的是哲學，不是實作」**

ProjectDR = Dark Room 的「漸進揭露 + 委託等待」節奏哲學，嫁接到戀愛養成的角色深度與撤離類探索玩法上。

### 此分析對後續設計的意義

- 若未來想「加更多 Dark Room 元素」，應檢查是否與戀愛養成/立繪 CG 本質衝突
- 若未來想「強化節奏感」，可回到漸進揭露哲學找靈感
- 若未來感到「不夠 Dark Room」，可能是漸進揭露的節奏不夠快（而非需要抄更多具體機制）

### 紅點觸發的兩種情境

基於此決定，紅點可以由兩種事件觸發：

| 觸發事件 | 對話類型 | 舉例 |
|---|---|---|

| 角色發問 | **角色發問**（角色想問玩家一個問題） | 「你最近怎麼都待在森林裡？」 |

這兩種紅點對話是**不同的觸發源**，可並存也可輪替：
- 若委託剛完成 + 有新的角色發問待觸發 → 紅點亮，進入後可能先播完成對話再播角色發問
- 或由系統依優先序決定先播哪個

### 玩家發問的 UX 流程（工作室提案）

玩家點擊[對話]進入後，可能呈現為：

```
點擊[對話]

    → 顯示可問的問題清單（40 題，依好感度分批解鎖）
      → 玩家選一個問題
        → 角色回答（打字機）
          → 繼續選問題 或 退出對話
```

類似 Visual Novel 的「question tree」結構，但需製作人確認具體 UX。

### 製作人觀察

> 如果是漸進揭露，一開始就開放四個 NPC 好像不行，不論如何都要想一個方法逐一介紹 NPC，最理想的方式是參考 Dark Room 跟 HuniePop，解鎖之後玩家自己決定下一步。

### 工作室承認：設計張力成立

- 本設計借鑒 Dark Room 的**漸進揭露**哲學作為主要精髓
- 但當前定案「玩家醒來時村子已有 4 人」 — **違反漸進揭露**
- 與 4/15 決策「NPC 漸進式解鎖，固定順序：村長夫人→農女→魔女→守衛」也不一致（該決策在 Phase D5 定案時被實質推翻）

此張力**應修訂當前設計**。

### 參考遊戲機制分析

**Dark Room**：開局空村 → 時間驅動自動出現（30 秒 ~ 1 分鐘首位）→ 事件驅動揭露

**HuniePop**：有引路人（Kyu）→ 條件式解鎖 → 玩家自由選擇攻略順序

**共通精神**：解鎖 = 開放新選項，不強制推進。

### 三種融合模型

#### 模型 α：時間驅動（純 Dark Room）
```
開局 1 位 → 時間自動觸發其他加入（5/15/30 分鐘）
```
- ✅ 最純漸進揭露節奏
- ❌ 玩家參與度低、缺少敘事理由

#### 模型 β：條件解鎖（純 HuniePop）
```
開局 1 位 → 達成條件 → 解鎖下一位
例：探索次數、主線任務、物資門檻、倉庫等級
```
- ✅ 玩家參與感強、敘事合理
- ❌ 需避免解鎖阻塞

#### 模型 γ：探索遇見（兩者融合）
```
開局 1 位在村裡
  → 玩家探索森林 → 在某區域遇到角色（劇情事件）
    → 劇情觸發 → 她來到村莊 → 村莊新增該角色
```
- ✅ 兼具 Dark Room 驚喜感 + HuniePop 玩家主動感
- ✅ 探索獎勵層次提升（不只物資，也有角色）
- ✅ A2 取消後以既有 4 位角色為對象的新形式復活
- ✅ 敘事合理化容易（角色在森林各有理由）
- ❌ 需設計探索地圖的相遇事件

### 工作室觀察

模型 γ 最整合既有設計：保留 4 位角色、保留探索系統、保留漸進揭露精神、不新增陌生人。

### 對既有 GDD 的影響

|---|---|
| `characters.md` | 加入方式：開局皆在 → 逐一加入 |
| `world-setting.md` | 開局狀態：為什麼只有 1 位？ |
| `core-definition.md` | 次循環 D（感情循環）的啟動方式 |

## 2026-04-17 第十三輪：製作人提案「交錯式劇情節拍」

### 製作人提案

> 村長夫人 → 玩家選擇 → 村長夫人 → 玩家選擇 2 → 村長夫人 → 最後一位

### 工作室理解的結構

```
節點 1：村長夫人（開局就在 / 第一個加入）
       ↓ 主線劇情事件
節點 2：玩家選擇 — 從 3 位候選中選 1 位加入
       ↓ 村長夫人相關劇情節點
節點 3：玩家選擇 2 — 從剩下 2 位中選 1 位加入
       ↓ 村長夫人相關劇情節點
節點 4：最後一位（自動加入，劇情固定）
```

### 此提案對應到四個已定案角色

- **村長夫人**：開局就在（主線承載者完全契合）
- **玩家選擇 1**：農女/魔女/守衛 → 選 1 加入
- **玩家選擇 2**：剩下 2 位 → 選 1 加入
- **最後一位**：剩餘 1 位自動加入

### 此方案同時實現兩個核心需求

| 需求 | 如何實現 |
|---|---|
| **漸進揭露**（Dark Room） | 固定節拍 4 位逐一加入，每位都是新的驚喜 |
| **玩家自由選下一步**（HuniePop） | 中段兩次玩家選擇，決定第 2、3 位加入順序 |
| **主線敘事串聯** | 村長夫人作為每次節點的引路人（類似 Kyu） |
| **開場體驗一致** | 村長夫人固定開局，所有玩家相同起步 |
| **收尾完整性** | 最後一位固定，劇情必然收束 |

### 設計優雅性分析

1. **固定節點 + 自由節點的交錯** — 既不是全自由（不可控）、也不是全固定（無主動感），而是兩者交織
2. **村長夫人的密集出現** — 強化她作為「主線承載者」的定位（與先前定案一致）
3. **最後一位的戲劇性** — 留下懸念，可能是最需要敘事準備的角色
4. **與既有 4 角色完美契合** — 不新增不刪減

### 與三個融合模型的關係

此提案**本質上是模型 β（條件解鎖）與模型 γ（探索遇見）的混合變體**：
- 村長夫人節點 = 條件觸發（主線推進）
- 玩家選擇 = 探索去哪遇見誰的選擇（可接軌模型 γ）
- 最後一位 = 劇情觸發自動加入

工作室先前觀察模型 γ 最整合既有設計 — **此提案進一步升級了模型 γ**：加入固定劇情節拍作為骨架，讓自由度被敘事結構包裹，避免純自由探索可能的節奏混亂。

## 🆕 第十三輪需釐清的細節

### MA：玩家選擇的呈現方式

- **選項 A — 明確分歧**：村長夫人直接給選項
  - 例：「森林南邊有位農女需要幫助，森林東邊有位守衛遇到困難，你先去救誰？」
  - 優點：選擇感明確
  - 缺點：不像探索遇見，較像 VN
- **選項 B — 探索引導**：村長夫人提示方向，玩家主動探索
  - 例：「森林裡可能還有其他人需要幫助」→ 玩家探索各區域 → 遇到誰就解鎖誰
  - 優點：保留探索遇見驚喜
  - 缺點：若玩家一路只去一個方向，另一個永遠沒遇到
- **選項 C — 時間平行**：多個角色可被發現，誰先遇到就先加入
  - 優點：極高自由度
  - 缺點：順序失去意義

**工作室觀察**：選項 A 最貼近「玩家選擇」語意，選項 B 最貼近模型 γ 探索遇見，選項 C 失去敘事控制。

### MB：村長夫人節點的劇情內容

村長夫人出現 3 次（節點 1、節點 3、節點 5 之間），每次提供什麼？
- 節點 1：開場 + 說明世界觀 + 介紹第一次選擇
- 節點 3：第一次選擇完成後的反應 + 引入第二次選擇
- 節點 5：第二次選擇完成後的反應 + 引入最後一位

每次節點的內容類型？
- 純劇情（只是對話，推進敘事）
- 劇情 + 解鎖新機制（搭配主線任務 = 機制引導）
- 劇情 + 物資/獎勵

### MC：最後一位是誰？

這是製作人的設計選擇 — 哪位角色最適合「最後加入」？

工作室觀察與候選：
- **魔女**：神秘、冷淡、不輕易信任人 → 最後才信任玩家（敘事張力強）
- **守衛**：最警戒、防衛心重 → 最後才接納外人
- **農女**：樸實、容易親近 → **不適合**最後才加入

**工作室觀察**：魔女或守衛都合理，具體由製作人決定。

### MD：玩家選擇的不可逆性

- 選擇後另一位會**暫時跳過**（下一輪再選）？還是**永久跳過**（要等最後一位來）？
- 工作室理解：若是「選擇 1 → 選擇 2 → 最後一位」的結構，選擇 1 沒選的那位會在選擇 2 時仍可選，這是**延後**而非**跳過**。

### ME：節點之間的間隔

每個節點之間玩家要做什麼？
- **時間間隔**：每個節點相隔遊玩時間 30 分鐘~1 小時
- **條件間隔**：完成特定任務/探索/物資後推進
- **混合**：既有時間下限（太快不推進），也有條件上限（達條件即推進）

**工作室觀察**：混合制最能控制節奏與玩家參與感。

### MF：最後一位加入的觸發

最後一位自動加入 — 觸發條件是什麼？
- 玩家選擇 2 完成後的時間/探索延遲
- 特定主線任務完成
- 最後一位的個別劇情事件（例：她遇到困難主動求助、被發現遭遇魔物需要救援等）

### MG：與主線任務系統的整合

當前 `main-quest-system.md` 定位任務為「機制引導」，此 4 位解鎖節點如何對應？
- **選項 A**：每個節點 = 一個主線任務（T1 村長夫人 → T2 解鎖第 2 位 → T3 村長夫人推進 → T4 解鎖第 3 位 → T5 村長夫人推進 → T6 最後一位）
- **選項 B**：4 位解鎖獨立於主線任務（角色解鎖 = 敘事觸發，主線任務 = 機制引導，兩條線並行）
- **選項 C**：混合（解鎖有主線任務引導，但解鎖本身不算主線完成）

### MH：前期機制引導在哪裡

若開局只有村長夫人，委託制（農女/魔女/守衛）要等到第 2 位加入才解鎖 — 前期玩家能做什麼？

- 主線任務首個引導可能是：「去森林探索看看」→ 玩家自己探索 → 帶回物資 → 倉庫變有用 → 遇到第 2 位

## MA-B 探索引導的完整運作方式（工作室設計，供製作人檢視）

### 基本流程

```
村長夫人給出提示
  ↓
玩家進入探索（親自從 Hub「探索」入口出發）
  ↓
探索過程中某處觸發遭遇事件
  ↓
播放登場 CG + 短劇情
  ↓
劇情結束 → 角色加入村莊
```

### 運作細節（工作室提案）

#### 1. 村長夫人的提示形式

**方向性提示（推薦）**：
- 第一輪：「我聽說森林裡最近有動靜…南邊和東邊都有人說見過人影」
- 這種提示**不指明是誰、不指明確切地點**，但給玩家兩個候選方向
- 玩家選擇要先探索哪裡 = MA 的「玩家選擇」

**為什麼不用精確提示**：
- 精確（「南邊有農女在求救」）→ 太像任務清單、失去探索感
- 模糊方向 → 保留驚喜（玩家到達時才知道是誰）、保留自主（選去哪）

#### 2. 候選角色的分布

- 第 1 輪玩家選擇：**農女、魔女**各分布於森林的不同區域（例：南邊農女、東邊魔女）
- 玩家選擇去南邊探索 → 在探索過程中遇到農女（觸發她的遭遇事件）
- 沒選的那位（魔女）**保留在原位**，第 2 輪仍可被遇到

#### 3. 遭遇事件的觸發

**觸發條件（可選擇一種或組合）**：
- **地圖區域**：玩家進入特定地圖區塊時觸發
- **格子觸發**：玩家踩到特定格子時觸發（類似遇敵）
- **條件組合**：採集到特定物品後才觸發，或擊敗區域魔物後才觸發

**工作室推薦：地圖區域觸發 + 任意格子**
- 進入目標區域後，玩家在該區域任意探索 → 第 1 次翻開特定格子觸發
- 理由：玩家有參與感（需真的探索），但不刁難（不用精確定位）

#### 4. 遭遇事件的敘事類型

不同角色的登場事件可以有不同敘事情境：

| 角色 | 可能的登場情境 |
|---|---|
| **農女** | 在田地/空地被魔物包圍，玩家擊退魔物救援；或玩家發現她在採藥草，她對玩家警戒再親近 |
| **魔女** | 玩家發現她在森林深處的小屋/煉金場做實驗，意外的相遇；或她在研究某種魔物，玩家協助 |
| **守衛** | 「外部探索歸來攻擊玩家」的戲劇性出現（特殊：是最後一位，固定觸發） |

#### 5. 玩家選擇 2 的處理

- 第 1 輪選擇結束（如玩家先去南邊遇到農女）
- 回村 → 村長夫人節點（反應 + 新提示）
- 第 2 輪玩家探索 → 去另一方向（東邊）→ 遇到魔女

**若玩家又去同方向會怎樣**：
- 該區域已無未解鎖角色 → 只有物資/魔物
- 玩家會自然想去另一方向找新人（好奇心驅動）
- 若玩家固執不去 → 村長夫人提示會更明確（逐步收斂自由度）

#### 6. 守衛的特殊觸發（最後一位）

守衛不走上述「玩家選擇」流程，而是**固定觸發**：
- 前兩位都加入村莊後
- 村長夫人第 3 次節點暗示「還有人在外面」

- 戰鬥/救援事件 → 登場 CG → 守衛加入

## 守衛「外部探索歸來攻擊玩家」的劇情解讀

工作室分析幾種可能的劇情包裝（供製作人選擇或修改）：

### 選項 A：精神污染（推薦）

守衛在村莊外部探索時，遭遇森林中的某種異常（魔物、詛咒、未知現象），精神被污染。歸來時已失去理智，將玩家視為威脅而攻擊。

- ✅ 強化「森林危險」世界觀（原設定：被不斷變化的森林圍困）
- ✅ 符合守衛「最警戒」人設（她知道外面的危險）
- ✅ 戲劇張力強（戰勝後她恢復 → 感激 + 羞愧 + 最終信任玩家）
- ✅ 為世界觀埋下鉤子（外面到底有什麼？）

### 選項 B：身份誤會

守衛剛回到村莊，不知道玩家已經存在。她將玩家視為入侵者而攻擊。

- ⚠️ 較輕度，戲劇性較弱
- ✅ 符合守衛職責（保護村莊）
- ❌ 為何她不在之前就回來？（時序問題）

### 選項 C：魔物偽裝

攻擊玩家的並非真守衛，而是模仿她外形的魔物。擊敗偽裝後，才在更深處發現真守衛。

- ✅ 敘事複雜，有謎團
- ❌ 涉及「怪物可變身」的世界觀設定，與既有 world-setting 落差
- ❌ 可能過度複雜化登場劇情（不「很短」）

### 工作室建議：選項 A

A 最能強化既有世界觀、最有戲劇張力、最符合守衛人設。但劇情包裝由製作人決定。

### 工作室之前的誤解

工作室在第十三/十四輪將「解鎖」理解為「玩家在森林探索遇見角色 → 帶回村莊」（模型 γ），但製作人的真意是：

### 🔑 關鍵設計洞察：探索功能本身也是漸進解鎖

此補充揭示一個**更深的漸進揭露結構**：**探索功能不是開局就有**，而是需要透過主線推進解鎖。

這完美對應 Dark Room 的解鎖節奏：
- Dark Room: stoke fire → gather wood → check traps → hunt → **go to the forest**（探索解鎖）
- ProjectDR: 村長夫人 → 認識農女/魔女 → 委託循環 → **探索功能開放**（解鎖守衛）

### 重新理解：守衛登場的時序

```
階段 1：開局（只有村長夫人）
階段 2：對話選擇 1 → 解鎖農女/魔女其一
階段 3：村長夫人節點 1
階段 4：對話選擇 2 → 解鎖第二位
階段 5：村長夫人節點 2 → 推進至「探索功能開放」的觸發
階段 6：【探索功能開放】→ 玩家首次點擊探索入口 → 進入森林
階段 7：【守衛歸來事件】在首次探索中觸發 → 戰鬥 → 守衛解鎖
```

**守衛的解鎖 = 探索功能的第一次體驗**，兩者綁定。

### 設計優勢

1. **真正的漸進揭露**：每個新機制都伴隨新角色/新系統
2. **Dark Room 節奏**：對應原作「go to the forest」是後期解鎖
3. **戲劇性強化**：玩家第一次出森林就遇到攻擊，衝擊感強
4. **世界觀合理**：村莊被森林圍困，探索森林是危險的（符合守衛精神污染的背景）
5. **教學自然**：第一次探索就遇到戰鬥，自然教會玩家戰鬥系統

### 節點的定義

**「節點」= 村長夫人在劇情中的每次主動推進時刻**。製作人原提案結構為「村長夫人 → 選擇 → 村長夫人 → 選擇 2 → 村長夫人 → 最後一位」，其中**三次村長夫人**即三個節點：

| 節點 | 位置 | 內容 |
|---|---|---|
| **節點 0**（開局） | 玩家醒來第一次見村長夫人 | 介紹世界觀 + 觸發選擇 1 |
| **節點 1** | 選擇 1 後累積條件達成，再訪村長夫人 | 對新角色加入的反應 + 推進主線 + 觸發選擇 2 |

節點 0 之後、節點 1 之前、節點 2 之前的時段，玩家在做**日常循環**（與已解鎖的角色互動、委託、累積）。
