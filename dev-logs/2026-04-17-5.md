# Dev Log 2026-04-17-5

## 概述

Sprint 4 試玩第三輪回饋：(1) 搜索應為「倒數結束後才加木材」(2) 蓋小屋要顯示進度。

## 需求 1：搜索改為「冷卻結束才結算」

**製作人意圖**：原本點搜索立刻 +1 木材、再等 1 秒冷卻。應改為「點搜索 → 翻找中倒數 → 結束才給木材」，符合「翻找」的敘事感。

**修改 `SearchSystem.cs`**：
- 新增 `_pending` 與 `_pendingFeedback` 狀態
- `TrySearch()`：啟動冷卻 + 登記 pending + 預先挑選 feedback line（不加木材、不發事件）
- 新增 `Tick(deltaSeconds)`：若 pending 且冷卻已結束，結算（加木材 + 發 MvpSearchCompletedEvent）
- `CanSearch` 改為 `!_pending && !IsOnCooldown`
- 新增 `IsPending` 查詢

**修改 `MvpEntryPoint.cs`**：
- Update 中新增 `_searchSystem?.Tick(dt)`，執行順序在 ActionTimeManager.Tick 之後

**測試更新**：
- `SearchSystemTests`：
  - 舊 `TrySearch_AddsWood` / `TrySearch_PublishesEventWithFeedback` 改為 `TrySearch_PendingUntilCooldownEnds` + `Tick_AfterCooldown_AddsWoodAndPublishesEvent`
  - 新增 `Tick_BeforeCooldownEnds_DoesNotResolve`（冷卻未結束 Tick 不結算）
  - 舊 `TrySearch_OnCooldown_Fails` 改為 `TrySearch_WhilePending_Fails`
  - 舊 `TrySearch_AfterCooldown_Succeeds` 改為 `TrySearch_AfterResolve_Succeeds`（需 Tick 結算後才能再次 TrySearch）
  - 寒冷倍率測試補上 `_sut.Tick()` 呼叫驗證最終加木材
- `MvpIntegrationFlowTests.AdvanceTime`：新增 `_search.Tick(dt)` 呼叫，讓整合測試模擬 MvpEntryPoint.Update 的完整 Tick 順序

## 需求 2：蓋小屋進度顯示

**問題**：Scene 中 HutBuildProgress Slider GameObject 的 `m_FillRect/m_HandleRect/m_TargetGraphic` 均為 `{fileID: 0}`（空殼 Slider 無視覺子物件），value 變化完全看不見。

**修復策略**：不重建 Slider 結構（手寫 YAML 複雜度高），改用既有 `_feedbackLabel` 顯示進度文字。

**修改 `MvpMainView.cs`**：
- Update 中新增 `RefreshHutProgressLabel()`：若 `_hutBuildSystem.IsBuilding`，`_feedbackLabel` 文字覆蓋為 `蓋小屋中...（剩 X.Xs）`
- `OnHutBuilt`：完成時將 `_feedbackLabel` 設為「小屋蓋好了，人口上限提升。」

**體驗流程**：
1. 點蓋小屋 → feedback label 變「蓋小屋中...（剩 9.8s）」並每幀倒數
2. 完成 → feedback label 變「小屋蓋好了，人口上限提升。」
3. NPC 來訪（既有 `MvpNpcArrivedEvent` → MvpEntryPoint 自動開 interact view）

搜索 feedback 與蓋屋進度共用 `_feedbackLabel`，實務上兩者不會同時發生（蓋屋期間通常不搜索，且搜索 feedback 被蓋屋文字覆蓋也合理）。
